# SCAFFOLD — author a new aion workflow package

You are authoring an aion workflow package: a Gleam project plus a `workflow.toml`, compiled and packaged into one `.aion` archive the server loads. This file is your walkthrough. Do not write from memory — the SDK and tooling described here are the real surface as of this repo; still CHECK the specifics against the live tree when a step says so.

Ground-truth references in the aion repo (read them if a detail is unclear): `examples/hello-world` (minimal, hand-written codecs), `examples/order-saga` (types-first canon: `_io` + `_activities` + generated `_codecs`/`_activity_wrappers`), `examples/agent-dev` and `examples/assistant` (production canon), `docs/packaging.md`, `docs/guides/codegen.md`.

## 0. Preflight — CHECK tooling, never assume

Run these first. Do not proceed on a tool you have not confirmed.

```sh
command -v gleam && gleam --version      # Gleam toolchain (required to build)
command -v aion  && aion --help          # aion CLI (package/generate/start/server)
command -v git   && git --version
```

- If `gleam` is absent: you cannot build or package. Stop and report it.
- If `aion` is absent: you can still author + `gleam build`/`gleam test`, but you cannot run `aion generate`, `aion package`, or deploy. Report which subcommands you actually have — `aion generate --help`, `aion package --help` — since surfaces evolve.
- The `aion_flow` SDK is a path dependency (`../../gleam/aion_flow`) inside the aion repo tree. In a standalone package outside that tree, CHECK how the operator wants `aion_flow` resolved (path vs published) before writing `gleam.toml`.

## 1. Package layout

A package is an ordinary Gleam project with one extra descriptor. Types-first canon (recommended — use this unless told otherwise), package name `greeter`:

```text
greeter/
├── gleam.toml                         # Gleam metadata; [dependencies] are load-bearing
├── manifest.toml                      # Gleam lockfile — written by the toolchain, never by hand
├── workflow.toml                      # aion packaging descriptor (§2)
├── src/
│   ├── greeter_io.gleam               # AUTHORED, types ONLY — the source of truth
│   ├── greeter_activities.gleam       # AUTHORED — activity bodies + manifest()
│   ├── greeter.gleam                  # AUTHORED — the workflow (entry_module)
│   ├── greeter_codecs.gleam           # GENERATED by `aion generate` — do not edit
│   └── greeter_activity_wrappers.gleam# GENERATED — typed activity.new wrappers
├── schemas/
│   ├── input.json                     # GENERATED from the Input type
│   └── output.json                    # GENERATED from the Output type
├── test/
│   └── greeter_scaffold_test.gleam    # GENERATED write-once scaffold, then YOURS to fill
├── worker/                            # GENERATED per-tier worker stubs (remote tiers)
└── build/                             # produced by `gleam build`
```

The `<package>` prefix is the `name` from `gleam.toml` (snake_case). `src/<package>_io.gleam`, `src/<package>_codecs.gleam`, `src/<package>_activity_wrappers.gleam` all key off it.

## 2. Verbatim minimal package (copy, then adapt the names)

### `gleam.toml`

```toml
name = "greeter"
version = "0.1.0"
target = "erlang"
description = "Minimal Aion workflow: greet a name via one recorded activity."
licences = ["MIT"]

[dependencies]
aion_flow = { path = "../../gleam/aion_flow" }
gleam_stdlib = ">= 0.34.0 and < 2.0.0"
gleam_json = ">= 2.0.0 and < 4.0.0"

[dev-dependencies]
gleeunit = ">= 1.0.0 and < 2.0.0"
```

`target = "erlang"` is required (the engine runs on the BEAM). `gleeunit` is a dev-dependency and is excluded from the archive.

### `workflow.toml` — every field annotated

```toml
# Packaging descriptor consumed by `aion package`. Ref: docs/packaging.md.
# NOTE: this is the PACKAGING file, unrelated to aion.toml (server config).

[[workflow]]                              # one entry = one .aion archive; repeatable
entry_module   = "greeter"                # REQUIRED. Compiled module the engine calls.
                                          #   THIS IS THE WORKFLOW TYPE clients pass to
                                          #   `aion start` — there is no separate type name.
                                          #   Nested module a/b.gleam compiles to "a@b".
entry_function = "run"                    # REQUIRED. Exported fn the engine invokes.
                                          #   "run" is convention, not a default — state it.
timeout_seconds = 30                      # REQUIRED. Whole seconds, integer >= 1.
input_schema   = "schemas/input.json"     # REQUIRED. Path rel. to project root; must parse.
output_schema  = "schemas/output.json"    # REQUIRED. Same, for the result payload.
activities     = ["greet"]                # REQUIRED. Names workers register. [] is legal;
                                          #   OMITTING the field is an error. Must match
                                          #   the activity names your manifest() declares.
output         = "greeter.aion"           # OPTIONAL. Derived as <entry_module>.aion if absent.
```

Unknown keys are hard errors. Declared paths must resolve inside the project root. When you use `aion generate`, it maintains the `activities` list for you from `manifest()` — CHECK by diffing after generate.

### `src/greeter_io.gleam` — types ONLY (the types-first rule)

```gleam
//// Boundary types for the greeter workflow — the authored source of truth
//// (ADR-014, types-first). Declare TYPES ONLY here. `aion generate` derives
//// the JSON codecs (src/greeter_codecs.gleam) and schemas/*.json from these.
//// A function, constant, or type alias in this module is a LOUD ERROR.

/// The workflow's start input.
pub type Input {
  Input(name: String)
}

/// The workflow's recorded result.
pub type Output {
  Output(greeting: String)
}

/// The workflow's typed error: a stage that could not execute at all.
pub type GreeterError {
  GreeterError(stage: String, message: String)
}
```

Types-first rule: codecs are GENERATED from these types, never hand-written beside them, so encoder/decoder/type can never drift. Supported shapes (CHECK `docs/guides/codegen.md` for the full table): single-constructor labelled records → JSON object; all-zero-arity multi-constructor → JSON string enum; `String/Int/Float/Bool`, `List(t)`, `option.Option(t)` at a field, and references to sibling types in the same module. Everything else (generics, aliases, tuples, `Dict`, cross-module field types, recursive types, unlabelled fields) fails loudly by design.

### `src/greeter_activities.gleam` — bodies + the manifest

```gleam
//// Hand-written activity bodies and the package's activity manifest (ADR-014).
//// manifest() declares each activity ONCE — name, tier, input/output value
//// types — and `aion generate` derives the codecs, the activity.new wrappers,
//// the worker plumbing, and the workflow.toml activities list. Bodies are the
//// deterministic in-VM doubles the test harness runs; in production the
//// activity name is served by a worker.

import aion/activity
import aion/error
import greeter_codecs as codecs
import greeter_io as io

pub fn greet(input: io.Input) -> Result(io.Output, error.ActivityError) {
  Ok(io.Output(greeting: "Hello, " <> input.name <> "! Welcome to Aion."))
}

/// The single source of truth the generator reads. Declaration ORDER is
/// load-bearing: it fixes the wrapper order, the worker registry, and the
/// workflow.toml activities list.
pub fn manifest() -> List(activity.Declaration) {
  [
    activity.declare(
      "greet",
      activity.RemotePython,
      activity.type_ref("Input", codecs.input_codec()),
      activity.type_ref("Output", codecs.output_codec()),
    ),
  ]
}
```

`activities.gleam` references `greeter_codecs` — a generated module. Bootstrap ordering: on a clean tree run `aion generate .` once to emit codecs, then `gleam build`. `aion generate` reads types via the compiler, so it tolerates the not-yet-built codecs call sites during its own type-read pass; if generate refuses on a truly empty tree, CHECK its `--help` for a bootstrap flag rather than guessing.

### `src/greeter.gleam` — the workflow module (define / pure execute / entrypoint run)

```gleam
//// The greeter workflow. This module is the DETERMINISM BOUNDARY: it issues
//// only recorded activity dispatches and branches on their recorded outputs.
//// No wall clock, no entropy, no direct IO.

import aion/error
import aion/workflow
import greeter_activity_wrappers as wrappers
import greeter_codecs as codecs
import greeter_io as io
import gleam/dynamic.{type Dynamic}

/// Typed definition binding the generated codecs to the execute function.
pub fn definition() -> workflow.WorkflowDefinition(
  io.Input,
  io.Output,
  io.GreeterError,
) {
  workflow.define(
    "greeter",
    codecs.input_codec(),
    codecs.output_codec(),
    codecs.greeter_error_codec(),
    execute,
  )
}

/// Engine entry point. `workflow.entrypoint` decodes the raw JSON with the
/// definition's input codec, drives `execute`, and encodes the outcome. An
/// undecodable input yields the SDK's {"aion_error":"input_decode",...} envelope.
pub fn run(raw_input: Dynamic) -> Result(String, String) {
  workflow.entrypoint(definition(), raw_input)
}

/// Pure workflow body: dispatch one recorded `greet` activity, map its result.
pub fn execute(input: io.Input) -> Result(io.Output, io.GreeterError) {
  case workflow.run(wrappers.greet_activity(input)) {
    Ok(output) -> Ok(output)
    Error(activity_error) ->
      Error(io.GreeterError(
        stage: "greet",
        message: error_message(activity_error),
      ))
  }
}

fn error_message(e: error.ActivityError) -> String {
  case e {
    error.Retryable(message: m, details: _) -> m
    error.Terminal(message: m, details: _) -> m
    error.ActivityDecodeFailed(_) -> "activity result could not be decoded"
    error.ActivityTimedOut(error.TimedOut(message: m)) -> m
    error.ActivityCancelled(error.Cancelled(reason: r)) -> r
    error.ActivityNonDeterministic(error.NonDeterminismViolation(message: m)) -> m
    error.ActivityEngineFailure(message: m) -> m
  }
}
```

`workflow.define(name, input_codec, output_codec, error_codec, entry_fn)` — argument order verbatim from the SDK. `workflow.run(activity)` is the ONLY recorded dispatch surface; `workflow.entrypoint(definition, raw_input)` is the one-line engine adapter. CHECK the exact `error.ActivityError` variant list in `gleam/aion_flow/src/aion/error.gleam` before writing the match — it must be exhaustive.

### `test/greeter_scaffold_test.gleam` — aion/testing mocks

```gleam
//// Pure aion/testing scenario: mock each activity, assert the entry function
//// replays deterministically. No engine, store, or beamr needed.

import aion/testing
import greeter
import greeter_activity_wrappers as wrappers
import greeter_io as io
import gleeunit

pub fn main() {
  gleeunit.main()
}

pub fn happy_path_test() {
  let assert Ok(env) = testing.new()

  // One typed mock per declared activity, keyed by the wrapper value.
  let assert Ok(env) =
    testing.mock_activity(env, wrappers.greet_activity(io.Input(name: "Ada")), fn(input) {
      Ok(io.Output(greeting: "Hello, " <> input.name <> "! Welcome to Aion."))
    })

  // Drive the REAL entry fn and assert deterministic replay.
  let result = testing.assert_replay(env, fn() { greeter.execute(io.Input(name: "Ada")) })
  let assert Ok(output) = result
  assert output == io.Output(greeting: "Hello, Ada! Welcome to Aion.")
}
```

Mock every declared activity — a dispatch that reaches an unmocked activity fails the test loudly. `testing.mock_activity(env, activity_value, handler)`, `testing.assert_replay(env, fn)`, `testing.new()` are the real surface; for timers use `testing.advance`, for child workflows `testing.mock_child`. Run with `gleam test`.

## 3. Generate — `aion generate .`

Run after every edit to `_io.gleam` or the `manifest()`:

```sh
aion generate .          # write every derived artifact
gleam build              # compile with the fresh codecs
gleam test
```

It emits: `src/<package>_codecs.gleam` (per type: `<prefix>_to_json`, `<prefix>_decoder`, `<prefix>_codec()`), `src/<package>_activity_wrappers.gleam` (typed `activity.new` wrappers, one per manifest entry: `greet_activity(input)`), one `schemas/<snake_type>.json` per public type, per-tier worker stubs under `worker/` (remote tiers only), a remote wire-compat golden, the `workflow.toml` `activities` list, and a write-once `test/<entry>_scaffold_test.gleam` (never overwritten — yours to fill).

RULES:
- NEVER hand-edit a generated file. Each opens with `//// Generated by aion generate — do not edit`; every schema carries a `$comment` generated marker. Change the TYPE, re-run generate.
- NEVER run a formatter over generated files, and do not commit formatter-reflowed generated output — generation is byte-exact and `--check` compares bytes.
- CI gate: `aion generate . --check` exits non-zero naming any stale/hand-edited artifact. A stray unmarked `schemas/*.json` is always an error.
- Commit the type edit and its regenerated artifacts together.

## 4. Determinism boundary (workflow module only)

`execute` and everything it calls synchronously must be a pure function of the input and the recorded activity outputs. No wall clock, no entropy, no direct IO, no unrecorded concurrency. Only `workflow.run` (and `all`/`race`/`map`, timers, signals, child spawns) cross the boundary and get recorded in history. Violations break replay: on recovery the engine re-runs `execute` and it must produce the identical command sequence.

- VIOLATION: reading the clock. `let now = erlang_now()` inside `execute`. FIX: obtain time only through the SDK's recorded time surface (CHECK `aion/workflow` / `aion/internal/ffi` for the deterministic timestamp accessor), or pass the timestamp in as input, or compute it inside an activity body (activities may touch the real world; their result is recorded).
- VIOLATION: entropy. `let id = random_uuid()` in `execute`. FIX: generate the id inside an activity and return it, so the value is recorded once and replayed identically; never in workflow code.
- VIOLATION: direct IO / network. `httpc.get(url)` in `execute`. FIX: move the call into an activity body and dispatch it with `workflow.run(wrappers.fetch_activity(...))`. The workflow only branches on the recorded result. Same for reading files, env vars, or a database.

## 5. Adding activities

Add a body in `<package>_activities.gleam`, add its `activity.declare(...)` to `manifest()` (order fixes wrapper/registry/`workflow.toml` order), re-run `aion generate .`. Then decorate the generated wrapper value in your workflow with the pipeline API from `gleam/aion_flow/src/aion/activity.gleam`:

- Plain vs agent: both are `activity.declare` entries. An "agent" activity is just one whose worker drives an LLM/harness; in canon its wire contract is one prompt `String` in, one terminal-text `String` out (`codec.json_codec(json.string, decode.string)`), and the workflow composes the prompt and decodes the reply defensively. There is no separate SDK type for agent vs plain.
- Retry policy (no default — an activity runs exactly once unless you attach one):
  ```gleam
  wrappers.greet_activity(input)
  |> activity.retry(activity.RetryPolicy(
       max_attempts: 3,
       backoff: activity.Fixed(delay: duration.seconds(5)),
     ))
  ```
  Backoff variants: `Fixed(delay)`, `Exponential(initial, multiplier, max)`, `Linear(initial, increment, max)`. Also `activity.timeout(a, duration)`, `activity.heartbeat(a, duration)`.
- Execution tier — the `activity.declare` tier drives which worker stub `generate` emits; a runtime `activity.execution_tier(a, tier)` selects the dispatch path:
  - `activity.InVm` — runner executes once, live, in a linked child process of the workflow; no remote worker, no task-queue routing. History and replay stay byte-identical to a remote dispatch.
  - `activity.RemotePython` / `activity.RemoteRust` — remote worker over the worker protocol (the default wire when no tier is selected). Emits a `worker/` stub in that language.
  Tier is routing, never recorded nondeterminism.
- Routing decorators (optional, no defaults): `activity.task_queue(a, name)` and `activity.node(a, name)` (pin to one host in the pool). Precedence for task queue: activity override > workflow default (`workflow.run_with_default`) > the `"default"` queue.

Confirm exact decorator signatures against `gleam/aion_flow/src/aion/activity.gleam` — it is the authoritative surface.

## 6. Build → package → run

```sh
gleam build                              # required before packaging
aion generate . --check                  # prove artifacts are fresh
aion package . --pretty                  # emits {packages:[{workflow_type, output, version, deployed_name}]}
```

`aion package` reads compiled BEAM from `build/dev/erlang`; it never compiles (except `--build`). `packages[].workflow_type` is the name for `aion start`; `packages[].output` is the archive to load via `aion server --workflow-package <file>`, a `workflow_packages` config entry, or `aion deploy <archive>` into a running server. Package version is the SHA-256 of compiled modules — manifest field edits (timeout/schemas/activities) do NOT change it; code changes do. On error `aion package` prints an `Error:` chain to stderr; CHECK `docs/packaging.md` troubleshooting table for the exact message.
