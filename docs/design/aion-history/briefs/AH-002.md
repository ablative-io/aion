# AH-002: Recorder terminal recording and list_active exclusion for ContinuedAsNew

> **Cluster:** aion-history
> **Depends on:** AH-001
> **Checklist:** C3, C6
> **Stories:** S4

## Purpose

Enable the Recorder to record WorkflowContinuedAsNew as a terminal event, and ensure that list_active (used by recovery) skips workflows whose latest run has ContinuedAsNew status. Without list_active exclusion, recovery would attempt to replay terminated runs, violating the durability contract.

## Task

Add record_workflow_continued_as_new to the Recorder, following the same pattern as record_workflow_completed and record_workflow_cancelled. Update the InMemoryStore and libSQL store list_active implementations to treat ContinuedAsNew as terminal (so workflows in this state are excluded from the active list). Update the conformance suite to cover the new terminal status in list_active tests.

## Requirements

### R1: Recorder records WorkflowContinuedAsNew

THE SYSTEM SHALL add a record_workflow_continued_as_new method to the Recorder that appends a WorkflowContinuedAsNew event with the carried input Payload, optional workflow type override, and parent_run_id. THE SYSTEM SHALL increment the sequence head and use the correct expected_seq, consistent with the single-writer discipline.

**Acceptance:**
- Recorder::record_workflow_continued_as_new exists with signature (timestamp, input, workflow_type, parent_run_id).
- After recording, the event appears in read_history with the correct sequence number.
- The Recorder's sequence head advances by one.
- The recorded event's envelope has the correct recorded_at timestamp.

**Files:**
- modify: crates/aion/src/durability/recorder.rs

**Checklist:**
- C3

**Stories:**
- S4

### R2: InMemoryStore list_active excludes ContinuedAsNew

THE SYSTEM SHALL update InMemoryStore::list_active to treat WorkflowContinuedAsNew as a terminal event. A workflow whose history ends with WorkflowContinuedAsNew SHALL NOT appear in list_active results.

**Acceptance:**
- A workflow with history [WorkflowStarted, WorkflowContinuedAsNew] does not appear in list_active.
- A workflow with history [WorkflowStarted] (no terminal event) still appears in list_active.
- The existing list_active tests continue to pass.

**Files:**
- modify: crates/aion-store/src/memory.rs

### R3: libSQL store list_active excludes ContinuedAsNew

THE SYSTEM SHALL update the libSQL EventStore implementation to treat ContinuedAsNew as a terminal status in its list_active query.

**Acceptance:**
- The libSQL list_active query excludes workflows whose projected status is ContinuedAsNew.
- The libSQL conformance suite passes with the updated query.

**Acceptance:**

- The test runs against both InMemoryStore and libSQL.
- cargo test -p aion-store passes.
