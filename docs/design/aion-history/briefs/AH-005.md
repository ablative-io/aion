# AH-005: Continue-as-new integration test with recovery

> **Cluster:** aion-history
> **Depends on:** AH-001, AH-002, AH-003, AH-004
> **Checklist:** C10
> **Stories:** S1, S3, S4

## Purpose

Prove end-to-end that a looping workflow can use continue-as-new to bound its history, and that recovery correctly replays only the current run. This is the integration-level verification that the entire AH cluster works together: the event, the status, the recorder, the engine method, list_active exclusion, and chain navigation.

## Task

Write an integration test (in crates/aion/tests/) that: (1) starts a workflow that calls continue-as-new after a few iterations; (2) verifies the old run is terminal with ContinuedAsNew status; (3) verifies the new run is active with the carried input; (4) simulates recovery by rebuilding the engine from the store and verifying only the new run is replayed; (5) verifies the continuation chain is navigable via read_run_chain. Use InMemoryStore for the test.

## Requirements

### R1: Integration test: continue-as-new lifecycle

THE SYSTEM SHALL have an integration test that starts a workflow, triggers continue-as-new via Engine::continue_as_new, and verifies: the old run's status is ContinuedAsNew, the new run's status is Running, the new run's input matches the carried payload, and the new run's WorkflowStarted has parent_run_id matching the old run.

**Acceptance:**
- Test starts a workflow and calls engine.continue_as_new with a carried payload.
- Old run's projected status is ContinuedAsNew.
- New run exists under the same WorkflowId with a different RunId.
- New run's history starts with WorkflowStarted carrying the carried payload.
- New run's WorkflowStarted has parent_run_id = old run's RunId.

**Files:**
- create: crates/aion/tests/continue_as_new.rs

**Checklist:**
- C10

**Stories:**
- S1

### R2: Integration test: recovery skips old runs

THE SYSTEM SHALL have an integration test that verifies list_active returns only the current (non-ContinuedAsNew) run after a continue-as-new. This proves recovery would replay only the current run.

**Acceptance:**
- After continue-as-new, list_active returns only the WorkflowId once (not twice).
- The active run is the new run, not the old one.
- A workflow that has not used continue-as-new still appears correctly in list_active.

**Files:**
- modify: crates/aion/tests/continue_as_new.rs

**Checklist:**
- C10

**Stories:**
- S4

### R3: Integration test: continuation chain navigation

THE SYSTEM SHALL have an integration test that calls continue-as-new twice and verifies read_run_chain returns three RunSummary entries in order with correct parent_run_id links.

**Acceptance:**
- After two continue-as-new calls, read_run_chain returns 3 entries.
- First entry has parent_run_id = None.
- Second entry has parent_run_id = first RunId.
- Third entry has parent_run_id = second RunId.
- Entries are in chronological order.
