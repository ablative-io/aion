# AH-001: WorkflowContinuedAsNew event and ContinuedAsNew terminal status

> **Cluster:** aion-history
> **Checklist:** C1, C2
> **Stories:** S1, S4

## Purpose

Add the core domain types for continue-as-new: the WorkflowContinuedAsNew event variant and the ContinuedAsNew terminal workflow status. These are the foundation every other AH brief builds on. The event carries the input payload for the new run, an optional workflow type override for migration, and the parent_run_id linking to the previous run. The status is derived by projection, consistent with the status-is-a-projection invariant.

## Task

Add a WorkflowContinuedAsNew variant to the Event enum in aion-core carrying an EventEnvelope, the carried Payload (new run input), an optional workflow_type override (String), and parent_run_id (RunId). Add ContinuedAsNew to the WorkflowStatus enum as a terminal status. Update the status projection (status_of) to derive ContinuedAsNew from WorkflowContinuedAsNew events. Update all exhaustive matches on Event and WorkflowStatus across aion-core (filter, is_terminal, display, serde) to handle the new variants.

## Requirements

### R1: WorkflowContinuedAsNew event variant

THE SYSTEM SHALL add a WorkflowContinuedAsNew variant to the Event enum carrying: an EventEnvelope, an input Payload (the carried state for the new run), an optional workflow_type (Option<String>) for migration, and a parent_run_id (RunId) linking to the current run that is being continued.

**Acceptance:**
- Event::WorkflowContinuedAsNew exists with fields envelope: EventEnvelope, input: Payload, workflow_type: Option<String>, parent_run_id: RunId.
- The variant serializes and deserializes correctly via serde (round-trip test).
- The variant is handled in every exhaustive match on Event in aion-core (filter.rs, status.rs, and any other match sites).

**Files:**
- modify: crates/aion-core/src/event.rs
- modify: crates/aion-core/src/filter.rs

**Checklist:**
- C1

**Stories:**
- S1

### R2: ContinuedAsNew terminal status

THE SYSTEM SHALL add ContinuedAsNew to the WorkflowStatus enum. WorkflowStatus::ContinuedAsNew SHALL be terminal (is_terminal returns true). The status_of projection SHALL derive ContinuedAsNew from WorkflowContinuedAsNew events.

**Acceptance:**
- WorkflowStatus::ContinuedAsNew exists.
- WorkflowStatus::ContinuedAsNew.is_terminal() returns true.
- status_of applied to a history ending with WorkflowContinuedAsNew returns Some(WorkflowStatus::ContinuedAsNew).
- A history with WorkflowStarted then WorkflowContinuedAsNew projects to ContinuedAsNew status.
- All exhaustive matches on WorkflowStatus in aion-core handle the new variant.

**Files:**
- modify: crates/aion-core/src/status.rs

**Checklist:**
- C2

**Stories:**
- S4

### R3: Filter and projection integration

THE SYSTEM SHALL treat WorkflowContinuedAsNew as a terminal lifecycle event in all filter and projection paths. THE SYSTEM SHALL include it in terminal-event filters and closed_at extraction alongside WorkflowCompleted, WorkflowFailed, WorkflowCancelled, and WorkflowTimedOut.

**Acceptance:**
- EventFilter::terminal() matches WorkflowContinuedAsNew.
- closed_at extraction returns the envelope's recorded_at for WorkflowContinuedAsNew.
- cargo clippy -p aion-core --all-targets -- -D warnings passes with no unhandled-variant warnings.
- cargo test -p aion-core passes.

**Files:**
- modify: crates/aion-core/src/filter.rs

**Checklist:**

- C2

**Stories:**
- S4
