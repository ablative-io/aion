# AS-008: Embedded-replica sync against a remote primary

> **Cluster:** aion-store-libsql
> **Depends on:** AS-007
> **Checklist:**
> - C26 — Embedded-replica sync is exercised by a test runtime-gated on AION_LIBSQL_TEST_URL: when the env var is unset the test emits a tracing skip line and returns Ok; it is never marked #[ignore].
> **Stories:**
> - S2 (Engine Operator, Deploying Aion with Durable Storage) — As an engine operator, I want the same store crate to drive a distributed deployment via an embedded replica syncing to a remote primary, so that I do not have to learn or wire a second backend to scale out.

## Purpose

Prove the distributed deployment path: a local embedded replica that syncs to a remote libSQL primary. This is the one part of the crate that needs an external service, so its test is runtime env-gated rather than ignored, per CLAUDE.md.

## Task

Add an embedded-replica sync integration test that, when a remote primary is configured via AION_LIBSQL_TEST_URL (and token), opens a LibSqlStore in EmbeddedReplica mode, appends and syncs, and asserts the data reaches the primary / is readable through a second replica after sync. When the env var is unset, the test emits a tracing skip line and returns Ok trivially. It is never #[ignore]. Confirm the same EventStore behaviour holds in replica mode as in embedded mode.

## Requirements

### R1: Env-gated embedded-replica sync test

WHILE AION_LIBSQL_TEST_URL is set, THE SYSTEM SHALL run the embedded-replica sync test against that primary; WHILE it is unset, THE SYSTEM SHALL emit a tracing skip line and return Ok without failing. THE SYSTEM SHALL NOT mark the test #[ignore].

**Acceptance:**
- tests/replica_sync.rs reads AION_LIBSQL_TEST_URL (and the accompanying auth token env var, e.g. AION_LIBSQL_TEST_TOKEN) at runtime.
- When unset, the test emits a tracing::info! skip line naming the missing env var and returns Ok(()) — it does not fail and is not #[ignore]d.
- When set, the test constructs a LibSqlConfig::EmbeddedReplica pointing at the primary with the token and opens a LibSqlStore.
- The gate is a runtime read of the env var, not a cfg!/feature/#[ignore] compile-time gate.

### R2: Sync round-trip assertion

WHEN the env-gated test runs against a real primary, THE SYSTEM SHALL append history through the embedded replica, sync to the primary, and observe the synced data so that the distributed path is proven end-to-end.

**Acceptance:**
- The test appends a known event batch through the replica store and triggers/awaits a sync to the primary.
- The test asserts the data is durably present on the primary — either by reading it back through a second freshly-opened replica that syncs from the primary, or by querying the primary directly — and asserts read_history matches what was appended.
- The same EventStore contract behaviour (append ordering, read_history) holds in replica mode as in embedded mode; the test reuses the contract assertions rather than asserting libSQL internals.
- Sync or connection failures surface as StoreError::Backend, not panics.

## Boundaries

- SHALL NOT require a live primary for the crate to build or for the rest of the test suite to pass — only this test consults the primary, and only when its env var is set.
- SHALL NOT use #[ignore] — the gate is a runtime env-var read per CLAUDE.md.
- SHALL NOT re-implement the EventStore methods — replica mode reuses the same impl; only the connection-open path differs (AS-002 R2).
- SHALL NOT add distributed shard-assignment or coordination logic — that is engine/distributed scope, not this store.
