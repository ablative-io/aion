# AS-002: libSQL connection opening and idempotent schema

> **Cluster:** aion-store-libsql
> **Depends on:** AS-001
> **Checklist:**
> - C6 — Opening in Embedded mode produces a working connection to a local file database.
> - C7 — Opening in EmbeddedReplica mode produces a local replica connection configured to sync with the remote primary URL using the auth token.
> - C8 — Opening a store runs idempotent DDL (CREATE TABLE IF NOT EXISTS) so a fresh file and an existing file both arrive at the same schema with no external migration tool.
> - C9 — The events table has columns workflow_id, seq, event (serialised blob), recorded_at, with primary key (workflow_id, seq) and a secondary index supporting query/list_active projections.
> - C10 — The timers table has columns workflow_id, timer_id, fire_at, with primary key (workflow_id, timer_id) and an index on fire_at.
> **Stories:**
> - S1 (Engine Operator, Deploying Aion with Durable Storage) — As an engine operator, I want a zero-infrastructure durable store I get by pointing at a file path, so that embedding Aion needs no database server to survive restarts.
> - S2 (Engine Operator, Deploying Aion with Durable Storage) — As an engine operator, I want the same store crate to drive a distributed deployment via an embedded replica syncing to a remote primary, so that I do not have to learn or wire a second backend to scale out.

## Purpose

Open a libSQL connection in either embedded or embedded-replica mode from a LibSqlConfig, and create the events and timers schema idempotently on open. This is the substrate the EventStore impl reads and writes through.

## Task

Build the connection module that turns a LibSqlConfig into a working libSQL connection — a local file in Embedded mode, a local replica syncing to a remote primary in EmbeddedReplica mode. Build the schema module with the events and timers table DDL and a migration function that creates them idempotently (CREATE TABLE IF NOT EXISTS). Do NOT implement EventStore methods here — only connection and schema.

## Requirements

### R1: Open an embedded (local file) connection

WHEN constructed from a LibSqlConfig in Embedded mode, THE SYSTEM SHALL open a libSQL connection to the local file at the configured path, applying the configured WAL/journal settings where present.

**Acceptance:**
- connection.rs provides an async function that, given an Embedded config, returns a libSQL connection (or database handle) to the local file.
- Configured WAL/journal settings present on the config are applied to the connection; absent settings are not overridden by crate-chosen defaults (per AS-001 R3).
- A test opens an embedded connection at a tempfile path and executes a trivial SELECT 1 successfully.
- Any libsql error during open is mapped to StoreError via the AS-001 error boundary.

**Files:**
- create: crates/aion-store-libsql/src/connection.rs
- modify: crates/aion-store-libsql/src/lib.rs

### R2: Open an embedded-replica connection

WHEN constructed from a LibSqlConfig in EmbeddedReplica mode, THE SYSTEM SHALL open a local replica database configured to sync with the remote primary_url using the auth_token, honouring the configured sync interval where present.

**Acceptance:**
- connection.rs provides the replica open path using libSQL's embedded-replica builder with the local path, primary_url, and auth_token from the config.
- The configured sync interval, when present, is applied; when absent, no crate-chosen interval is assumed.
- The open path returns a connection usable identically to the embedded path (same downstream type), so the EventStore impl is mode-agnostic.
- Connection-open failures (bad URL, auth) are mapped to StoreError::Backend via the error boundary; this requirement does NOT require a live primary to test the embedded-replica open call path's error mapping.

### R3: events and timers table DDL

THE SYSTEM SHALL define the DDL for an append-only events table and a durable timers table with the keys and indexes the store requires.

**Acceptance:**
- schema.rs defines the events table DDL: workflow_id TEXT, seq INTEGER, event BLOB, recorded_at TEXT, PRIMARY KEY (workflow_id, seq).
- schema.rs defines a secondary index on events supporting query/list_active projection (e.g. on the lifecycle-event discriminant and recorded_at, or on workflow_id with the columns query needs). COORDINATION NOTE: the exact index columns depend on the query shapes finalised in AS-005 (query/list_active). Whoever implements AS-002 must coordinate with AS-005 — if the query plan needs different columns, AS-005 amends schema.rs (schema.rs is in AS-005's modify set for exactly this) rather than leaving a wrong index. Prefer settling the AS-005 query shapes before fixing this index.
- schema.rs defines the timers table DDL: workflow_id TEXT, timer_id TEXT, fire_at TEXT, PRIMARY KEY (workflow_id, timer_id), plus an index on fire_at.
- DDL statements use IF NOT EXISTS so they are idempotent.

### R4: Idempotent migration on open

WHEN a store is opened against a fresh or an existing database file, THE SYSTEM SHALL ensure the events and timers schema exists by running the idempotent DDL, arriving at the same schema in both cases with no external migration tool.

**Acceptance:**
- schema.rs provides an async ensure_schema(conn) that executes the events and timers DDL.
- A test runs ensure_schema twice against the same connection and asserts the second run succeeds (idempotent, no error).
- A test opens a fresh tempfile, runs ensure_schema, and asserts both tables and their indexes exist (e.g. via a query against sqlite_master / pragma).
- Migration failures are mapped to StoreError::Backend via the error boundary.

## Boundaries

- SHALL NOT implement append, read_history, list_active, query, or timer methods — those are AS-004, AS-005, AS-006.
- SHALL NOT construct LibSqlStore or implement EventStore — that is AS-003.
- SHALL NOT require a live remote primary to compile or to run its tests; replica open is built but live sync is exercised only in AS-008.
- SHALL NOT introduce a mutable workflow/status table — summaries are derived from event rows (AS-005).
