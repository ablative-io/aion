# AS-007: Conformance suite run and persistence-across-reopen tests

> **Cluster:** aion-store-libsql
> **Depends on:** AS-004, AS-005, AS-006
> **Checklist:**
> - C24 — The shared AC-007 conformance suite (run_event_store_suite) runs against LibSqlStore over a temporary local file as a real, non-gated runtime test and passes.
> - C25 — Persistence-across-reopen tests append history, drop the store, reopen the same file, and assert read_history, list_active, and timers are intact.
> **Stories:**
> - S5 (Engine, Persisting and Replaying Workflow State) — As the engine, I want a workflow's complete history to survive process restarts so that I can replay it from durable storage after a crash or reboot.
> - S7 (Store Author, Maintaining the libSQL Backend) — As a store author, I want this backend to pass the shared behavioural conformance suite unmodified so that I have objective proof it matches the in-memory oracle exactly.

## Purpose

Prove LibSqlStore is correct by running the shared AC-007 behavioural conformance suite against it, and prove it is durable — the one thing the in-memory oracle cannot demonstrate — by reopening a closed database and asserting state survived. These tests are the cluster's correctness gate.

## Task

Add a conformance test that invokes the AC-007 run_event_store_suite with a factory yielding a fresh LibSqlStore over a temporary local file, run as a real (non-gated) test. Add persistence tests that append history and schedule timers, drop the store, reopen the same file, and assert read_history, list_active, and expired_timers all reflect the persisted state. These exercise the embedded path and need no external service.

## Requirements

### R1: Run the AC-007 conformance suite against LibSqlStore

THE SYSTEM SHALL run the shared aion-store conformance suite (run_event_store_suite) against LibSqlStore over a temporary local file, as a real runtime test that is not gated or ignored, and the suite SHALL pass.

**Acceptance:**
- tests/conformance.rs invokes run_event_store_suite (imported from aion-store's test-support surface) with an async factory closure that creates a fresh LibSqlStore over a unique tempfile path per invocation and returns it as Arc<dyn EventStore>.
- The test runs under cargo test with no env gating (embedded mode needs no external service) and passes.
- Each suite invocation gets a fresh database (a new tempfile), so scenarios do not contaminate each other.
- The test is NOT marked #[ignore] and uses no #[allow]/#[expect].

### R2: Persistence across close-and-reopen

WHEN a store is closed (dropped) and reopened against the same file, THE SYSTEM SHALL return the previously appended history, the previously active workflows, and the previously scheduled timers unchanged.

**Acceptance:**
- tests/persistence.rs appends a multi-event history for one or more workflows and schedules one or more timers, then drops the LibSqlStore.
- A new LibSqlStore is opened against the same file path; read_history returns the same events in the same order, list_active returns the same active workflows, and expired_timers returns the previously scheduled timers due as of an appropriate instant.
- A workflow that was terminal before reopen is still absent from list_active after reopen (status projection survives because the events do).
- The test asserts byte-for-byte event equality across the reopen, proving the blobs round-tripped through durable storage.

**Files:**
- create: crates/aion-store-libsql/tests/persistence.rs

**Checklist:**

### R3: Sequence guard survives reopen

WHEN a store is reopened against a file with existing history, THE SYSTEM SHALL compute the head sequence from the persisted events so that the expected-sequence guard continues correctly after a restart.

**Acceptance:**
- tests/persistence.rs appends a batch, drops the store, reopens, and then appends a further batch with the correct expected_seq (derived from the persisted head) and asserts it succeeds.
- The same reopened store rejects an append with a stale expected_seq with SequenceConflict, proving the head was reconstructed from durable rows rather than reset to zero.
- read_history after the second append shows the full combined history in contiguous seq order across the reopen boundary.
