# AS-006: Durable timers — schedule_timer and expired_timers

> **Cluster:** aion-store-libsql
> **Depends on:** AS-003
> **Checklist:**
> - C22 — schedule_timer persists a TimerEntry into the timers table keyed by (workflow_id, timer_id); re-scheduling the same timer id replaces the prior row (idempotent under replay).
> - C23 — expired_timers(as_of) returns all timers with fire_at <= as_of and excludes timers not yet due.
> **Stories:**
> - S6 (Engine, Persisting and Replaying Workflow State) — As the engine, I want to list active workflows and durable timers from storage on startup so that I know what to replay and which timers are due after downtime.

## Purpose

Implement the durable timer substrate over the timers table: persist scheduled timers idempotently and retrieve those due to fire as of a given instant. This is what lets a workflow sleep for months and have the engine, on restart, discover which timers are due.

## Task

Implement schedule_timer and expired_timers in timer.rs against the timers table from AS-002. schedule_timer upserts a TimerEntry keyed by (workflow_id, timer_id) so re-scheduling replaces (idempotent under replay). expired_timers(as_of) range-scans fire_at <= as_of. The store returns due timers; it does not fire them.

## Requirements

### R1: schedule_timer (idempotent upsert)

THE SYSTEM SHALL persist a durable timer keyed by (workflow_id, timer_id). WHEN the same timer id is scheduled again, THE SYSTEM SHALL replace the prior row rather than insert a duplicate, so scheduling is idempotent under replay.

**Acceptance:**
- timer.rs implements schedule_timer(&self, &WorkflowId, &TimerId, fire_at: DateTime<Utc>) -> Result<(), StoreError> as an INSERT ... ON CONFLICT(workflow_id, timer_id) DO UPDATE (upsert), writing fire_at as RFC3339.
- Re-scheduling the same (workflow_id, timer_id) with a new fire_at updates the row to the new fire_at and does not create a second row.
- A test schedules a timer, re-schedules it with a different fire_at, and asserts exactly one row exists with the latest fire_at.
- libsql errors map to StoreError::Backend; serialisation of the timestamp is lossless and round-trips.

### R2: expired_timers (due-as-of range scan)

THE SYSTEM SHALL return all timers whose fire_at is less than or equal to a given instant, and SHALL NOT return timers whose fire_at is after that instant.

**Acceptance:**
- timer.rs implements expired_timers(&self, as_of: DateTime<Utc>) -> Result<Vec<TimerEntry>, StoreError> as an indexed range scan on fire_at <= as_of.
- Each returned TimerEntry carries workflow_id, timer_id, and fire_at reconstructed from the row.
- A test schedules timers at distinct instants (past, exactly-as_of, future) and asserts expired_timers returns exactly the due ones (<= as_of), excluding the future one.
- The boundary is inclusive at as_of (a timer with fire_at == as_of is returned), matching the AC-005/AC-006 contract; the test asserts the boundary case.

## Boundaries

- SHALL NOT fire timers or schedule wall-clock callbacks — the store returns due timers; firing is the engine (AE) and durability/replay (AD).
- SHALL NOT implement append/read/query — AS-004, AS-005.
- SHALL NOT store timers in the events table — they live in the dedicated timers table from AS-002.
- SHALL NOT create duplicate rows for a re-scheduled timer id.
