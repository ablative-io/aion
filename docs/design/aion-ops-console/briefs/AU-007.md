# AU-007: App shell — routing, providers, layout

> **Cluster:** aion-dashboard
> **Depends on:** AU-004, AU-005, AU-006, AU-003
> **Checklist:**
> - C36 — The app shell wires routing (/ list, /workflows/:id detail) and the QueryClient, namespace, and websocket providers.

## Purpose

Assemble the features into a coherent application: routes for the list and detail views and the provider stack (query client, namespace, websocket) they all depend on, with the whole bundle passing the house quality gates.

## Task

Build the app shell: React Router routes for / (list) and /workflows/:id (detail), the provider stack (QueryClient, namespace, websocket), and the layout chrome (header with namespace selector and connection indicator). Bring the whole bundle to a clean Biome and tsc pass.

## Requirements

### R1: Routing and layout chrome

THE SYSTEM SHALL define routes for / (workflow list) and /workflows/:id (workflow detail) and a layout shell with a header carrying the namespace selector and the connection indicator.

**Acceptance:**
- src/app/routes.tsx defines / → WorkflowList and /workflows/:id → WorkflowDetail using React Router v7.
- src/app/App.tsx renders the layout shell (header + routed content); the header includes NamespaceSelector and ConnectionIndicator.
- Navigating from a list row to a detail route and back works in a render test.
- Configurable base path from AU-001 is respected by the router.

**Files:**
- create: apps/aion-dashboard/src/app/App.tsx
- create: apps/aion-dashboard/src/app/routes.tsx
- modify: apps/aion-dashboard/src/main.tsx

### R2: Provider stack

THE SYSTEM SHALL wire the provider stack — TanStack QueryClient, the namespace context, and the websocket connection — around the routed application.

**Acceptance:**
- src/app/providers.tsx nests QueryClientProvider → NamespaceProvider → (websocket connection lifecycle) around the router.
- The websocket connects on mount and closes on unmount, and the QueryClient is configured consistently with apps/web's defaults.
- A render test asserts the app mounts with all providers present and the list route renders.
