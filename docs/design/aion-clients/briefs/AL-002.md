## Purpose

Build the foundation of the Rust caller SDK: a crate that connects to an aion-server, exposes the unary operations (start, signal, query, cancel, list, describe) over the transport, and maps every server failure to a branchable ClientError. This is the network client every Rust application uses to drive workflows, and the reference against which the other-language SDKs are compared.

## Task

Create the aion-client crate as a workspace member depending on aion-proto and aion-core (and external crates), never on aion/aion-store/beamr by default. Implement ClientBuilder/Client for connection with auth + TLS, a transport module mapping the six unary operations onto the aion-proto request/response types, and the ClientError taxonomy from the AL-001 contract. The WorkflowHandle, typed-payload helpers, event Stream, and embedded binding land in AL-003 — this brief returns bare WorkflowId/RunId from start and uses raw Payload arguments.

### R2: ClientError taxonomy

THE SYSTEM SHALL define ClientError as a closed enum covering the nine variants from the AL-001 contract, implemented with thiserror. IF a server condition matches a known variant, THEN THE SYSTEM SHALL map it to that variant; IF it is an unexpected server error, THEN THE SYSTEM SHALL map it to Server carrying the server's detail. THE SYSTEM SHALL NOT collapse distinct conditions into one variant.

**Acceptance:**
- error.rs defines ClientError with variants NotFound, AlreadyExists, QueryFailed, QueryTimeout, Cancelled, Unavailable, Unauthenticated, InvalidArgument, and Server { detail: String }.
- ClientError derives Debug, implements Display and std::error::Error via thiserror, and is Send + Sync + 'static.
- A mapping function converts an aion-proto / transport error into the correct ClientError variant, with a unit test asserting at least the NotFound, AlreadyExists, QueryTimeout, and Unavailable mappings.
- No mapping path returns a generic Server for a condition that has a specific variant.

### R3: ClientBuilder, Client, and transport connection

THE SYSTEM SHALL provide a ClientBuilder that connects to an aion-server endpoint with an auth credential and TLS configuration and produces a reusable Client. THE SYSTEM SHALL NOT require re-connection per operation.

**Acceptance:**
- client.rs defines Client and ClientBuilder; ClientBuilder::new(endpoint) plus with_auth(...) and with_tls(...) configure the connection and build() yields a Client (async).
- transport.rs holds the network transport over aion-proto; Client holds a cheaply-cloneable transport handle reused across operations.
- Connecting with a bad endpoint or failed handshake yields ClientError::Unavailable; a rejected credential yields ClientError::Unauthenticated.
- Client is Clone and Send + Sync so it can be shared across tasks.
- A doc example shows constructing a Client from an endpoint and credential.

### R4: Unary operations: start, signal, query, cancel, list, describe

THE SYSTEM SHALL implement start, signal, query, cancel, list, and describe on Client over the transport, each returning Result<_, ClientError>, mapping inputs and outputs to the aion-proto types. THE SYSTEM SHALL surface a query deadline as QueryTimeout and SHALL NOT treat cancel as a confirmation that the workflow has stopped.

**Acceptance:**
- ops.rs implements start(workflow_type, input: Payload, opts) -> Result<(WorkflowId, RunId), ClientError>, signal(&WorkflowId, run, name, payload: Payload), query(&WorkflowId, run, name, args: Payload, deadline) -> Result<Payload, ClientError>, cancel(&WorkflowId, run, reason), list(&WorkflowFilter, page) -> Result<Vec<WorkflowSummary>, ClientError>, and describe(&WorkflowId, run) -> Result<WorkflowDescription, ClientError>.
- Operations target the latest run by default and a specific RunId when the run argument is Some, per the AL-001 contract.
- query enforces the supplied deadline and returns ClientError::QueryTimeout when it elapses; a handler error returns QueryFailed.
- Operating on an unknown workflow returns ClientError::NotFound.
- WorkflowSummary and WorkflowDescription reuse aion-core types where they exist (status from aion-core::WorkflowStatus); list pagination matches the contract.
- Each operation has a unit test against a stub transport asserting the request mapping and the error mapping for at least one failure case.
