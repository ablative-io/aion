## Purpose

Build the Python caller SDK so Python services (the home of ML and data-science work) can drive Aion workflows idiomatically. It implements the AL-001 contract in Python idioms: a connect-and-operate Client, a WorkflowHandle, JSON/model-typed payloads with a bytes escape hatch, an async event iterator that resumes across disconnects, and an exception hierarchy that maps the shared error taxonomy. mypy --strict clean.

## Task

Create the aion-client-python package (PyPI distribution name aion-client-python, import name aion_client). Implement an async Client that connects to an aion-server (auth + TLS), exposes start/signal/query/cancel/list/describe with start returning a WorkflowHandle, JSON-and-model typed payload helpers with a raw bytes escape hatch, a subscribe async iterator with sequence-based resumption, and an exception hierarchy over the shared taxonomy. Behaviour must match the AL-001 contract; this brief does not run the conformance suite (that is AL-007) but must be structured so the AL-007 harness can drive it.

## Requirements

### R1: Package scaffold and Client connection

THE SYSTEM SHALL create the aion_client package with an async Client that connects to an aion-server with an auth credential and TLS configuration. THE SYSTEM SHALL expose start, signal, query, cancel, list, and describe, with start returning a WorkflowHandle. THE SYSTEM SHALL NOT require reconnection per operation.

**Acceptance:**
- sdks/python/aion-client/pyproject.toml declares the distribution name aion-client-python, the import package aion_client, runtime dependencies, and the build backend.
- Client(endpoint, *, auth=..., tls=...) (or a connect() classmethod/async factory) establishes a reusable connection.
- Client exposes async start, signal, query, cancel, list, describe matching the contract's inputs and outputs; start returns a WorkflowHandle.
- WorkflowHandle exposes signal/query/cancel/describe/subscribe bound to the workflow_id and run_id, defaulting to the latest run and targeting a specific run when given one.
- A connection failure raises Unavailable; a rejected credential raises Unauthenticated.

### R2: Error exception hierarchy

THE SYSTEM SHALL define an exception hierarchy mapping the shared taxonomy with a common base. THE SYSTEM SHALL NOT raise a bare Exception for a condition that has a specific type and SHALL NOT swallow server errors.

**Acceptance:**
- errors.py defines AionClientError (base) and subclasses NotFound, AlreadyExists, QueryFailed, QueryTimeout, Cancelled, Unavailable, Unauthenticated, InvalidArgument, and ServerError (carrying server detail).
- A mapping function turns a server/transport failure into the correct subclass; a unit test asserts the NotFound, AlreadyExists, QueryTimeout, and Unavailable mappings.
- Every public method's docstring lists the exceptions it can raise.
- No code path catches and discards an exception without re-raising or converting it.

### R3: Typed payload helpers with bytes escape hatch

THE SYSTEM SHALL accept any JSON-serialisable value as input and optionally decode results into a target type or model, with a raw bytes escape hatch on every payload-bearing operation. THE SYSTEM SHALL NOT remove the raw path.

**Acceptance:**
- payload.py serialises a JSON-serialisable value to a JSON-content-type Payload and decodes a result Payload into a dict, a provided target type, or a Pydantic model when one is supplied.
- start/signal/query accept a value (JSON-encoded) and also accept raw bytes plus a content-type for the escape hatch.
- A non-serialisable input raises InvalidArgument; a decode failure raises a non-swallowed error.
- A unit test round-trips a value and (separately) a model through encode/decode and asserts equality.
- Type hints are complete enough that mypy --strict passes for the payload surface.

### R4: subscribe async iterator with resumption

THE SYSTEM SHALL implement subscribe as an async iterator of decoded events. WHILE open, IF the connection drops transiently, THEN THE SYSTEM SHALL reconnect and resume from the last delivered sequence number with no gaps or duplicates. IF the subscription fails terminally, THEN THE SYSTEM SHALL raise rather than ending silently.

**Acceptance:**
- stream.py exposes an async iterator (async for event in handle.subscribe(): ...) yielding decoded events over the WebSocket transport.
- The iterator tracks the last delivered sequence number and resumes from sequence+1 after a transient disconnect.
- A test with a stub transport that drops mid-stream asserts each event is delivered exactly once across the reconnect.
- A terminal failure raises the mapped exception out of the iterator; the iterator does not silently StopAsyncIteration on failure.
- mypy --strict passes for the stream surface.
