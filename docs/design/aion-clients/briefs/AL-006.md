## Purpose

Build the Gleam caller SDK so Gleam services — including those authoring workflows with aion_flow — can drive workflows from outside with full static typing. It implements the AL-001 contract in Gleam idioms: connect plus the operations, encoder/decoder-based payloads with a raw escape hatch, an event stream that resumes across disconnects, and an error custom type. gleam check, format, and test clean.

## Task

Create the aion_client Gleam package (Hex name aion_client). Implement connect plus start/signal/query/cancel/list/describe with start returning a workflow handle value, gleam/json encoder + gleam/dynamic decoder typed payloads with a raw-Payload escape hatch, a subscribe event stream with sequence-based resumption, and an error custom type over the shared taxonomy. Behaviour must match the AL-001 contract; the conformance harness is AL-007, but the package must be drivable by it. Use the established beamr/Gleam HTTP + WebSocket capabilities for transport.

## Requirements

### R1: Package scaffold, connect, and operations

THE SYSTEM SHALL create the aion_client package exposing connect plus start, signal, query, cancel, list, and describe, with start returning a handle value usable in per-workflow operations. THE SYSTEM SHALL NOT require reconnecting per operation.

**Acceptance:**
- gleam/aion_client/gleam.toml declares name aion_client, the Hex metadata, and the gleam_stdlib / gleam_json / HTTP-client dependencies.
- aion_client.gleam exposes connect(config) returning a Client value, and start/signal/query/cancel/list/describe functions taking the Client.
- start returns a WorkflowHandle value carrying workflow_id + run_id; handle.gleam offers signal/query/cancel/describe/subscribe taking that handle.
- Operations target the latest run by default and a specific run_id when a Some(run_id) is supplied, per the contract.
- Connection and credential failures map to the Unavailable and Unauthenticated error variants (R2).

### R2: Error custom type

THE SYSTEM SHALL define an error custom type mapping the shared taxonomy and return Result(_, Error) from every operation. THE SYSTEM SHALL NOT collapse distinct conditions into one variant.

**Acceptance:**
- error.gleam defines a public Error type with variants NotFound, AlreadyExists, QueryFailed, QueryTimeout, Cancelled, Unavailable, Unauthenticated, NamespaceDenied(detail: String), InvalidArgument, and Server(detail: String).
- A mapping function converts a server/transport failure into the correct variant; a test asserts the NotFound, AlreadyExists, QueryTimeout, and Unavailable mappings.
- Every operation returns Result(value, Error); none returns a bare value that hides a failure.
- The variant set matches the AL-001 taxonomy exactly (no extra, no missing).

### R3: Encoder/decoder typed payloads with raw escape hatch

THE SYSTEM SHALL take a gleam/json encoder for input and a gleam/dynamic decoder for the result, staying statically typed, and SHALL keep a raw-Payload escape hatch on every payload-bearing operation. THE SYSTEM SHALL NOT remove the raw path.

**Acceptance:**
- payload.gleam provides encode(value, encoder) -> Payload (JSON content-type) and decode(payload, decoder) -> Result(value, Error).
- start/signal accept an input value plus its encoder; query accepts an args encoder and a result decoder and returns the decoded reply.
- A raw variant of each payload-bearing operation accepts a Payload (bytes + content-type) directly.
- A decode failure returns Error InvalidArgument (or a documented variant), not a panic or a swallowed result.
- A test round-trips a value through encode then decode and asserts equality.

### R4: subscribe event stream with resumption

THE SYSTEM SHALL implement subscribe returning an event stream. WHILE open, IF the connection drops transiently, THEN THE SYSTEM SHALL reconnect and resume from the last delivered sequence number with no gaps or duplicates. IF the subscription fails terminally, THEN THE SYSTEM SHALL surface the error through the stream rather than ending silently.

**Acceptance:**
- stream.gleam exposes subscribe returning a stream/subject of decoded events over the WebSocket transport.
- The stream tracks the last delivered sequence number and resumes from sequence+1 after a transient disconnect.
- A test with a stub transport that drops mid-stream asserts each event is delivered exactly once across the reconnect.
- A terminal failure is delivered as an error value through the stream channel, distinguishable from a normal end-of-stream.
- gleam check passes for the stream module.
