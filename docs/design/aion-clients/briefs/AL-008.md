# AL-008: Packaging, publishing metadata, READMEs, and seven-operation examples for all four SDKs

> **Cluster:** aion-clients
> **Depends on:** AL-003, AL-004, AL-005, AL-006
> **Checklist:**
> - C29 — aion-client (crates.io), aion-client-python (PyPI), aion-client-typescript (npm), and aion_client (Hex) each carry complete package metadata, a README, and a type-checked public surface.
> - C30 — Each package ships a runnable example exercising all seven operations against a server, referenced from its README.
> **Stories:**
> - S12 (Polyglot Platform Team, Consuming Aion from Multiple Languages) — As a platform team, I want each client published to its native registry with a README and a runnable example so that adopting it is a one-line dependency add.

## Purpose

Make all four SDKs adoptable with a one-line dependency add. Each gets complete registry metadata, a README that teaches the seven operations, and a runnable example that exercises every operation against a server. Without this, the SDKs exist but no one can find, install, or learn them.

## Task

Finalise packaging and documentation for aion-client (crates.io), aion-client-python (PyPI), aion-client-typescript (npm), and aion_client (Hex). Each package: complete metadata (name, version, description, license, repository, keywords/categories, authors), a README walking through connect + the seven operations, and a runnable example program covering start/signal/query/cancel/list/describe/subscribe against a server. This brief does not change SDK behaviour; it packages and documents what AL-002..AL-006 built.

## Requirements

### R1: Registry metadata for all four packages

THE SYSTEM SHALL complete the publish metadata for each package so it is uploadable to its registry. THE SYSTEM SHALL NOT leave a placeholder version, description, license, or repository field.

**Acceptance:**
- crates/aion-client/Cargo.toml has name aion-client, a version, description, license, repository, keywords, and categories.
- sdks/python/aion-client/pyproject.toml has distribution name aion-client-python, version, description, license, project URLs, and classifiers; the import package is aion_client.
- sdks/typescript/aion-client/package.json has name aion-client-typescript, version, description, license, repository, main/module/types entry points, and a files allowlist.
- gleam/aion_client/gleam.toml has name aion_client, version, description, licences, and repository links.
- No metadata field is a TODO/placeholder; a dry-run publish/pack for each (cargo package, python -m build, npm pack, gleam build) succeeds.

**Files:**

- modify: sdks/python/aion-client/pyproject.toml

### R2: READMEs teaching the seven operations

THE SYSTEM SHALL provide a README per package that shows connect plus each of the seven operations with a typed payload. THE SYSTEM SHALL NOT ship a README that omits any of the seven operations or the error-handling pattern.

**Acceptance:**
- Each README has an install line, a connect snippet, and a snippet per operation (start, signal, query, cancel, list, describe, subscribe) in that language.
- Each README shows the typed payload path and mentions the raw escape hatch.
- Each README shows catching/branching on at least two error-taxonomy variants idiomatically.
- Each README links to the runnable example from R3 and states the server-URL prerequisite.
- The Rust README is the crate-level doc (//! or README referenced from Cargo.toml) so it renders on docs.rs and crates.io.

- create: sdks/python/aion-client/README.md

- create: gleam/aion_client/README.md

### R3: Runnable seven-operation examples

THE SYSTEM SHALL ship a runnable example per package that connects to a server and exercises all seven operations end to end. THE SYSTEM SHALL surface errors from the taxonomy rather than crashing opaquely.

**Acceptance:**
- crates/aion-client/examples/ has an example binary that connects, starts a workflow, signals it, queries it, lists, describes, cancels, and subscribes to its events, reading the server URL from an env var / arg.
- sdks/python/aion-client/examples/ and sdks/typescript/aion-client/examples/ have the equivalent runnable scripts.
- gleam/aion_client/test/example_test.gleam (or an examples module) provides the equivalent runnable Gleam flow.
- Each example handles at least one error-taxonomy variant explicitly (e.g. prints a clear message on Unavailable) rather than panicking/throwing uncaught.
- Each example is referenced from its README and runs against the AL-007 server fixture.

- create: crates/aion-client/examples/seven_operations.rs
- create: sdks/python/aion-client/examples/seven_operations.py
- create: sdks/typescript/aion-client/examples/seven-operations.ts
