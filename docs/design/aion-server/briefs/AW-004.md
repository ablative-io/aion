# AW-004: Event-streaming wire shape (SubscriptionRequest + streamed Event envelope)

> **Cluster:** aion-server
> **Depends on:** AW-002
> **Checklist:**
> - C11 — events.proto defines a SubscriptionRequest with per-workflow, filtered (by type/status/namespace), and firehose variants, plus the streamed Event envelope.
> - C12 — The streamed Event envelope carries the aion-core Event unmodified plus the owning namespace, and round-trips losslessly.
> **Stories:**
> - S6 (Dashboard / Observer, Watching workflow execution in real time) — As an observer, I want to open a WebSocket and subscribe to one workflow, a filtered class, or the whole firehose so that I see events as they happen.

## Purpose

Define the wire shape the WebSocket layer parses: how a client expresses what slice of the event firehose it wants, and how a streamed event is framed. This is the contract the server's stream module (AW-010) maps onto the engine's EventFilter.

## Task

Add events.proto declaring a SubscriptionRequest with per-workflow, filtered (by type/status/namespace), and firehose variants, and the streamed Event envelope that carries an aion-core Event unmodified plus the owning namespace. Provide serde types and assert lossless round-trip. This brief defines the wire shape only; the server-side subscription loop is AW-010.

## Requirements

### R1: SubscriptionRequest variants

THE SYSTEM SHALL define a SubscriptionRequest with three subscription models: per-workflow (a single WorkflowId), filtered (by workflow type, status, and/or namespace), and firehose (all events in the caller's namespace).

**Acceptance:**
- events.proto defines a SubscriptionRequest expressing the three models (e.g. a oneof or a tagged variant).
- The per-workflow variant carries a WorkflowId; the filtered variant carries optional type, status, and namespace selectors; the firehose variant carries the caller's namespace.
- Every variant carries the caller's namespace so the server can scope it before mapping to an EventFilter (per D8).
- The SubscriptionRequest has a serde type deriving Serialize + Deserialize and round-trips losslessly.

### R2: Streamed Event envelope

THE SYSTEM SHALL define the streamed event frame as an envelope carrying the aion-core Event unmodified plus the owning namespace, round-tripping losslessly.

**Acceptance:**
- The streamed frame carries the aion-core Event (via the AW-002 envelope, not a wire clone) plus the namespace it belongs to.
- A unit test round-trips a streamed frame containing an Event of at least one variant and asserts equality with the source aion-core Event.
- The frame derives Serialize + Deserialize (CO7).
- Inspection confirms the Event is carried by aion-core's type, not re-declared (CO12).

## Boundaries

- SHALL NOT implement the WebSocket forward loop or the EventFilter mapping — that is AW-010.
