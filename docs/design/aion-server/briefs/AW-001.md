# AW-001: aion-proto crate scaffold and tonic-build wiring

> **Cluster:** aion-server
> **Checklist:**
> - C1 — aion-proto is a workspace member depending on aion-core plus external crates only (prost, tonic, serde) — no aion, aion-server, or store-backend dependency.
> - C2 — build.rs compiles the .proto files via tonic-build and the generated code is included through a thin lib.rs that contains only declarations and re-exports.
> - C3 — aion-proto sets unsafe_code = "deny" and inherits the workspace clippy lints; cargo check -p aion-proto passes.
> **Stories:**
> - S1 (Client SDK Author, Building a caller SDK (cluster AL) against the wire contract) — As a client SDK author, I want one published wire contract with gRPC service definitions so that I can generate stubs for my language instead of inventing my own request and response shapes.
> - S4 (Worker SDK Author, Building a remote-worker SDK (cluster AR) against the wire contract) — As a worker SDK author, I want my build to depend only on the wire contract and the domain types, never the engine, so that my worker binary stays small and decoupled.

## Purpose

Create the aion-proto leaf crate — the shared wire contract every client SDK, worker SDK, and the server depend on. Wire up the proto-compilation toolchain so later briefs add .proto files and get generated Rust types. This is the foundation of the entire wire layer.

## Task

Create the aion-proto crate as a workspace member depending only on aion-core plus external crates (prost, tonic, serde, and tonic-build as a build dependency). Add a build.rs that compiles the crate's proto/ directory via tonic-build, and a thin lib.rs that includes the generated code and re-exports it alongside the hand-written serde/convert/error modules added in later briefs. Empty placeholder proto compilation is acceptable here; the message families land in AW-002..AW-005.

## Requirements

### R1: Crate scaffold and workspace wiring

THE SYSTEM SHALL create the aion-proto crate as a workspace member with a lib.rs that contains only module declarations and re-exports. THE SYSTEM SHALL declare dependencies on aion-core and external crates only, with no dependency on aion, aion-server, or any store backend.

**Acceptance:**
- crates/aion-proto exists and is registered in the workspace Cargo.toml members list.
- aion-proto/Cargo.toml declares aion-core and external crates (prost, tonic, serde) plus tonic-build under [build-dependencies] — and no aion-* dependency other than aion-core.
- src/lib.rs contains only pub mod declarations and pub use re-exports, no logic.
- The crate sets unsafe_code = "deny" and inherits the workspace clippy lints.
- cargo check -p aion-proto passes.

- create: crates/aion-proto/src/lib.rs

### R2: tonic-build proto compilation

THE SYSTEM SHALL add a build.rs that compiles every .proto under the crate's proto/ directory via tonic-build, emitting generated Rust into OUT_DIR. THE SYSTEM SHALL include the generated code through lib.rs.

**Acceptance:**
- build.rs invokes tonic-build (or prost-build via tonic-build) over the proto/ directory and reruns when a .proto changes (cargo:rerun-if-changed).
- lib.rs includes the generated module(s) via include!(concat!(env!("OUT_DIR"), ...)) behind a generated module, kept to declarations and re-exports.
- A minimal committed .proto (or the AW-002 common.proto if landed first) compiles cleanly so the build succeeds.
- cargo build -p aion-proto succeeds and the generated types are reachable from the crate root.
