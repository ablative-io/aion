# AW-002: Common wire types, proto<->core conversions, and the WireError taxonomy

> **Cluster:** aion-server
> **Depends on:** AW-001
> **Checklist:**
> - C4 — common.proto and convert.rs define the proto<->aion-core mapping for WorkflowId, RunId, ActivityId, TimerId, Payload, and WorkflowStatus with lossless round-trip.
> - C5 — The wire Event/WorkflowFilter/WorkflowSummary reuse aion-core's types via a thin envelope (namespace, request id, serialised core payload); no wire clone of those types exists.
> - C6 — WireError is a closed taxonomy with stable codes covering not-found, namespace-denied, sequence-conflict, unknown-query, query-timeout, not-running, lagged, and backend categories.
> - C7 — WireError maps from aion's EngineError, aion-store's StoreError, and query/signal/namespace failures, with a documented mapping and no Debug-formatted internals on the wire.
> **Stories:**
> - S2 (Client SDK Author, Building a caller SDK (cluster AL) against the wire contract) — As a client SDK author, I want failures to arrive as stable error codes so that my SDK can branch on 'workflow not found' versus 'namespace denied' without parsing strings.
> - S4 (Worker SDK Author, Building a remote-worker SDK (cluster AR) against the wire contract) — As a worker SDK author, I want my build to depend only on the wire contract and the domain types, never the engine, so that my worker binary stays small and decoupled.

## Purpose

Define the shared primitives every other message family uses: the proto representations of aion-core's identifiers, payload, and status; the lossless conversions between proto and aion-core types; and the WireError taxonomy that turns the engine's and store's typed errors into stable, language-neutral codes.

## Task

Add common.proto declaring proto messages for WorkflowId, RunId, ActivityId, TimerId, Payload, WorkflowStatus, and a thin envelope that wraps a serialised aion-core value with a namespace and request id. Write convert.rs giving infallible-where-possible, lossless conversions between these proto messages and aion-core types. Define WireError as a closed taxonomy with stable codes and write the mapping from aion's EngineError, aion-store's StoreError, and query/signal/namespace failures onto it. Reuse aion-core's Event/WorkflowFilter/WorkflowSummary directly via the envelope — do not clone them as wire structs.

## Requirements

### R1: Common proto messages and core conversions

THE SYSTEM SHALL define proto messages for WorkflowId, RunId, ActivityId, TimerId, Payload, and WorkflowStatus, and SHALL provide lossless conversion between each and its aion-core counterpart.

**Acceptance:**
- common.proto defines messages mirroring aion-core WorkflowId, RunId, ActivityId, TimerId, Payload, and WorkflowStatus.
- convert.rs implements From/TryFrom both directions for each type; a unit test round-trips every type proto->core->proto and core->proto->core and asserts equality.
- Payload conversion preserves the content-type tag and bytes exactly (lossless for all JSON value kinds, mirroring aion-core's Payload guarantee).
- Conversions that cannot fail use From; conversions that can (malformed proto) use TryFrom returning a WireError.

### R2: Envelope reuses aion-core types; no wire clones

THE SYSTEM SHALL carry aion-core Event, WorkflowFilter, and WorkflowSummary across the wire via a thin envelope (namespace, request id, serialised aion-core value), NOT via re-declared wire-clone structs.

**Acceptance:**
- A WireEnvelope (or equivalent) carries a namespace, an optional request id, and a serialised aion-core value (e.g. a Payload-encoded or bytes-encoded core type).
- Event, WorkflowFilter, and WorkflowSummary are serialised from aion-core's own serde derives, not from any wire-specific duplicate type.
- A grep/inspection confirms aion-proto declares no struct that re-fields Event, WorkflowFilter, WorkflowSummary, or WorkflowStatus (per CO12).
- A unit test round-trips an aion-core Event of at least one variant through the envelope and asserts equality.

### R3: WireError closed taxonomy

THE SYSTEM SHALL define WireError as a closed taxonomy carrying a stable code and a human message, covering not-found, namespace-denied, sequence-conflict, unknown-query, query-timeout, not-running, lagged, and backend categories.

**Acceptance:**
- WireError has variants (with stable integer or string codes) for NotFound, NamespaceDenied, SequenceConflict, UnknownQuery, QueryTimeout, NotRunning, Lagged, and Backend.
- WireError derives Serialize + Deserialize, implements std::error::Error and Display, and is Send + Sync.
- Each code is documented as stable so client SDKs may branch on it; the human message is informational only.
- WireError is representable as a proto message in common.proto and round-trips losslessly.

### R4: Mapping engine and store errors onto WireError

THE SYSTEM SHALL map aion's EngineError, aion-store's StoreError, and query/signal/namespace failures onto WireError, never exposing Debug-formatted internals on the wire.

**Acceptance:**
- A documented mapping turns each EngineError and StoreError variant into a specific WireError code (e.g. StoreError::SequenceConflict -> WireError::SequenceConflict, EngineError unknown-workflow -> NotFound).
- Namespace authorisation failures map to NamespaceDenied; query timeout and unknown-query map to QueryTimeout and UnknownQuery; a terminated/unknown workflow maps to NotRunning.
- No mapping path forwards a Debug or Display of the underlying error verbatim as the code; the human message is the only free-text field (per D3).
- A unit test asserts representative mappings for an EngineError and a StoreError variant.
- The mapping lives in aion-proto and references aion-store's StoreError; it references aion's EngineError type only structurally — note that if EngineError is not reachable from a leaf crate, the mapping for engine errors is provided as a documented conversion the server (AW-008/AW-009) applies, and aion-proto owns only the StoreError + namespace + query/signal mappings. State which split is used.

## Boundaries

- SHALL NOT define the workflow, events, or worker service messages — AW-003, AW-004, AW-005.
- SHALL NOT depend on aion (the engine crate); if EngineError mapping cannot live in a leaf crate, the engine-error mapping is applied server-side and documented as such (R4 records the split).
- SHALL NOT re-declare Event/WorkflowFilter/WorkflowSummary/WorkflowStatus as wire-clone structs (CO12).
- SHALL NOT implement any server handler or transport — those are AW-008 onward.
