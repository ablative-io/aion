# AT-012: map: dynamic fan-out from a runtime list

> **Cluster:** aion-time-signals
> **Depends on:** AT-010
> **Checklist:**
> - C32 — map applies a function to each element of a runtime list to produce child specs (dynamic fan-out), then collects like all (ordered, fail-fast).
> **Stories:**
> - S6 (Workflow Author, Writing Durable, Interactive, Concurrent Workflows) — As a workflow author, I want to fan work out with all, race the fastest of several with race, and fan out dynamically from data with map so that I can express concurrency directly without writing a join runtime.

## Purpose

Implement the map primitive: apply a function to each element of a runtime list to produce the child specs (dynamic fan-out), then collect like all — ordered, fail-fast. This is how a workflow fans out work derived from data (e.g. one child per order item), built on the all collector from AT-010.

## Task

Implement map over a runtime list and a spec-producing function. Apply the function to each element to build the child specs, then delegate collection to the all primitive (AT-010) so the result is ordered by input position and fails fast on any child failure (cancelling the rest).

## Requirements

### R1: Produce child specs from a runtime list

WHEN map is called with a runtime list and a spec-producing function, THE SYSTEM SHALL apply the function to each element to produce one child spec per element (dynamic fan-out).

**Acceptance:**
- map.rs defines map(items, f) that applies f to each element of items to produce a Vec of child specs, one per element, preserving element order.
- An empty input list produces zero children and yields an empty result (documented and tested).
- A test passes a 4-element list and asserts 4 child specs are produced in element order.

### R2: Collect like all (ordered, fail-fast)

WHEN the child specs are produced, THE SYSTEM SHALL collect them via the all primitive so results are returned in input order and any child failure fails the whole map and cancels the rest.

**Acceptance:**
- map delegates collection to all (AT-010) over the produced specs; it does not reimplement collection or cancellation.
- The result list is ordered by input element position; a child failure fails the whole map and cancels remaining children (inherited from all).
- A test fans out a list, delivers results out of order, and asserts ordered output.
- A test delivers one failure mid-flight and asserts map returns the failure and remaining children are cancelled.
