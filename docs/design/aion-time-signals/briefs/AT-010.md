# AT-010: all: fan-out, ordered collect, fail-fast

> **Cluster:** aion-time-signals
> **Depends on:** AT-009
> **Checklist:**
> - C30 — all spawns N linked children, blocks until all N results arrive, and returns them in input order; any child failure fails the whole all and cancels the remaining children.
> **Stories:**
> - S6 (Workflow Author, Writing Durable, Interactive, Concurrent Workflows) — As a workflow author, I want to fan work out with all, race the fastest of several with race, and fan out dynamically from data with map so that I can express concurrency directly without writing a join runtime.

## Purpose

Implement the all primitive: spawn N children concurrently, collect every result, and return them in input order. Any child failure fails the whole all and cancels the rest. This is the fan-out workhorse for parallel activities and child workflows, built on the correlation/cancellation substrate from AT-009.

## Task

Implement all over a fixed list of child specs. Spawn each as a linked child carrying its correlation token (AT-009), block in selective receive until all N results arrive, and return them ordered by input position. On the first child failure, fail the whole all and cancel the remaining in-flight children via the AT-009 cancel path.

## Requirements

### R1: Spawn N and collect in input order

WHEN all is called with N child specs, THE SYSTEM SHALL spawn N linked children, block until all N results arrive, and return the results in input order.

**Acceptance:**
- all.rs defines all(specs) that spawns each spec as a linked child tagged with its correlation token (AT-009 R1) and collects via the AT-009 matcher.
- The returned collection preserves input order regardless of the order results arrive (verified by delivering results out of order in a test).
- all blocks until exactly N results have arrived before returning success.
- A test fans out 3 children, delivers results out of order, and asserts the returned list is in input order with the correct values.

### R2: Fail-fast: a child failure fails all and cancels the rest

IF any child fails, THEN THE SYSTEM SHALL fail the whole all with that child's error AND cancel the remaining in-flight children via the AT-009 cancel path.

**Acceptance:**
- On receiving a failure result for any child, all stops collecting, cancels the remaining in-flight children (AT-009 R3, recording their cancellation), and returns the failing child's error.
- Children that already completed before the failure are not resurrected; the all result is the error.
- A test fans out 3 children, delivers one failure while two are in flight, and asserts: all returns the failure error, and the two in-flight children were cancelled and their cancellation recorded.
- A test asserts no panic and no partial success value is returned on failure.
