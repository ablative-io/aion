# AT-006: Non-resident signal delivery and terminal/unknown handling

> **Cluster:** aion-time-signals
> **Depends on:** AT-005
> **Checklist:**
> - C17 — Signalling a workflow whose process is not currently resident still records SignalReceived; the signal is delivered to the mailbox once AE makes the process resident.
> - C18 — Signalling a terminal or unknown workflow returns a typed error and records nothing.
> **Stories:**
> - S4 (Workflow Author, Writing Durable, Interactive, Concurrent Workflows) — As a workflow author, I want a signal sent while my workflow was momentarily not resident to still reach me so that no external event is silently dropped.

## Purpose

Make signals durable regardless of whether the target process is currently resident. A signal sent while a workflow is momentarily suspended or being restored must still be recorded and delivered once the workflow is resident again — no external event is silently dropped. A signal to a terminal or unknown workflow must fail cleanly with a typed error.

## Task

Extend the SignalRouter so that when the target process is NonResident, the signal is still recorded as SignalReceived and queued for delivery on resume via the engine seam; and when the target is Terminal or Unknown, the router records nothing and returns a typed error. Implement the resume handoff path so a recorded-but-undelivered signal reaches the mailbox once AE makes the process resident.

## Requirements

### R1: Record-and-defer for non-resident targets

WHEN a signal targets a workflow whose process is NonResident, THE SYSTEM SHALL still append SignalReceived AND register the signal for delivery to the mailbox once AE makes the process resident.

**Acceptance:**
- resume.rs handles the NonResident branch: the SignalReceived event is appended (so it is durable / replayable) and the signal is registered with the seam for delivery-on-resume.
- The router does not fail or drop the signal when the target is NonResident (CO7 — no silent drop).
- A test marks a workflow NonResident, routes a signal, asserts SignalReceived is appended and the signal is registered for resume delivery (observable via the fake seam).
- A follow-up in the same test transitions the workflow to Resident, drives the resume handoff, and asserts the mailbox receives the deferred signal.

**Files:**

- modify: crates/aion/src/signal/mod.rs

### R2: Resume handoff delivers deferred signals in order

WHEN AE makes a previously non-resident workflow resident, THE SYSTEM SHALL deliver any recorded-but-undelivered signals to its mailbox, in the order they were recorded.

**Acceptance:**
- The resume handoff delivers deferred signals to the mailbox in recorded order (the SignalReceived sequence order).
- Once delivered live, a deferred signal is not delivered a second time on a subsequent resume (delivery is tracked so resume is not a duplicate source); on a full replay, AD — not this path — returns the recorded signal.
- A test defers two signals for a NonResident workflow, marks it Resident, runs the handoff, and asserts both arrive in recorded order exactly once.

### R3: Terminal/unknown target returns a typed error, records nothing

IF a signal targets a workflow that is Terminal or Unknown, THEN THE SYSTEM SHALL return a typed error AND SHALL NOT append any SignalReceived event.

**Acceptance:**
- Routing to a Terminal workflow returns a typed SignalError (e.g. WorkflowTerminal) and appends nothing.
- Routing to an Unknown workflow returns a typed SignalError (e.g. WorkflowNotFound) and appends nothing.
- The error type is a thiserror enum in this module's error surface (no anyhow, CO5).
- Tests assert: terminal target -> typed error + zero events appended; unknown target -> typed error + zero events appended.

## Boundaries

- SHALL NOT manage workflow residency/lifecycle itself — it reacts to seam-reported residency and registers deferred delivery; AE owns the transition to Resident.
- SHALL NOT implement replay of deferred signals — AD returns recorded signals on a full restore; this brief covers live resume handoff only.
- SHALL NOT change the resident happy-path delivery from AT-005 beyond adding the non-resident/terminal/unknown branches.
