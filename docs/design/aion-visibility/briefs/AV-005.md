# AV-005: Server visibility endpoints and integration test

> **Cluster:** aion-visibility
> **Depends on:** AV-001, AV-002, AV-003, AV-004
> **Checklist:** C10, C12
> **Stories:** S1, S2, S4, S6

## Purpose

Expose visibility queries via gRPC and HTTP endpoints in aion-server, and provide an end-to-end integration test proving workflows can be found by custom search attributes.

## Task

Add visibility query endpoints to aion-server: ListWorkflows (filtered) and CountWorkflows. Write an integration test that starts a workflow, sets custom search attributes from the engine, and verifies the workflow can be found by attribute value through the visibility store.

## Requirements

### R1: gRPC visibility query endpoints

THE SYSTEM SHALL add gRPC service methods: ListWorkflows (takes filter, returns Vec<WorkflowSummary>) and CountWorkflows (takes filter, returns count).

**Acceptance:**
- Proto definitions exist for ListWorkflows and CountWorkflows.
- Server handlers delegate to the VisibilityStore.
- Filter fields are mappable from proto to ListWorkflowsFilter.

**Files:**
- modify: crates/aion-proto/src/lib.rs
- modify: crates/aion-server/src/grpc.rs

**Checklist:**
- C10

**Stories:**
- S6

### R2: HTTP visibility query endpoints

THE SYSTEM SHALL add HTTP endpoints: GET /workflows (list with query params for filters) and GET /workflows/count (count with query params).

**Acceptance:**
- GET /workflows returns a JSON array of WorkflowSummary.
- Query params filter by type, status, time range, and custom attributes.
- GET /workflows/count returns a JSON count.
- Pagination via limit and offset query params.

**Files:**
- modify: crates/aion-server/src/http.rs

**Checklist:**
- C10

**Stories:**
- S4

### R3: Integration test: search by custom attributes

THE SYSTEM SHALL have an integration test that: starts a workflow, records SearchAttributesUpdated with a customer_id attribute, and verifies list_workflows with a customer_id filter returns the workflow.

**Acceptance:**
- Workflow with customer_id=12345 appears in list_workflows filtered by customer_id=12345.
- Workflow does NOT appear when filtered by customer_id=99999.
- Filtering by workflow_type + customer_id narrows results correctly.
- count_workflows returns 1 for the matching filter.

**Files:**
- create: crates/aion/tests/visibility.rs

**Checklist:**
- C12

**Stories:**
- S1
- S2
