# AV-004: Engine visibility integration and SDK search attribute binding

> **Cluster:** aion-visibility
> **Depends on:** AV-001, AV-002, AV-003
> **Checklist:** C7, C8
> **Stories:** S3, S4

## Purpose

Wire visibility into the engine: update the visibility index on every state-changing event, and provide the SDK binding for workflow authors to set custom search attributes from workflow code.

## Task

Add a VisibilityStore to the EngineBuilder. After every state-changing event (WorkflowStarted, WorkflowCompleted, WorkflowFailed, WorkflowCancelled, WorkflowContinuedAsNew, SearchAttributesUpdated), the engine upserts the visibility record. Add a Recorder method record_search_attributes_updated for the SDK to call. The SDK binding (workflow.upsert_search_attributes) records a SearchAttributesUpdated event which triggers the visibility update.

## Requirements

### R1: Engine updates visibility on state-changing events

THE SYSTEM SHALL update the VisibilityStore on WorkflowStarted, WorkflowCompleted, WorkflowFailed, WorkflowCancelled, WorkflowContinuedAsNew, and SearchAttributesUpdated events. Each update upserts the full WorkflowSummary with current status, times, and search attributes.

**Acceptance:**
- After WorkflowStarted, the workflow appears in list_workflows with status Running.
- After WorkflowCompleted, the workflow's status is Completed and close_time is set.
- After WorkflowFailed, the workflow's status is Failed.
- After SearchAttributesUpdated, the workflow's search attributes are updated in the visibility index.

**Files:**
- modify: crates/aion/src/engine/builder.rs
- modify: crates/aion/src/lifecycle/start.rs
- modify: crates/aion/src/lifecycle/terminate.rs

### R2: Recorder records SearchAttributesUpdated

THE SYSTEM SHALL add record_search_attributes_updated to the Recorder that appends a SearchAttributesUpdated event with the provided attribute map. The Recorder SHALL validate attribute types against the registered schema.

**Acceptance:**
- Recorder::record_search_attributes_updated exists.
- After recording, the event appears in read_history.
- Setting an attribute with a type that doesn't match the schema returns an error.
