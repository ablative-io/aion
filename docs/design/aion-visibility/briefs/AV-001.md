# AV-001: Search attribute types, schema, and SearchAttributesUpdated event

> **Cluster:** aion-visibility
> **Checklist:** C1, C2, C9
> **Stories:** S3, S5

## Purpose

Define the core domain types for workflow visibility: search attribute type taxonomy, attribute schema registration, and the SearchAttributesUpdated event. These are the foundation the visibility store, engine integration, and SDK bindings build on.

## Task

Add search attribute types to aion-core: SearchAttributeType enum (String, Int, Float, Bool, Datetime, KeywordList), SearchAttributeValue enum (typed values), SearchAttributeSchema (per-namespace registry of attribute name to type), and SearchAttributesUpdated event variant carrying a map of attribute name to SearchAttributeValue. Type mismatches on set are errors.

## Requirements

### R1: SearchAttributeType and SearchAttributeValue

THE SYSTEM SHALL define SearchAttributeType with variants String, Int, Float, Bool, Datetime, KeywordList. THE SYSTEM SHALL define SearchAttributeValue as a typed enum with a variant per type carrying the concrete value.

**Acceptance:**
- SearchAttributeType has exactly six variants.
- SearchAttributeValue has matching variants with concrete types (String, i64, f64, bool, DateTime<Utc>, Vec<String>).
- Both round-trip through serde correctly.

**Files:**
- create: crates/aion-core/src/search.rs
- modify: crates/aion-core/src/lib.rs

**Checklist:**
- C1

**Stories:**
- S3

### R2: SearchAttributeSchema with type registration and validation

THE SYSTEM SHALL define SearchAttributeSchema as a per-namespace registry mapping attribute names to their declared SearchAttributeType. Setting an attribute with a value that doesn't match its declared type SHALL return a typed error.

**Acceptance:**
- SearchAttributeSchema::register(name, type) records the type for a name.
- SearchAttributeSchema::validate(name, value) returns Ok if the value matches the declared type, Err otherwise.
- Registering the same name with a different type returns an error.
- Setting an unregistered attribute name returns an error.

**Files:**
- modify: crates/aion-core/src/search.rs

**Checklist:**
- C9

**Stories:**
- S5

### R3: SearchAttributesUpdated event variant

THE SYSTEM SHALL add SearchAttributesUpdated to the Event enum carrying an EventEnvelope, workflow_id, and attributes (HashMap<String, SearchAttributeValue>). All exhaustive matches on Event SHALL handle the new variant.

**Acceptance:**
- Event::SearchAttributesUpdated exists with the specified fields.
- The variant serializes and deserializes correctly.
- All exhaustive matches in aion-core handle the new variant.
- cargo clippy -p aion-core --all-targets -- -D warnings passes.

**Files:**

- modify: crates/aion-core/src/filter.rs
