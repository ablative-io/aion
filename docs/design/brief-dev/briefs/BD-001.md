# BD-001: Stage contracts, brief document types, and generated codecs

> **Cluster:** brief-dev
> **Design anchor:**

Breaking changes are made cleanly and consumers move forward. Rejected: incremental deprecation cycles — they double the surface under test for an audience of zero.

Rejected: a sibling runs/ ledger — the brief as a single spec-plus-record document was the original intent, and aion's event history already provides the append-only audit trail.
> **Checklist:**

> - C4 — Brief document types (authored fields + optional scout/dev/review per requirement + optional execution block) are defined in Gleam with hand-written codecs that round-trip the cluster's own brief files (fixture: docs/design/brief-dev/briefs/BD-001.json) losslessly with deterministic encoding.

## Purpose

Lands the v2 stage contracts inside the stacked-dev package and makes the brief document itself a typed value the workflow family can carry. The three stage-contract schemas (scout-report, dev-report, review-report) are copied from the design-system canon, drift-gated byte-for-byte, and fed to aion codegen so the stage payload codecs are generated from the same schemas authors validate against (ADR-007, S5). The brief document and the pre-resolved reference context get hand-written Gleam types and codecs, round-tripped against this cluster's own seeded brief files, so every later brief (prompts, pipeline, enrichment, dispatch) builds on one typed vocabulary instead of unstructured document strings.

## Task

Three strands of work in the examples/stacked-dev package. (1) Schemas: copy the three stage-contract canon files from docs/design-system/schemas/ into examples/stacked-dev/schemas/ under the package naming convention (canon uses hyphens and a .schema.json suffix: scout-report.schema.json becomes scout_report.json, dev-report.schema.json becomes dev_report.json, review-report.schema.json becomes review_report.json — codegen requires snake_case stems). Content is byte-identical; only the filename changes. A drift-gate script encodes exactly that rename mapping and runs from the gleeunit suite. (2) Codegen: aion codegen reads EVERY schemas/*.json in the project (verified against docs/guides/codegen.md and crates/aion-cli/src/codegen.rs) and fails the whole run on $ref/$defs. Of the six existing workflow-IO schemas, exactly one — schemas/onatopp_input.json — uses $ref/$defs; the other five are already inside the codegen v1 subset. So this brief inlines onatopp_input.json's $defs/workspace into its workspace property (a semantics-preserving rewrite; the file itself is retired later by BD-003, but C3 requires codegen --check green at THIS brief's landing, so the inline happens here — called out as a scope-forward edit). The generated module src/aion_stacked_dev_io.gleam (module name comes from gleam.toml name = aion_stacked_dev) will therefore also contain types for the six workflow-IO schemas; those are an unused byproduct kept in sync by --check — the workflow modules keep using the hand-written codecs in codecs_workflows.gleam, because workflow ERROR unions can never be generated (codegen v1 rejects unions; design non-goal). (3) Types and codecs: brief document types (authored fields, optional scout/dev/review blocks per requirement, optional execution block — mirroring docs/design-system/schemas/brief.schema.json field-for-field) and resolved-context types in src/stacked_dev/types.gleam, with hand-written codecs in a new module src/stacked_dev/codecs_brief.gleam (hand-written because the brief document is not a package dispatch schema; its canon lives in the design system — C4). Round-trip tests use docs/design/brief-dev/briefs/BD-001.json — this very file — as the seeded fixture (the checklist's 'seeded RM-011 brief files' phrasing predates the seeding; the brief-dev cluster's own briefs are the real seeded files). 'Byte-faithfully' is delivered as lossless-and-deterministic: decode loses nothing, encode is canonical and stable (decode-encode-decode yields an equal value and a byte-identical re-encode); reproducing the pretty-printed on-disk formatting is NOT required — on-disk writing belongs to BD-004's enrichment. Scope-forward additions, called out per BRIEF.md: ResolvedAdr carries decided_by beyond the minimal {id,title,decision,quote} so projections can attribute quotes to their speaker (P6/C7), and ResolvedContext carries intention, constraints, and design_path because PROMPTING.md's 'What Every Stage Gets' requires design context and the design file reference in every stage prompt — BD-002 consumes these. Note on C2/C3 wording: the drift gate 'runs as part of the test gate' via a gleeunit test in this brief; registering codegen --check 'in CI' beyond the command passing locally rides the cluster gates item C38 (BD-007). Coverage split: C4 and C5 are delivered across R4 (types), R5 (codecs), and R6 (round-trip tests). The new files codecs_brief.gleam, test/schema_drift_test.gleam, test/brief_codecs_test.gleam, and test/support/fixtures.* are not in the design.json structure array (the structure pins no unit-test or codec-module paths for this work); per CLAUDE.md the brief is authoritative over the structure annotation.
