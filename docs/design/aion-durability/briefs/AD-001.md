# AD-001: Durability module scaffold and error taxonomy

> **Cluster:** aion-durability
> **Checklist:**
> - C1 — crates/aion/src/durability/mod.rs declares the durability submodules and re-exports the public surface, containing no logic.
> - C2 — DurabilityError is a thiserror enum covering at least sequence/append failure (wrapping StoreError), non-determinism violation, and history-shape errors, with no panics or unwrap in its construction paths.
> - C3 — NonDeterminismError is a distinct typed error carrying the WorkflowId, the sequence position of the mismatch, the expected command shape, and the found recorded-event shape.
> - C4 — DurabilityError implements std::error::Error and Display via thiserror, and no library code path constructs an error by panicking.

## Purpose

Establish the aion::durability module subtree and the error types every other brief in the cluster returns. The taxonomy is foundational: the Recorder, Resolver, replay, and recovery all surface failures through DurabilityError, and the NonDeterminismError shape is the contract the violation-detection path fills in.

## Task

Create the durability module under the aion crate (crates/aion/src/durability/) with a mod.rs that only declares submodules and re-exports. Define the error taxonomy in error.rs: a DurabilityError thiserror enum and a distinct NonDeterminismError carrying enough context to locate the offending workflow code. Wire pub mod durability into the aion crate's lib.rs. This brief defines types and module structure only — no recording, replay, or resolution logic. Coordinate with AE (which owns aion's lib.rs and the rest of the crate): add only the pub mod durability declaration, do not modify unrelated engine wiring.

## Requirements

### R1: Durability module scaffold

THE SYSTEM SHALL create the durability module subtree under crates/aion/src/durability/. THE mod.rs SHALL contain only pub mod declarations and pub use re-exports and SHALL NOT contain any type, function, or trait definition.

**Acceptance:**
- crates/aion/src/durability/mod.rs declares submodules seq, recorder, correlation, cursor, command, resolver, error, determinism, executor, replay, recovery (declarations may be added incrementally by later briefs, but error is declared here).
- mod.rs contains no fn, struct, enum, trait, or impl item — only pub mod and pub use.
- crates/aion/src/lib.rs gains exactly one new line, pub mod durability;, and nothing else in this brief.
- cargo build -p aion succeeds with the new empty/partial module tree (placeholder submodules declared by later briefs are not required to exist yet beyond error).

**Files:**
- create: crates/aion/src/durability/mod.rs
- modify: crates/aion/src/lib.rs

**Checklist:**
- C1 — crates/aion/src/durability/mod.rs declares the durability submodules and re-exports the public surface, containing no logic.

### R2: DurabilityError taxonomy

THE SYSTEM SHALL define DurabilityError as a thiserror enum covering append/sequence failure (wrapping aion_store::StoreError), non-determinism violation (wrapping NonDeterminismError), and history-shape errors. THE SYSTEM SHALL NOT construct any DurabilityError by panicking or via unwrap/expect.

**Acceptance:**
- error.rs defines DurabilityError with at least: Store(#[from] StoreError), NonDeterminism(NonDeterminismError), and a HistoryShape variant for malformed/inconsistent recorded history (e.g. a result event with no preceding schedule).
- DurabilityError derives Debug and implements Display + std::error::Error via thiserror.
- No unwrap, expect, panic, todo, or unimplemented appears in error.rs.
- A unit test formats one value of each variant via Display and asserts the message is non-empty and mentions the underlying cause.

**Files:**
- create: crates/aion/src/durability/error.rs
- modify: crates/aion/src/durability/mod.rs

**Checklist:**
- C2 — DurabilityError is a thiserror enum covering at least sequence/append failure (wrapping StoreError), non-determinism violation, and history-shape errors, with no panics or unwrap in its construction paths.
- C4 — DurabilityError implements std::error::Error and Display via thiserror, and no library code path constructs an error by panicking.

### R3: NonDeterminismError shape

THE SYSTEM SHALL define NonDeterminismError as a typed error carrying the WorkflowId, the sequence position of the mismatch, a description of the expected command shape, and a description of the found recorded-event shape.

**Acceptance:**
- NonDeterminismError carries workflow_id: WorkflowId, seq: u64 (the cursor position of the mismatch), expected: String (or a small enum) naming the command family + correlation key the workflow issued, and found: String (or a small enum) naming the recorded event family + key.
- NonDeterminismError derives Debug, implements Display via thiserror, and its Display includes the workflow id, the position, and both shapes.
- NonDeterminismError derives Clone and PartialEq so tests can assert exact expected/found shapes.
- A unit test constructs a NonDeterminismError and asserts its Display string contains the workflow id, the sequence position, the expected, and the found descriptors.

**Files:**
- modify: crates/aion/src/durability/error.rs

**Checklist:**
- C3 — NonDeterminismError is a distinct typed error carrying the WorkflowId, the sequence position of the mismatch, the expected command shape, and the found recorded-event shape.
