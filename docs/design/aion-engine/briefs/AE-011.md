# AE-011: EngineBuilder and build()

> **Cluster:** aion-engine
> **Depends on:** AE-003, AE-004, AE-005, AE-006
> **Checklist:**
> - C28 — EngineBuilder offers new(), store(impl EventStore), scheduler_threads(n), load_workflows(...), register_nifs(...), and build().
> - C29 — build() wires the runtime, registers NIFs and loaded modules, repopulates the registry and re-creates supervisors from the store's list_active (delegating replay to AD), and returns EngineError::MissingStore if no store was supplied.
> **Stories:**
> - S1 (Engine Operator, Embedding and Running the Engine) — As an engine operator, I want to build an Engine from a store, loaded workflows, and my NIFs with one builder so that embedding durable workflows in my Rust app is a single, clear assembly step.
> - S2 (Engine Operator, Embedding and Running the Engine) — As an engine operator, I want to set the scheduler thread count (and have a sensible runtime default when I don't) so that I tune concurrency to my host without the engine guessing a number for me.

## Purpose

Provide the single assembly path for the engine. The builder collects a store, scheduler thread count, loaded workflows, and NIFs, then build() wires the runtime, installs NIFs and modules, repopulates the registry from the store, and returns a live Engine. This is the one construction path (D10).

## Task

Implement engine/builder.rs: EngineBuilder with new(), store(...), scheduler_threads(n), load_workflows(...), register_nifs(...), and build(). build() constructs the RuntimeHandle from the thread count, installs the collected NIFs, loads the workflow packages (registering namespaced modules and recording versions), re-creates per-type supervisors and repopulates the registry from the store's list_active (delegating the actual replay to AD), and returns the Engine. A missing store is EngineError::MissingStore. Scope is the builder and build(); the Engine methods are AE-012.

## Requirements

### R1: Builder surface

THE SYSTEM SHALL provide EngineBuilder with new(), store(impl EventStore), scheduler_threads(n), load_workflows(...), register_nifs(...), and build(). THE SYSTEM SHALL accept the scheduler thread count from scheduler_threads and SHALL NOT default it to a hardcoded number.

**Acceptance:**
- builder.rs defines EngineBuilder::new() and the chainable setters store, scheduler_threads, load_workflows, and register_nifs returning Self.
- store accepts any impl EventStore + Send + Sync + 'static, stored as Arc<dyn EventStore>.
- scheduler_threads(n) records Some(n); if never called, the builder holds None (passed through to beamr per AE-002).
- load_workflows accepts package source(s) (a path and/or an already-loaded Package) and register_nifs accepts a host NIF set.
- engine/mod.rs contains only pub mod and pub use.

### R2: build() assembles the runtime, NIFs, and modules

WHEN build() is called, THE SYSTEM SHALL construct the RuntimeHandle from the configured thread count, install the engine's own and host-supplied NIFs before any workflow runs, and load the configured workflow packages (registering namespaced modules and recording versions). IF no store was supplied, THEN THE SYSTEM SHALL return EngineError::MissingStore.

**Acceptance:**
- build() with no store returns EngineError::MissingStore.
- build() constructs the RuntimeHandle (AE-002) with the configured Option<usize> thread count.
- build() installs all NIFs (AE-003) before loading or starting anything; a duplicate MFA surfaces as EngineError::NifRegistration.
- build() loads each configured package via the loader (AE-005); a load failure surfaces as EngineError::Load with nothing partially registered.
- A test builds an Engine over InMemoryStore with one loaded fixture package and asserts success; a test omitting the store asserts MissingStore.

### R3: Repopulate registry and supervisors from list_active

WHEN build() completes runtime setup, THE SYSTEM SHALL read the store's list_active, re-create the per-type supervisor for each active workflow's type, and repopulate the registry, delegating the actual state reconstruction (replay) to AD.

**Acceptance:**
- build() calls EventStore::list_active and, for each active workflow, ensures its per-type supervisor (AE-006) and inserts a registry handle whose status is reconciled from its event history (AE-004 reconcile).
- The actual replay of workflow state is delegated to AD via a documented seam (a hook/trait call), not implemented here.
- A test seeds InMemoryStore with one active workflow's WorkflowStarted, builds the Engine, and asserts the registry contains a handle for it with the projected status and its per-type supervisor exists.
- An empty store yields an Engine with an empty registry and no per-type supervisors.

## Boundaries

- SHALL NOT implement the replay engine — build() triggers AD's replay via a seam; reconstruction is AD's.
- SHALL NOT implement the Engine's runtime methods (start/cancel/result/list/shutdown) — those are AE-012.
- SHALL NOT add networking or a server — build() wires only the embedded runtime, store, registry, and supervision.
