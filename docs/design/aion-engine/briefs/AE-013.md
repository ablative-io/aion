# AE-013: Delegated signal, query, and subscribe surface

> **Cluster:** aion-engine
> **Depends on:** AE-012
> **Checklist:**
> - C33 — Engine surfaces signal, query, and subscribe whose registry-lookup step lives here while their delivery mechanics delegate to AT (signal routing, query dispatch) and AD/AT (event publishing).
> **Stories:**
> - S7 (Workflow Caller, Starting, Cancelling, and Inspecting Workflows) — As a workflow caller, I want to list and query workflows by type, status, and time so that I can see what is running and what has finished without loading full histories.

## Purpose

Surface signal, query, and subscribe on Engine for API completeness, performing the registry-lookup step that AE owns and delegating the delivery mechanics to AT (signal routing, query dispatch) and AD/AT (event publishing). This fixes the method surface and the seam without implementing AT's machinery.

## Task

Implement engine/delegated.rs: signal, query, and subscribe methods on Engine. Each resolves the target workflow via the registry (the AE-owned lookup) and then calls into the AT/AD delivery seam (a trait/hook the Engine holds) to perform routing/dispatch/publishing. This brief provides the method surface, the lookup, and the seam wiring — it does not implement signal routing, query dispatch, or the event publisher.

## Requirements

### R1: signal surface with registry lookup

WHEN signal(&id, &run, name, payload) is called, THE SYSTEM SHALL resolve the target workflow handle via the registry and SHALL hand the signal to the AT signal-routing seam. IF the workflow is not registered, THEN THE SYSTEM SHALL return EngineError::WorkflowNotFound. THE SYSTEM SHALL NOT implement the mailbox routing itself.

**Acceptance:**
- delegated.rs defines Engine::signal that looks up the handle and, on success, calls the AT signal-routing seam (a SignalRouter trait/handle the Engine holds) with the target pid, name, and payload.
- An unregistered workflow returns EngineError::WorkflowNotFound.
- grep of delegated.rs shows no direct mailbox-send / selective-receive routing logic — only the seam call.
- A test installs a fake SignalRouter seam, signals a registered workflow, and asserts the seam received the (pid, name, payload); signalling an unknown id returns WorkflowNotFound.

### R2: query surface with registry lookup

WHEN query(&id, &run, name) is called, THE SYSTEM SHALL resolve the target workflow handle via the registry and SHALL hand the query to the AT query-dispatch seam, returning the reply Payload. IF the workflow is not registered, THEN THE SYSTEM SHALL return EngineError::WorkflowNotFound. THE SYSTEM SHALL NOT implement the query dispatch itself.

**Acceptance:**
- delegated.rs defines Engine::query that looks up the handle and calls the AT query-dispatch seam (a QueryService trait/handle) with the target pid and query name, returning the reply Payload.
- An unregistered workflow returns EngineError::WorkflowNotFound.
- grep of delegated.rs shows no query-handler invocation logic — only the seam call.
- A test installs a fake QueryService seam returning a known Payload and asserts query returns it for a registered workflow.

### R3: subscribe surface delegating to the event publisher

WHEN subscribe(filter: EventFilter) is called, THE SYSTEM SHALL return an event stream produced by the AD/AT event-publishing seam. THE SYSTEM SHALL NOT implement the broadcast/publishing mechanism itself.

**Acceptance:**
- delegated.rs defines Engine::subscribe(filter) -> stream that delegates to the event-publisher seam (an EventPublisher trait/handle the Engine holds) and returns its filtered stream.
- grep of delegated.rs shows no broadcast-channel implementation — only the seam call.
- A test installs a fake EventPublisher seam that emits two events and asserts subscribe yields them filtered by the given EventFilter.

## Boundaries

- SHALL NOT implement signal routing into mailboxes, query dispatch to workflow processes, or the durable signal/query event recording — those are AT (delivery) and AD (recording).
- SHALL NOT implement the event publisher / broadcast channel — that is AD/AT; this brief consumes its seam.
- SHALL NOT add networking — WebSocket fan-out of the subscribe stream is AW; subscribe here returns an in-process stream.
