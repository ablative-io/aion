# AE-015: Eliminate silent visibility store fallback in EngineBuilder

> **Cluster:** aion-engine

## Purpose

EngineBuilder silently falls back to an in-memory visibility store when none is configured. In production this means all visibility data is lost on restart with zero warning, violating the principle of no silent failures. This brief makes the missing visibility store an explicit error and adds an opt-in method for tests that intentionally want ephemeral visibility.

## Task

Remove the silent InMemoryStore fallback from EngineBuilder::build(). When no visibility store is configured and in_memory_visibility() has not been called, build() returns EngineError::MissingVisibilityStore. Add an explicit in_memory_visibility() builder method that opts in to ephemeral visibility for test scenarios. Update all test code to call in_memory_visibility() instead of relying on the silent fallback.

## Requirements

### R1: EngineBuilder returns error when no visibility store is configured

WHEN EngineBuilder::build() is called AND no visibility store has been configured AND in_memory_visibility() has not been called, THE SYSTEM SHALL return EngineError::MissingVisibilityStore instead of silently falling back to InMemoryStore.

**Acceptance:**
- EngineError has a MissingVisibilityStore variant with a descriptive message explaining what to do.
- EngineBuilder::build() checks whether a visibility store was provided or in_memory_visibility() was called; if neither, it returns Err(EngineError::MissingVisibilityStore).
- The error message names both the visibility_store() and in_memory_visibility() methods so the caller knows their options.
- A test confirms that building without a visibility store returns the MissingVisibilityStore error.

**Files:**
- modify: crates/aion/src/engine/builder.rs
- modify: crates/aion/src/engine/mod.rs

### R2: Add explicit in_memory_visibility() opt-in method

THE SYSTEM SHALL expose an in_memory_visibility() method on EngineBuilder that explicitly configures an ephemeral in-memory visibility store. THIS method SHALL be the only path to get an in-memory visibility store; the silent fallback is removed.

**Acceptance:**
- EngineBuilder::in_memory_visibility(self) -> Self is a public method that sets the visibility store to InMemoryStore.
- The method is documented as intended for test scenarios and explicitly warns that visibility data will not survive restarts.
- Calling in_memory_visibility() followed by build() succeeds without error.
- Calling visibility_store() after in_memory_visibility() (or vice versa) uses the last-set store, matching the builder's general last-write-wins semantics.

**Files:**
- modify: crates/aion/src/engine/builder.rs

### R3: Update all test code to use in_memory_visibility()

WHEN test code constructs an EngineBuilder, THE SYSTEM SHALL call in_memory_visibility() explicitly instead of relying on the removed silent fallback. All existing tests SHALL continue to pass after the migration.

**Acceptance:**
- Every test that previously relied on the silent InMemoryStore fallback now calls in_memory_visibility() on the builder.
- No test breaks due to the fallback removal; cargo test -p aion passes.
- A grep for EngineBuilder usage confirms every test-context builder calls either visibility_store() or in_memory_visibility().
- No test uses #[ignore] or #[allow] to suppress the missing-store error.

**Files:**
- modify: crates/aion/src/engine/builder.rs
- modify: crates/aion/src/engine/mod.rs
