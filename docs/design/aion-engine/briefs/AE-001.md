# AE-001: aion crate scaffold and EngineError taxonomy

> **Cluster:** aion-engine
> **Checklist:**
> - C1 — The aion crate exists as a workspace member depending only on aion-core, aion-store, aion-package, and beamr among workspace crates, with unsafe_code = "deny".
> - C2 — src/lib.rs contains only pub mod declarations and pub use re-exports, no logic.
> - C3 — EngineError is a thiserror enum covering at least MissingStore, Load, Store(StoreError), Package(PackageError), Runtime, RegistryPoisoned, WorkflowNotFound, and NifRegistration.
> - C4 — No #[allow]/#[expect]/#[ignore] appears in the crate, and no .unwrap()/.expect() appears outside #[cfg(test)] code.
> **Stories:**
> - S4 (Engine Operator, Embedding and Running the Engine) — As an engine operator, I want the engine to depend on no network stack so that I can run durable workflows in a CLI or single service with nothing but a file-backed store.

## Purpose

Create the aion engine crate and define the error taxonomy every other module returns. This is the foundation the runtime, registry, lifecycle, supervision, loader, and engine modules build on, and it fixes the crate's dependency boundary up front.

## Task

Create the aion crate as a workspace member depending only on aion-core, aion-store, aion-package, and beamr among workspace crates, with a thin lib.rs re-export surface and an error module defining EngineError. The crate must contain no transport (HTTP/gRPC/WebSocket) dependency. Define the EngineError thiserror enum with the variants the rest of the cluster needs. Scope is the crate skeleton and the error type only — no runtime, registry, lifecycle, or engine logic.

## Requirements

### R1: Crate scaffold and dependency boundary

THE SYSTEM SHALL create the aion crate as a workspace member whose only workspace dependencies are aion-core, aion-store, aion-package, and beamr. THE SYSTEM SHALL NOT add any HTTP, gRPC, or WebSocket dependency, and SHALL NOT depend on aion-server or any store backend crate.

**Acceptance:**
- crates/aion exists and is registered in the workspace Cargo.toml members list.
- aion/Cargo.toml lists aion-core, aion-store, aion-package, and beamr as the only workspace-internal dependencies; external deps are limited to async/serde/thiserror-style support crates with no networking crate present.
- The crate sets unsafe_code = "deny" and inherits the workspace clippy lints.
- cargo check -p aion passes.
- grep of the crate shows no http/grpc/tonic/axum/hyper/tungstenite dependency or import.

### R3: EngineError thiserror taxonomy

THE SYSTEM SHALL define EngineError as a thiserror enum covering MissingStore, Load, Store wrapping aion_store StoreError, Package wrapping aion_package PackageError, Runtime, RegistryPoisoned, WorkflowNotFound, and NifRegistration. THE SYSTEM SHALL NOT use anyhow in the crate.

**Acceptance:**
- error.rs defines EngineError with at least the variants MissingStore, Load { reason }, Store(StoreError), Package(PackageError), Runtime { reason }, RegistryPoisoned, WorkflowNotFound(WorkflowId), and NifRegistration { reason }.
- EngineError derives thiserror::Error and Debug with a Display message per variant.
- From<StoreError> and From<PackageError> are implemented (or derived via #[from]) so the ? operator lifts those errors.
- No anyhow import appears anywhere in the crate.
