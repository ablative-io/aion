> **Cluster:** aion-package
> **Checklist:**
> - C1 — crates/aion-package exists, is registered as a workspace member, and depends only on aion-core among Aion crates (plus external crates: zip, serde, a hashing crate, thiserror).
> - C2 — src/lib.rs contains only pub mod declarations and pub use re-exports, no logic, and sets unsafe_code = "deny".
> - C3 — PackageError is a closed thiserror enum with distinct variants for: not-a-ZIP/archive-read failure, missing manifest, manifest parse failure, unknown format_version, missing entry module, integrity (hash) mismatch, and malformed beam entry.
> - C4 — BeamModule carries a logical module name and its exact compiled bytes; BeamSet is an ordered collection of BeamModules.
> - C5 — BeamSet exposes a canonical ordering (sorted by logical module name) that is deterministic and independent of insertion order.
> **Stories:**
> - S1 (Engine, Loading and Registering Workflows) — As the engine, I want to load a .aion from a path or byte buffer and get a validated package so that I can register its modules without trusting loose files on disk.
> - S5 (Workflow Author, Packaging and Deploying a Workflow) — As a workflow author, I want to package my compiled beams and a manifest into a single .aion file so that deploying my workflow is copying one file.

## Purpose

Create the aion-package leaf crate and the two foundational types every later brief builds on: the PackageError taxonomy that every failure path maps into, and the BeamModule/BeamSet types that carry compiled modules with a canonical ordering. This is the foundation for the manifest, hashing, write, and read paths.

## Task

Create the aion-package crate as a workspace member depending only on aion-core among Aion crates. Add a thin lib.rs re-export surface. Define PackageError as a closed thiserror enum covering every malformed-input case the read path can hit. Define BeamModule (logical name + exact bytes) and BeamSet (an ordered collection of BeamModules) with a deterministic canonical ordering sorted by logical name. Do not define the manifest, hashing, namespacing, or the write/read paths — those land in later briefs.

### R2: PackageError taxonomy

THE SYSTEM SHALL define PackageError as a closed thiserror enum with a distinct variant for every malformed-input case: archive-read/not-a-ZIP failure, missing manifest, manifest parse failure, unknown format_version, missing entry module, integrity (hash) mismatch, and malformed beam entry.

**Acceptance:**
- error.rs defines PackageError as a thiserror enum (no anyhow).
- Distinct variants exist for: ArchiveRead (not a ZIP / read failure, wrapping the underlying cause), MissingManifest, ManifestParse, UnknownFormatVersion { found }, MissingEntryModule { module }, IntegrityMismatch { expected, computed }, and MalformedBeamEntry { entry }.
- PackageError derives Debug, implements std::error::Error and Display via thiserror, and is Send + Sync.
- Each variant's Display message names the failing condition precisely (no generic catch-all message).
- No variant is a bare String catch-all that collapses distinct failures together.

**Files:**
- create: crates/aion-package/src/error.rs
- modify: crates/aion-package/src/lib.rs

**Checklist:**
- C3 — PackageError is a closed thiserror enum with distinct variants for: not-a-ZIP/archive-read failure, missing manifest, manifest parse failure, unknown format_version, missing entry module, integrity (hash) mismatch, and malformed beam entry.

**Stories:**

### R3: BeamModule and BeamSet with canonical ordering

THE SYSTEM SHALL define BeamModule carrying a logical module name and its exact compiled bytes, and BeamSet as an ordered collection of BeamModules with a deterministic canonical ordering sorted by logical module name and independent of insertion order.

**Acceptance:**
- beam.rs defines BeamModule { name: String, bytes: Vec<u8> } (or equivalent) preserving bytes exactly, deriving Clone, Debug, PartialEq.
- BeamSet stores BeamModules and exposes a canonical iteration order sorted by logical module name.
- Constructing a BeamSet from the same modules in different insertion orders yields the same canonical order (a unit test asserts this).
- BeamSet rejects or is documented to reject duplicate logical module names so the canonical order is unambiguous.
- BeamSet exposes lookup of a module's bytes by logical name.

**Files:**
- create: crates/aion-package/src/beam.rs
- modify: crates/aion-package/src/lib.rs
