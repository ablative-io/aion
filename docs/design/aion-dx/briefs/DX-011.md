# DX-011: Document undocumented features and fix dashboard references

> **Cluster:** aion-dx
> **Depends on:** DX-007

## Purpose

Multiple server features are undocumented: 12 AION_ environment variable overrides, the AION_LOG log-level control, workflow package loading feedback, and the dashboard is a stub but docs claim it works. A developer debugging server behavior has no reference for any of these.

## Task

Add documentation for all environment variables, log a confirmation when workflow packages load, add a notice that the dashboard UI is under development, and document that server logs are JSON (recommend piping through jq).

## Requirements

### R1: Document environment variables

GETTING-STARTED.md or a dedicated docs/CONFIGURATION.md SHALL list all AION_ environment variable overrides (AION_SERVER_LISTEN_ADDRESS, AION_STORE_BACKEND, AION_STORE_URL, AION_RUNTIME_SCHEDULER_THREADS, AION_DRAIN_TIMEOUT_SECONDS, AION_AUTH_ENABLED, AION_METRICS_ENABLED, AION_NAMESPACES_DEFAULT, AION_LOG). The Python worker's env vars (AION_WORKER_ENDPOINT, AION_TASK_QUEUE, AION_WORKER_IDENTITY, AION_WORKER_CONCURRENCY) SHALL be documented in the worker example or SDK README.

**Acceptance:**
- All AION_ server env vars are listed with descriptions.
- AION_LOG is documented with example: AION_LOG=debug.
- Python worker env vars are documented.

**Files:**
- modify: GETTING-STARTED.md
- modify: examples/hello-world/README.md

### R2: Log workflow package loading and fix dashboard references

WHEN the server loads a workflow package, THE SYSTEM SHALL emit an INFO log line with the package logical name and content hash. All references to the dashboard SHALL note that the UI is under development and suggest using the CLI or HTTP API to observe workflows.

**Acceptance:**
- Server startup with a workflow package shows 'loaded workflow package <name>' in logs.
- Dashboard references in docs say 'under development' and suggest alternatives.
- No doc claims the dashboard is functional.

**Files:**
- modify: crates/aion/src/engine/builder.rs
- modify: GETTING-STARTED.md
- modify: examples/hello-world/README.md

## Boundaries

- SHALL NOT add new environment variables — only document existing ones.
- SHALL NOT implement the dashboard UI — only fix the docs about its current state.
- SHALL NOT change existing config parsing behavior.

## Verification

- grep -r 'AION_LOG' GETTING-STARTED.md returns at least one match.
- Server loading a .aion package emits a log line with the package name.
- No README claims the dashboard shows workflow runs.
