# DX-010: Fix hello-world workflow and example code

> **Cluster:** aion-dx

## Purpose

The hello-world workflow ignores its input and always greets 'Alice' regardless of what the developer sends. The Gleam source has 7 unused functions that confuse anyone reading it. The example should actually demonstrate passing data through a workflow — that's the whole point of a demo.

## Task

Fix hello_world.gleam to use the input parameter so the workflow greets the name the developer provides. Remove dead code (unused definition, codecs, error functions). Ensure the workflow demonstrates the core concept: input flows through an activity and produces a result.

## Requirements

### R1: Workflow uses input parameter

WHEN a developer starts the hello_world workflow with input {"name":"Ada"}, THE SYSTEM SHALL pass "Ada" to the greet activity and the workflow result SHALL contain "Hello, Ada!". THE SYSTEM SHALL NOT hardcode any name.

**Acceptance:**
- Starting with {"name":"Ada"} produces a greeting containing 'Ada'.
- Starting with {"name":"Bob"} produces a greeting containing 'Bob'.
- The Gleam run function uses its input parameter (no leading underscore).

**Files:**
- modify: examples/hello-world/src/hello_world.gleam

### R2: Remove dead code from example

THE SYSTEM SHALL remove unused functions from hello_world.gleam: definition(), greet_activity(), local_greet(), greeting_input_codec(), greeting_output_codec(), workflow_error_codec(), activity_error_message(). Only the run function and its direct dependencies SHALL remain.

**Acceptance:**
- hello_world.gleam contains only the run function and imports it actually uses.
- gleam build succeeds with no warnings about unused functions.
- The example is short enough that a developer can read it in under 30 seconds.

**Files:**
- modify: examples/hello-world/src/hello_world.gleam

## Boundaries

- SHALL NOT change the .aion package format or the NIF bridge contract.
- SHALL NOT add new Gleam dependencies.
- SHALL rebuild the hello-world.aion package if the Gleam source changes.

## Verification

- gleam build succeeds in examples/hello-world/.
- The rebuilt .aion package loads in the server.
- Workflow started with {"name":"Test"} returns a result containing 'Test'.
