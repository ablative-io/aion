# DX-003: Human-readable JSON payloads in API responses and requests

> **Cluster:** aion-dx

## Purpose

The describe endpoint returns payloads as raw byte arrays ([123, 34, 119, ...]). Starting a workflow requires pre-encoding input as {content_type, bytes}. Both are hostile to developers using curl or any HTTP client. The API should accept and return human-readable JSON where the content type is application/json.

## Task

Add JSON decoding/encoding at the HTTP/gRPC boundary: responses with content_type application/json decode the bytes to a JSON value before returning. Requests accept plain JSON objects as input (auto-wrapping in the Payload envelope). Preserve raw bytes for non-JSON content types.

## Requirements

### R1: Decode JSON payloads in describe and history responses

WHEN a response includes a Payload with content_type application/json, THE SYSTEM SHALL decode the bytes field to a JSON value and return it as a structured object (e.g. {"content_type": "application/json", "data": {"name": "Ada"}}). For non-JSON content types, THE SYSTEM SHALL return the raw bytes as base64.

**Acceptance:**
- Describe endpoint returns decoded JSON input/output fields, not byte arrays.
- History events with JSON payloads show decoded data.
- Non-JSON payloads (binary) return base64-encoded bytes.
- The content_type field is always present.

**Files:**
- modify: crates/aion-server/src/api/http.rs
- modify: crates/aion-server/src/api/handlers.rs

### R2: Accept plain JSON input for workflow start

WHEN a start-workflow request includes an 'input' field that is a plain JSON object (not a Payload envelope), THE SYSTEM SHALL auto-wrap it as {content_type: 'application/json', bytes: <serialized>}. WHEN the input is already a Payload envelope (has content_type and bytes fields), THE SYSTEM SHALL use it as-is.

**Acceptance:**
- curl with '"input": {"name": "Ada"}' starts the workflow successfully.
- curl with the full envelope format still works (backwards compatible).
- The error message for invalid input shows the expected format with an example.

**Files:**
- modify: crates/aion-server/src/api/http.rs
- modify: crates/aion-server/src/api/handlers.rs

## Boundaries

- SHALL NOT change the internal Payload type in aion-core.
- SHALL NOT change the gRPC wire format (proto stays as-is). This is HTTP-layer sugar only.
- SHALL NOT decode payloads in the engine — only at the server boundary.

## Verification

- cargo clippy --workspace --all-targets -- -D warnings passes.
- cargo fmt --check passes.
- cargo test --workspace passes.
- curl start-workflow with plain JSON input succeeds.
- curl describe returns human-readable JSON payloads.
