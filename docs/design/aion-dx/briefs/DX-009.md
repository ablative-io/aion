# DX-009: Fix Python worker connection lifecycle and reconnect

> **Cluster:** aion-dx

## Purpose

The Python worker logs 'Connected successfully' before any TCP connection occurs (gRPC channels are lazy), then crashes on first real RPC if the server isn't running. The reconnect loop logs 'reconnecting' but immediately re-raises the exception, terminating the program. Both behaviors mislead developers and prevent production use.

## Task

Fix the Python worker SDK so that (1) 'Connected' is logged only after a successful register handshake with the server, and (2) the reconnect loop actually retries with exponential backoff instead of crashing.

## Requirements

### R1: Defer connection success log until server handshake

WHEN the Python worker starts, THE SYSTEM SHALL NOT log 'Connected successfully' until the register request has been sent and acknowledged by the server. IF the server is unreachable, THE SYSTEM SHALL log an ERROR with the connection failure reason.

**Acceptance:**
- Starting the worker with no server running does NOT show 'Connected successfully'.
- Starting the worker with no server running shows an error message with the address.
- Starting the worker with a running server shows 'Connected' only after registration completes.

**Files:**
- modify: sdks/python/aion-worker/aion_worker/loop.py
- modify: sdks/python/aion-worker/aion_worker/session.py

### R2: Fix reconnect loop to actually retry

WHEN a worker session fails, THE SYSTEM SHALL retry with exponential backoff using the ReconnectConfig parameters (initial_delay, max_delay, max_attempts). THE SYSTEM SHALL NOT immediately re-raise the exception. THE SYSTEM SHALL log each retry attempt with the delay and attempt number.

**Acceptance:**
- Worker reconnects after server restart without manual intervention.
- Reconnect logs show 'Reconnecting in Xs (attempt N/M)'.
- Worker exits cleanly after max_attempts is exceeded.
- ReconnectConfig parameters (delays, max_attempts) are respected.

**Files:**
- modify: sdks/python/aion-worker/aion_worker/loop.py
- modify: sdks/python/aion-worker/aion_worker/reconnect.py

## Boundaries

- SHALL NOT change the worker gRPC protocol.
- SHALL NOT add new Python dependencies.
- SHALL NOT change the public WorkerConfig API.

## Verification

- Worker started without server shows error, not 'Connected successfully'.
- Worker started with server shows 'Connected' after registration.
- Worker survives server restart and reconnects automatically.
