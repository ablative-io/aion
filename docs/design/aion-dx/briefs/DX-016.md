# DX-016: Rewrite hello-world to use aion_flow SDK

> **Cluster:** aion-dx

## Purpose

The hello-world example currently calls raw_nif() directly, bypassing the typed Gleam SDK (aion_flow). The README claims the example uses aion_flow but the code never imports it. This mismatch misleads new developers and fails to demonstrate the recommended authoring path. The example should showcase workflow.define(), activity.new(), and workflow.run() from the aion_flow SDK so it serves as a correct, copy-pasteable starting point.

## Task

Rewrite hello_world.gleam to import and use the aion_flow SDK: define the workflow with workflow.define(), declare activities with activity.new(), and execute them with workflow.run(). Update gleam.toml to add the aion_flow dependency if missing. Update the README to accurately reflect the code. Remove all raw_nif() calls from the example.

## Requirements

### R1: Use typed aion_flow SDK for workflow definition

WHEN a developer opens the hello-world example, THE SYSTEM SHALL define the workflow using workflow.define() from the aion_flow SDK. THE SYSTEM SHALL NOT call raw_nif() or any internal FFI functions directly.

**Acceptance:**
- hello_world.gleam imports from aion/workflow, aion/activity, or aion_flow.
- The workflow is defined using workflow.define() with a named workflow type.
- No calls to raw_nif() or aion/internal/ffi remain in the example source.
- gleam.toml lists aion_flow as a dependency.

**Files:**
- modify: examples/hello-world/src/hello_world.gleam
- modify: examples/hello-world/gleam.toml

### R2: Demonstrate typed Activity with input and output

THE SYSTEM SHALL declare at least one activity using activity.new() with explicit input and output types, so the example demonstrates the typed activity pattern end-to-end.

**Acceptance:**
- At least one activity is created via activity.new() with a descriptive name.
- The activity has typed input (e.g. String) and typed output (e.g. String).
- The activity is invoked within the workflow via workflow.run() or the equivalent SDK call.
- The example compiles with gleam build without warnings.

**Files:**
- modify: examples/hello-world/src/hello_world.gleam

### R3: README matches the actual code

THE SYSTEM SHALL update the hello-world README so that every code snippet, import path, and description accurately reflects the rewritten hello_world.gleam source.

**Acceptance:**
- README references aion_flow SDK, not raw NIFs.
- Any code snippets in the README compile and match the actual source.
- The README does not mention raw_nif() or internal FFI functions.
- Build and run instructions in the README still work.

**Files:**
- modify: examples/hello-world/README.md

## Boundaries

- SHALL NOT change the aion_flow SDK itself — only consume it.
- SHALL NOT change the worker SDK or server code.
- SHALL NOT add new dependencies beyond aion_flow to the hello-world example.
- SHALL preserve the existing hello-world worker.py so the Python worker side still works.

## Verification

- gleam build succeeds in the hello-world directory without warnings.
- The .aion package can be built from the rewritten source.
- hello-world workflow completes end-to-end when run against the dev server with the Python worker.
- README code snippets match the actual source in hello_world.gleam.
- No references to raw_nif() or aion/internal/ffi remain in the example directory.
