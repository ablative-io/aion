# DX-008: CLI tool for workflow operations

> **Cluster:** aion-dx
> **Depends on:** DX-007

## Purpose

Developers and AI agents should not have to construct curl commands with byte-array payloads and auth headers to interact with workflows. The aion-client library already has start/signal/query/cancel/list/describe operations. A thin CLI binary on top gives both humans and AI a natural command-line interface: 'aion start hello_world --input {"name":"Ada"}'.

## Task

Create an aion-cli binary crate that wraps the aion-client library with clap-based subcommands for start, signal, query, cancel, list, and describe. Support --endpoint, --namespace, --subject flags. Output JSON by default, with --pretty for human-readable formatting. Update GETTING-STARTED.md and hello-world README to use the CLI instead of curl.

## Requirements

### R1: CLI binary with workflow subcommands

THE SYSTEM SHALL provide an aion-cli binary with subcommands: start <workflow_type> [--input JSON], signal <workflow_id> <signal_name> [--payload JSON], query <workflow_id> <query_name>, cancel <workflow_id> [--reason TEXT], list [--status STATUS], describe <workflow_id>. Global flags: --endpoint (default 127.0.0.1:50051), --namespace (default 'default'), --subject (default 'cli-user').

**Acceptance:**
- cargo run -p aion-cli -- start hello_world --input '{"name":"Ada"}' starts a workflow and prints the workflow ID.
- cargo run -p aion-cli -- describe <workflow_id> prints workflow status and events.
- cargo run -p aion-cli -- list shows active workflows.
- --help on any subcommand shows usage.
- Output is JSON by default, parseable by jq.

**Files:**
- create: crates/aion-cli/Cargo.toml
- create: crates/aion-cli/src/main.rs
- modify: Cargo.toml

### R2: Update docs to use CLI instead of curl

GETTING-STARTED.md and examples/hello-world/README.md SHALL show the CLI commands as the primary interaction method. Curl examples MAY be retained in a 'HTTP API' section for reference but SHALL NOT be the main path.

**Acceptance:**
- GETTING-STARTED.md 'Start a workflow' step uses aion-cli, not curl.
- hello-world README uses aion-cli for start, describe, and signal.
- A developer can complete the hello-world tutorial using only the CLI.

**Files:**
- modify: GETTING-STARTED.md
- modify: examples/hello-world/README.md

## Boundaries

- SHALL NOT modify the aion-client library API — only build a CLI wrapper on top.
- SHALL NOT add schedule management subcommands in this brief — keep it to core workflow operations.
- SHALL use clap for argument parsing — no hand-rolled CLI parser.

## Verification

- cargo clippy --workspace --all-targets -- -D warnings passes.
- cargo fmt --check passes.
- cargo run -p aion-cli -- --help prints usage for all subcommands.
- cargo run -p aion-cli -- start hello_world --input '{"name":"Ada"}' returns a workflow ID when the server is running.
