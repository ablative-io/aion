# DX-013: Fix remaining doc inconsistencies and minor issues

> **Cluster:** aion-dx

## Purpose

Multiple doc inconsistencies remain after DX rounds 1 and 2: ghost config file reference, CLI flag name mismatch, GitHub org name inconsistency, Python version mismatch. These erode trust — a developer who hits wrong information in docs assumes the project is unmaintained.

## Task

Fix all remaining documentation inconsistencies in GETTING-STARTED.md, example READMEs, and pyproject.toml. Ensure every command, flag name, URL, and version number in documentation matches the actual code.

## Requirements

### R1: Fix all doc inconsistencies

THE SYSTEM SHALL ensure: (1) no references to nonexistent config files like aion-hello-world.toml, (2) CLI flag names match the actual parser (--drain-timeout not --drain-timeout-seconds), (3) GitHub org name is consistent everywhere (ablative-io/aion), (4) Python version requirement matches pyproject.toml (>=3.10). Every command shown in docs SHALL be copy-pasteable and work.

**Acceptance:**
- grep -r 'aion-hello-world.toml' returns zero matches.
- grep -r 'drain-timeout-seconds' docs/ returns zero matches.
- GitHub org name is consistent across all docs, Cargo.toml, and pyproject.toml.
- Python version requirement is consistent between docs and pyproject.toml.
- Every shell command in GETTING-STARTED.md is syntactically correct.

**Files:**
- modify: GETTING-STARTED.md
- modify: examples/hello-world/README.md

## Boundaries

- SHALL NOT change any Rust or Python code — docs only.
- SHALL NOT rewrite sections that are already correct — targeted fixes only.

## Verification

- All commands in GETTING-STARTED.md match the actual CLI parser.
- No references to nonexistent files.
- GitHub URLs are consistent.
