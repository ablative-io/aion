# DX-005: Fix README status, add aion_flow README, and root-level getting started

> **Cluster:** aion-dx

## Purpose

The root README says 'implementation not yet started' (false — 135+ briefs are landed and working). The aion_flow SDK has no README. There are no root-level prerequisites or getting-started docs. New developers either think the project isn't built or can't find where to start.

## Task

Update the root README status to reflect reality. Write a README for the aion_flow Gleam SDK explaining the API, types, and how to write a workflow. Add a root-level GETTING-STARTED.md with prerequisites and a quickstart path.

## Requirements

### R1: Fix root README status section

THE SYSTEM SHALL update the root README's Status section to accurately reflect: engine complete, server functional, worker SDKs available, three working examples. THE SYSTEM SHALL NOT claim features that don't work (e.g. don't claim dashboard is functional if it's an empty shell).

**Acceptance:**
- Status section says the engine is implemented and functional.
- Status section lists what works and what's in progress honestly.
- No claim of 'implementation not yet started' exists anywhere.

**Files:**
- modify: README.md

### R2: Write aion_flow SDK README

THE SYSTEM SHALL create a README.md in the aion_flow Gleam SDK directory explaining: what it is, how to define a workflow, how to declare activities, how to use signals/timers/queries, and a minimal example. THE SYSTEM SHALL reference the hello-world example for a complete working sample.

**Acceptance:**
- gleam/aion_flow/README.md exists.
- README explains the core API: workflow definition, activity calls, signals.
- README includes a minimal code example.
- README links to the hello-world example for a full walkthrough.

**Files:**
- create: gleam/aion_flow/README.md

### R3: Root-level GETTING-STARTED.md

THE SYSTEM SHALL create a GETTING-STARTED.md at the repo root listing prerequisites (Rust, Gleam, Python for workers), the quickstart path (hello-world example), and links to deeper documentation. THE SYSTEM SHALL be copy-pasteable — a developer can go from clone to running workflow in the listed steps.

**Acceptance:**
- GETTING-STARTED.md exists at repo root.
- Prerequisites listed: Rust toolchain, Gleam, Python (optional for workers).
- Steps to run hello-world are complete and copy-pasteable.
- Links to examples/, docs/API.md, and aion_flow README.

**Files:**
- create: GETTING-STARTED.md
- modify: README.md

## Boundaries

- SHALL NOT rewrite the entire README — only fix the Status section and add links.
- SHALL NOT document features that don't work yet (dashboard, some SDK features).
- SHALL NOT duplicate the hello-world README — link to it instead.

## Verification

- Root README Status section is accurate.
- gleam/aion_flow/README.md exists and is informative.
- GETTING-STARTED.md exists with copy-pasteable quickstart.
- No broken links in any of the documents.
