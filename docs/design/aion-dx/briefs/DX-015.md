# DX-015: Fix Python worker namespace authorization

> **Cluster:** aion-dx

## Purpose

The Python worker is immediately rejected with PERMISSION_DENIED when connecting to the dev server. The server requires x-aion-namespaces and x-aion-subject gRPC metadata on the worker stream, but the worker SDK doesn't send this metadata. The WorkerConfig has no namespace or subject fields. This blocks the entire hello-world tutorial at step 4.

## Task

Add namespace and subject fields to the Python worker SDK's WorkerConfig. Send them as gRPC metadata (x-aion-namespaces, x-aion-subject) on the worker stream connection. Update the hello-world worker.py example to set these. Also update the Rust worker SDK if it has the same issue.

## Requirements

### R1: Worker SDK sends namespace authorization metadata

WHEN a Python worker connects to the server, THE SYSTEM SHALL send x-aion-namespaces and x-aion-subject gRPC metadata derived from WorkerConfig fields. Default namespace SHALL be 'default' and default subject SHALL be 'worker'. THE SYSTEM SHALL also check and fix the Rust worker SDK if it has the same issue.

**Acceptance:**
- Python worker connects without PERMISSION_DENIED when using default config.
- Worker logs show 'Registered activities: greet' without namespace errors.
- The hello-world workflow completes end-to-end: start via CLI, activity dispatched to worker, result returned.
- WorkerConfig has namespace and subject fields with sensible defaults.

**Files:**
- modify: sdks/python/aion-worker/aion_worker/worker.py
- modify: sdks/python/aion-worker/aion_worker/session.py
- modify: examples/hello-world/worker.py
- modify: crates/aion-worker/src/protocol/session.rs

## Boundaries

- SHALL NOT change the gRPC protocol definitions.
- SHALL NOT change the server's namespace authorization logic.
- SHALL use existing gRPC metadata headers (x-aion-namespaces, x-aion-subject).

## Verification

- Python worker connects to dev server without PERMISSION_DENIED.
- hello-world workflow completes end-to-end with the Python worker.
- pip install still works without new dependencies.
