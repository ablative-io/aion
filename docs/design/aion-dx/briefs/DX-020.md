# DX-020: Publish SDKs to language package registries

> **Cluster:** aion-dx

## Purpose

The aion_flow Gleam SDK, Python SDKs (worker + client), and TypeScript SDKs (worker + client) are not published to their respective package registries (Hex, PyPI, npm). The three worker SDKs (Python, TypeScript, Rust) also lack README files. Until the SDKs are published and documented, developers cannot add Aion as a dependency in their projects through standard package management workflows.

## Task

Write README files for the three worker SDKs that lack them. Prepare and publish aion_flow to Hex, the Python packages (aion-worker, aion-client) to PyPI, and the TypeScript packages (aion-worker, aion-client) to npm. Ensure each package has correct metadata (name, version, description, license, repository URL) before publishing.

## Requirements

### R1: Worker SDK READMEs

THE SYSTEM SHALL create README.md files for the Python worker SDK, TypeScript worker SDK, and Rust worker SDK. Each README SHALL contain: a one-line description, installation instructions, a minimal usage example showing activity registration and worker startup, and a link to the main Aion repository.

**Acceptance:**
- sdks/python/aion-worker/README.md exists with installation and usage example.
- sdks/typescript/aion-worker/README.md exists with installation and usage example.
- crates/aion-worker/README.md exists with installation and usage example.
- Each README shows how to register an activity and start the worker.
- Each README includes the correct package name for its registry.

**Files:**
- create: sdks/python/aion-worker/README.md
- create: sdks/typescript/aion-worker/README.md
- create: crates/aion-worker/README.md

### R2: Publish Python packages to PyPI

THE SYSTEM SHALL publish aion-worker and aion-client Python packages to PyPI with correct metadata in pyproject.toml including name, version, description, license, repository URL, and Python version constraints.

**Acceptance:**
- aion-worker is installable via pip install aion-worker.
- aion-client is installable via pip install aion-client.
- pyproject.toml for each package has correct name, version, description, and license.
- Both packages include their README as the long description.

**Files:**
- modify: sdks/python/aion-worker/pyproject.toml
- modify: sdks/python/aion-client/pyproject.toml

### R3: Publish TypeScript packages to npm

THE SYSTEM SHALL publish aion-worker and aion-client TypeScript packages to npm with correct metadata in package.json including name, version, description, license, repository URL, and main/types entry points.

**Acceptance:**
- aion-worker is installable via npm install @aion/worker.
- aion-client is installable via npm install @aion/client.
- package.json for each package has correct name, version, description, and license.
- TypeScript type declarations are included in the published package.

**Files:**
- modify: sdks/typescript/aion-worker/package.json
- modify: sdks/typescript/aion-client/package.json

### R4: Publish aion_flow to Hex

THE SYSTEM SHALL publish the aion_flow Gleam package to Hex with correct metadata in gleam.toml including name, version, description, licences, and repository link.

**Acceptance:**
- aion_flow is installable via gleam add aion_flow.
- gleam.toml has correct name, version, description, and licences fields.
- The package includes all source files under src/.
- The published version matches the current source.

**Files:**
- modify: gleam/aion_flow/gleam.toml

## Boundaries

- SHALL NOT change any SDK source code — only metadata files and READMEs.
- SHALL NOT publish packages with version 1.0.0 — use 0.x.y pre-release versions.
- SHALL NOT include test files, build artifacts, or example code in published packages.
- SHALL use the @aion npm scope for TypeScript packages.
- SHALL coordinate package names to avoid conflicts with existing registry entries.

## Verification

- All three worker SDK README.md files exist and contain installation + usage sections.
- pyproject.toml files pass python -m build --sdist --wheel dry run.
- package.json files pass npm pack --dry-run without errors.
- gleam.toml passes gleam publish --dry-run.
- No test files or build artifacts are included in package manifests.
