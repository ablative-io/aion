# AC-005: aion-store crate, EventStore trait, and StoreError

> **Cluster:** aion-core
> **Depends on:** AC-002, AC-003, AC-004
> **Checklist:**
> - C23 — StoreError is a closed enum with SequenceConflict, NotFound, Backend, and Serialization variants.
> - C24 — EventStore is an async trait that is Send + Sync + 'static.
> - C25 — append accepts an expected sequence number, applies all events atomically, and returns SequenceConflict if the stored head does not match.
> - C26 — read_history returns the complete event history for a workflow in sequence order.
> - C27 — list_active returns the WorkflowIds of all non-terminal workflows.
> - C28 — query returns WorkflowSummary values matching a WorkflowFilter.
> - C29 — schedule_timer persists a durable timer and expired_timers returns timers due to fire as of a given instant.
> **Stories:**
> - S3 (Engine, Producing and Consuming Events) — As the engine, I want to list active workflows so that on startup I know which workflows to replay and resume.
> - S5 (Store Author, Implementing a Persistence Backend) — As a store author, I want a single clear EventStore trait so that there is exactly one reasonable way to implement a backend correctly.
> - S7 (Store Author, Implementing a Persistence Backend) — As a store author, I want to implement the trait without depending on the engine so that my backend crate stays decoupled.
> - S11 (Worker, Implementing Activities) — As a worker, I want to depend on the domain types without pulling in the store trait so that my worker binary stays small and decoupled.

## Purpose

Define the single persistence contract for Aion. The EventStore trait is what the engine talks to and what every backend implements, including external ones.

## Task

Create the aion-store leaf crate depending only on aion-core. Define StoreError as a closed enum. Define the EventStore async trait with append (expected-sequence guard), read_history, list_active, query, schedule_timer, and expired_timers. Define the TimerEntry type the timer methods exchange.

## Requirements

THE SYSTEM SHALL create the aion-store crate depending only on aion-core (no other Aion crate). THE SYSTEM SHALL define StoreError as a closed enum with SequenceConflict, NotFound, Backend, and Serialization variants.

**Acceptance:**

### R2: EventStore trait — event methods

THE SYSTEM SHALL define EventStore as an async trait that is Send + Sync + 'static, with append (taking an expected sequence and applying events atomically), read_history, list_active, and query.

**Acceptance:**
- store.rs defines the EventStore trait, async, bound Send + Sync + 'static.
- append(&self, workflow_id: &WorkflowId, events: &[Event], expected_seq: u64) -> Result<(), StoreError>, documented as atomic (all events apply or none) and returning SequenceConflict when the stored head does not equal expected_seq.
- read_history(&self, workflow_id: &WorkflowId) -> Result<Vec<Event>, StoreError> returns events in sequence order, and an empty Vec (never NotFound) for a workflow with no recorded events — the first append with expected_seq 0 creates the workflow implicitly, so 'unknown' and 'empty history' are the same observable state.
- list_active(&self) -> Result<Vec<WorkflowId>, StoreError> returns non-terminal workflows.
- query(&self, filter: &WorkflowFilter) -> Result<Vec<WorkflowSummary>, StoreError> returns matching summaries.
- read_history, list_active, and query never return StoreError::NotFound for absent workflows; NotFound is reserved for operations that target a must-exist workflow.
- Trait method docs state the contract precisely, including the atomicity, sequence-guard, and empty-history-for-unknown semantics.

### R3: Durable timer methods and TimerEntry

THE SYSTEM SHALL define TimerEntry and add schedule_timer and expired_timers to EventStore. schedule_timer SHALL persist a durable timer; expired_timers SHALL return timers due to fire as of a given instant.

**Acceptance:**
- timer.rs defines TimerEntry { workflow_id: WorkflowId, timer_id: TimerId, fire_at: DateTime<Utc> } deriving the standard traits.
- EventStore gains schedule_timer(&self, &WorkflowId, &TimerId, fire_at: DateTime<Utc>) -> Result<(), StoreError>.
- EventStore gains expired_timers(&self, as_of: DateTime<Utc>) -> Result<Vec<TimerEntry>, StoreError> returning timers whose fire_at <= as_of.
- Docs state that scheduling a timer is part of the same logical transaction as the TimerStarted event the engine appends.
- The trait remains object-safe (usable as Arc<dyn EventStore>).
