# AC-002: Event model and payload

> **Depends on:** AC-001

> **Stories:**
> - S1 (Engine, Producing and Consuming Events) — As the engine, I want to append a batch of events atomically with an expected sequence so that concurrent or duplicate writes are detected and replay integrity is preserved.
> - S2 (Engine, Producing and Consuming Events) — As the engine, I want to read a workflow's complete ordered history so that I can replay it to reconstruct the workflow's state.
> - S8 (Workflow Author, Writing Workflows via the SDK) — As a workflow author, I want time and identifiers to be deterministic so that my workflow replays to exactly the same state every time.

## Purpose

Define the Event enum — the spine of the event-sourcing system — and the Payload type that carries type-erased values through it. Every recorded fact in a workflow's life is an Event variant.

## Task

Define a Payload newtype carrying opaque serialised bytes plus a content-type tag, with lossless conversion to and from serde_json::Value. Define the Event enum with all lifecycle, activity, timer, signal, and child-workflow variants, each wrapped in an envelope carrying a per-workflow sequence number, a recorded timestamp, and the owning WorkflowId. Event must be type-erased — it carries Payload values, never a generic type parameter.

## Requirements

### R1: Payload newtype

THE SYSTEM SHALL define Payload as a newtype over a byte buffer plus a content-type tag. THE SYSTEM SHALL provide lossless conversion between Payload and serde_json::Value.

**Acceptance:**
- payload.rs defines Payload holding bytes and a content-type tag (an enum or stable string identifying the encoding).
- Payload::from_json(&serde_json::Value) and Payload::to_json() -> Result round-trip losslessly for all JSON value kinds.
- Payload derives Serialize, Deserialize, Clone, Debug, PartialEq.
- A unit test asserts round-trip fidelity for null, bool, number, string, array, and object.

### R2: Event envelope

THE SYSTEM SHALL define an event envelope carrying a monotonic per-workflow sequence number, a recorded UTC timestamp, and the owning WorkflowId. Every event SHALL carry this envelope.

**Acceptance:**
- event.rs defines an envelope (a struct field group or wrapper) with seq: u64, recorded_at: DateTime<Utc>, and workflow_id: WorkflowId.
- The envelope is part of every Event value, accessible via accessors regardless of variant.
- The recorded_at timestamp is the documented determinism source for workflow.now (a doc comment states this).
- A unit test constructs an event and reads back its seq, recorded_at, and workflow_id.

**Files:**
- create: crates/aion-core/src/event.rs
- modify: crates/aion-core/src/lib.rs

**Checklist:**

**Stories:**

### R3: Event variants

THE SYSTEM SHALL define the Event enum with workflow-lifecycle, activity, timer, signal, and child-workflow variants. Variants carrying user data SHALL carry it as Payload, and the enum SHALL NOT take a generic type parameter.

**Acceptance:**
- Workflow lifecycle: WorkflowStarted (workflow_type + input Payload), WorkflowCompleted (result Payload), WorkflowFailed (WorkflowError), WorkflowCancelled (reason).
- Activity: ActivityScheduled (ActivityId + activity_type + input Payload), ActivityStarted (ActivityId), ActivityCompleted (ActivityId + result Payload), ActivityFailed (ActivityId + ActivityError + attempt).
- Timer: TimerStarted (TimerId + fire_at), TimerFired (TimerId), TimerCancelled (TimerId).
- Signal: SignalReceived (name + payload Payload).
- Child workflow: ChildWorkflowStarted (child WorkflowId + workflow_type + input), ChildWorkflowCompleted (child WorkflowId + result), ChildWorkflowFailed (child WorkflowId + WorkflowError).
- Event has no generic type parameter; all user data is Payload.

### R4: Event serialisation round-trip

THE SYSTEM SHALL derive Serialize and Deserialize on Event and all its variants. Serialising an Event and deserialising it back SHALL produce an equal value.

**Acceptance:**
- Event derives Serialize, Deserialize, Clone, Debug, PartialEq.
- A unit test round-trips at least one event of every variant through serde_json and asserts equality.
- Enum representation is explicit (tagged) so the wire form is stable and forward-readable.
