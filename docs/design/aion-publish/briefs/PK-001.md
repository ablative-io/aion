# PK-001: Crate metadata and dependency audit for crates.io

> **Cluster:** aion-publish
> **Checklist:**
> - C1 — Every crate has a description in Cargo.toml
> - C2 — Every crate has keywords and categories for crates.io discovery
> - C3 — Every crate inherits license, repository, edition, rust-version from workspace
> - C4 — Every crate has a documentation URL pointing to docs.rs
> - C5 — No path-only dependencies remain — all inter-crate deps use version + path
> - C6 — All external dependencies are available on crates.io
> - C7 — beamr dependency is pinned to a published crates.io version
> **Stories:**
> - S2 (Rust Developer, External consumer who wants to embed Aion or build on its primitives) — I can discover Aion crates on crates.io by searching for 'workflow engine' or 'durable execution'.
> - S4 (Rust Developer, External consumer who wants to embed Aion or build on its primitives) — I can run cargo publish --dry-run on any crate to verify it's publishable.
> - S6 (Aion Maintainer, Team member who publishes new versions to crates.io) — I can verify that no crate has path-only dependencies that would prevent crates.io publication.

## Purpose

Add crates.io-required metadata to all 11 Aion crates and ensure inter-crate dependencies use version+path so cargo publish succeeds.

## Task

For each crate in the workspace: add description, keywords, categories, and documentation fields to Cargo.toml. Ensure license, repository, edition, and rust-version inherit from workspace. Audit all inter-crate dependencies to use version = "0.1.0" alongside path references. Verify beamr is pinned to a published crates.io version. Run cargo publish --dry-run for every crate in dependency order.

## Requirements

### R1: Per-crate description and discovery metadata

THE SYSTEM SHALL have a description, keywords array, and categories array in every crate's Cargo.toml. Keywords SHALL include 'workflow', 'durable-execution', and crate-specific terms. Categories SHALL include 'development-tools' or 'asynchronous' as appropriate.

**Acceptance:**
- All 11 crates have a non-empty description field.
- All 11 crates have keywords and categories arrays.
- cargo metadata --format-version 1 shows description for every workspace member.

**Files:**
- modify: crates/aion-core/Cargo.toml
- modify: crates/aion-store/Cargo.toml
- modify: crates/aion-store-libsql/Cargo.toml
- modify: crates/aion-package/Cargo.toml
- modify: crates/aion-nif/Cargo.toml
- modify: crates/aion/Cargo.toml
- modify: crates/aion-proto/Cargo.toml
- modify: crates/aion-server/Cargo.toml
- modify: crates/aion-worker/Cargo.toml
- modify: crates/aion-client/Cargo.toml
- modify: crates/aion-cli/Cargo.toml

**Checklist:**
- C1 — Every crate has a description in Cargo.toml
- C2 — Every crate has keywords and categories for crates.io discovery
- C4 — Every crate has a documentation URL pointing to docs.rs

**Stories:**
- S2 (Rust Developer, External consumer who wants to embed Aion or build on its primitives) — I can discover Aion crates on crates.io by searching for 'workflow engine' or 'durable execution'.

### R2: Workspace inheritance verification

THE SYSTEM SHALL ensure every crate inherits license, repository, edition, and rust-version from the workspace package section.

**Acceptance:**
- Every crate's Cargo.toml contains license.workspace = true.
- Every crate's Cargo.toml contains repository.workspace = true.
- cargo metadata confirms uniform license and repository across all members.

**Files:**
- modify: crates/aion-core/Cargo.toml
- modify: crates/aion-store/Cargo.toml
- modify: crates/aion-package/Cargo.toml
- modify: crates/aion-nif/Cargo.toml
- modify: crates/aion/Cargo.toml

**Checklist:**
- C3 — Every crate inherits license, repository, edition, rust-version from workspace

### R3: Inter-crate version+path dependencies

THE SYSTEM SHALL specify both version and path for every inter-crate dependency so that cargo publish resolves the version from crates.io while local development uses the path.

**Acceptance:**
- Every dependency on a workspace crate uses { version = "0.1.0", path = "../crate-name" } syntax.
- No dependency uses path-only without a version.
- cargo publish --dry-run succeeds for every crate.

**Files:**
- modify: Cargo.toml
- modify: crates/aion/Cargo.toml
- modify: crates/aion-store/Cargo.toml
- modify: crates/aion-store-libsql/Cargo.toml
- modify: crates/aion-server/Cargo.toml
- modify: crates/aion-worker/Cargo.toml
- modify: crates/aion-client/Cargo.toml
- modify: crates/aion-cli/Cargo.toml

**Checklist:**
- C5 — No path-only dependencies remain — all inter-crate deps use version + path
- C6 — All external dependencies are available on crates.io
- C7 — beamr dependency is pinned to a published crates.io version

**Stories:**
- S4 (Rust Developer, External consumer who wants to embed Aion or build on its primitives) — I can run cargo publish --dry-run on any crate to verify it's publishable.
- S6 (Aion Maintainer, Team member who publishes new versions to crates.io) — I can verify that no crate has path-only dependencies that would prevent crates.io publication.

## Boundaries

- SHALL add metadata to all 11 workspace crates.
- SHALL verify beamr is available on crates.io at the pinned version.
- SHALL NOT change any crate's public API or functionality.
- SHALL NOT bump version numbers (stays at 0.1.0 for initial publish).
- SHALL NOT actually publish to crates.io — dry-run only.

## Verification

- cargo clippy --workspace --all-targets -- -D warnings passes.
- cargo test --workspace passes.
- cargo fmt --check passes.
- cargo publish --dry-run succeeds for every crate in dependency order.
- cargo metadata --format-version 1 shows description, license, and repository for every member.
