# NB-004: Timer NIFs (sleep, start_timer, cancel_timer, with_timeout)

> **Cluster:** aion-nif-bridge
> **Depends on:** NB-001

> **Stories:**

## Purpose

Implement the timer NIFs that workflow code calls for durable sleep, named timer management, and timeout wrapping. Each NIF goes through the durability handoff: on replay, recorded timer events are returned without scheduling real timers; on live execution, the NIF delegates to AT's timer scheduling services (TimerService, SleepTimer, named timer API) through the engine's DelegatedSeams (D4) and records the durable events through the Recorder.

## Task

Create nif_timer.rs with production NIF implementations for sleep/1, start_timer/2, cancel_timer/1, and with_timeout/2. Each NIF constructs a NifContext (NB-001), builds the appropriate Command, calls resolve_or_execute_live. On replay, return the recorded result. On live, delegate to AT's concrete timer services (D4), record events through the Recorder (CO2), and return the result. Replace the stubs in engine_nifs.rs.

## Requirements

### R1: sleep/1 NIF records TimerScheduled and blocks until TimerFired

WHEN sleep/1 is called on the live path, THE SYSTEM SHALL record TimerScheduled through the Recorder, delegate to AT's sleep service to schedule the timer, block the dirty NIF thread until the timer fires or is cancelled, and record TimerFired through the Recorder (CO2). WHEN called during replay, THE SYSTEM SHALL return the recorded TimerFired result without scheduling a real timer (C18).

**Acceptance:**
- nif_timer.rs implements sleep_impl that constructs a NifContext, resolves the sleep Command, and on ResumeLive delegates to AT's sleep function (crate::time::sleep or equivalent).
- On the live path, TimerScheduled is recorded before the sleep, and TimerFired is recorded when the timer fires.
- On the replay path, the recorded TimerFired is returned immediately without blocking.
- A test records a sleep's full event trail on the live path and asserts TimerScheduled + TimerFired events appear in the store; a replay test asserts the NIF returns immediately.

### R4: with_timeout/2 NIF wraps an operation with a deadline

WHEN with_timeout/2 is called on the live path, THE SYSTEM SHALL schedule a deadline timer and execute the wrapped operation. IF the operation completes first, THE SYSTEM SHALL cancel the deadline timer and return the result. IF the deadline fires first, THE SYSTEM SHALL record the timeout event and return {error, timeout} to the BEAM process. WHEN called during replay, THE SYSTEM SHALL return the recorded outcome (C18).

**Acceptance:**
- nif_timer.rs implements with_timeout_impl that takes a duration and an operation descriptor, schedules a deadline timer through AT's service, and races the operation against the deadline.
- If the operation wins, the deadline timer is cancelled and the result is returned.
- If the deadline wins, a timeout event is recorded and {error, timeout} is returned.
- On replay, the recorded outcome (either the operation result or the timeout) is returned without scheduling or executing.
- A test verifies the operation-wins path records the correct events; a second test verifies the timeout-wins path.

## Boundaries

- SHALL NOT reimplement AT's timer scheduling, wheel management, or durable timer recovery — those are consumed via the existing time module API (D4).
- SHALL NOT bypass the Recorder to call EventStore::append directly (CO2).
- SHALL NOT bypass the Resolver to skip replay checks (CO1).
- SHALL NOT import beamr outside the runtime module (CO8).
- SHALL NOT add wall-clock reads in timer NIFs — durations come from the workflow; timestamps come from recorded events.
