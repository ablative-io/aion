# NB-003: Deterministic time and random NIFs

> **Cluster:** aion-nif-bridge
> **Depends on:** NB-001
> **Checklist:**
> - C10 — now/0 returns the recorded_at timestamp of the last event in the workflow's history.
> - C11 — random/0 returns a deterministic float seeded from WorkflowId + RunId + sequence position.
> - C12 — random_int/2 returns a deterministic integer in the given range seeded from WorkflowId + RunId + sequence position.
> - C13 — now, random, and random_int produce identical values on replay as on first execution.
> **Stories:**
> - S2 (Workflow Author, Writing durable workflows in Gleam using the aion_flow SDK) — As a workflow author, I want workflow.now to return a deterministic timestamp so that my workflow logic produces the same decisions on replay as on first execution.
> - S3 (Workflow Author, Writing durable workflows in Gleam using the aion_flow SDK) — As a workflow author, I want workflow.random to return a deterministic value so that randomised branching in my workflow replays correctly.

## Purpose

Implement the deterministic now/0, random/0, and random_int/2 NIFs that workflow code uses for time-dependent and randomised branching. These are pure functions that read from recorded history and deterministic seeds (D3) — they never touch a wall clock or OS entropy source. On replay, they produce identical values to first execution, which is the core determinism guarantee (CO3).

## Task

Create nif_determinism.rs with production NIF implementations for now/0, random/0, and random_int/2. now/0 returns the recorded_at timestamp of the last event in the workflow's history via the DeterminismContext (or Resolver cursor). random/0 returns a deterministic float seeded from WorkflowId + RunId + the current command sequence position. random_int/2 returns a deterministic integer in [min, max] using the same seed. Replace the stubs in engine_nifs.rs with calls to these implementations.

## Requirements

### R1: now/0 returns last recorded event timestamp

WHEN now/0 is called, THE SYSTEM SHALL return the recorded_at timestamp of the last event consumed by the Resolver's history cursor (D3, CO3). THE SYSTEM SHALL NOT read the wall clock.

**Acceptance:**
- nif_determinism.rs implements now_impl that constructs a NifContext and reads the last consumed event's recorded_at timestamp from the DeterminismContext or Resolver cursor.
- The returned value is an integer (milliseconds since epoch) or an ISO-8601 binary — matching the aion_flow SDK's expected term format.
- A test pre-populates a history with events at known timestamps, constructs a NifContext, calls now_impl, and asserts the returned timestamp matches the last event's recorded_at.
- No std::time::SystemTime, chrono::Utc::now(), or other wall-clock call appears in the now path.

### R2: random/0 returns deterministic float

WHEN random/0 is called, THE SYSTEM SHALL return a deterministic f64 in [0.0, 1.0) seeded from WorkflowId + RunId + the current sequence position (D3, CO3). THE SYSTEM SHALL NOT use any OS entropy source.

**Acceptance:**
- nif_determinism.rs implements random_impl that derives a seed from workflow_id bytes + run_id bytes + the sequence position counter, hashes or feeds them into a deterministic PRNG, and returns an f64 in [0.0, 1.0).
- The NIF returns the float as a BEAM float term.
- A test calls random_impl twice with the same workflow_id, run_id, and sequence position and asserts the same value; a test with a different sequence position asserts a different value.
- No rand::thread_rng(), OsRng, or getrandom call appears in the random path.

### R3: random_int/2 returns deterministic integer in range

WHEN random_int/2 is called with min and max arguments, THE SYSTEM SHALL return a deterministic integer in [min, max] seeded from WorkflowId + RunId + the current sequence position (D3, CO3). IF min > max, THEN THE SYSTEM SHALL return an error term.

**Acceptance:**
- nif_determinism.rs implements random_int_impl that takes min: i64 and max: i64, derives a seed identically to random/0, and maps the output to [min, max].
- The NIF returns the integer as a BEAM integer term.
- A test calls random_int_impl with min=1, max=100 twice at the same position and asserts the same value; a test with min > max asserts an error term.
- The mapping from seed to integer range is uniform and deterministic.

### R4: Replay produces identical values

WHEN now/0, random/0, or random_int/2 are called during replay, THE SYSTEM SHALL produce the same values as the original execution (C13, CO3). THE SYSTEM SHALL NOT introduce any non-deterministic input that could vary between executions.

**Acceptance:**
- A test runs a sequence of now, random, and random_int calls against a fresh history, records the returned values, then replays the same sequence over the recorded history and asserts all values match.
- The seed derivation uses only workflow_id, run_id, and sequence position — no wall clock, no thread id, no random salt.
- The same DeterminismContext (or cursor) advances consistently during both first execution and replay.

## Boundaries

- SHALL NOT read the wall clock or any OS entropy source from any code path in this brief (CO3).
- SHALL NOT record any events — these are pure read-only functions, not command-issued side effects.
- SHALL NOT depend on any AT or AD service — these NIFs read from the Resolver's history cursor and deterministic seeds only (D3).
- SHALL NOT change the Resolver, HistoryCursor, or DeterminismContext types — consume them as-is.
