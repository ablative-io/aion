# NB-007: Child workflow NIFs (spawn_child, await_child)

> **Cluster:** aion-nif-bridge
> **Depends on:** NB-001
> **Checklist:**

> - C26 — await_child/1 NIF blocks until the child completes and records ChildWorkflowCompleted or ChildWorkflowFailed.
> - C27 — Child NIFs on replay return recorded results without spawning real child processes.
> **Stories:**
> - S7 (Workflow Author, Writing durable workflows in Gleam using the aion_flow SDK) — As a workflow author, I want to spawn child workflows and await their results so that I can compose complex workflows from simpler building blocks.

## Purpose

Implement the child workflow NIFs that workflow code calls to spawn subordinate workflows and await their results. Each NIF goes through the durability handoff: on replay, recorded child workflow events are returned without spawning real child processes; on live execution, the NIF delegates to AT's child workflow spawning service (D4) and records durable events through the Recorder. Child workflows enable hierarchical composition of complex workflows from simpler building blocks.

## Task

Create nif_child.rs with production NIF implementations for spawn_child/3 and await_child/1. Each NIF constructs a NifContext (NB-001), builds the appropriate Command, calls resolve_or_execute_live. On replay, return the recorded result. On live, delegate to AT's child workflow spawning service (crate::child::spawn, spawn_and_wait), record events through the Recorder (CO2), and return the result. Replace the stubs in engine_nifs.rs.

## Requirements

### R1: spawn_child/3 starts a child workflow and records ChildWorkflowStarted

WHEN spawn_child/3 is called on the live path, THE SYSTEM SHALL delegate to AT's child workflow spawn service, record ChildWorkflowStarted through the Recorder (CO2), and return the child workflow identifier to the BEAM process. WHEN called during replay, THE SYSTEM SHALL return the recorded child workflow identifier without spawning a real child process (C27).

**Acceptance:**
- nif_child.rs implements spawn_child_impl that takes a workflow type, input payload, and options, constructs a NifContext, resolves the spawn-child Command, and on ResumeLive delegates to crate::child::spawn.
- On the live path, ChildWorkflowStarted is recorded with the child's WorkflowId, workflow type, and parent WorkflowId.
- On the replay path, the recorded child WorkflowId is returned immediately without spawning.
- A test runs the live path with a mock engine seam and asserts ChildWorkflowStarted is recorded; a replay test asserts the recorded child ID is returned without spawning.

### R2: await_child/1 blocks until child completes and records terminal event

WHEN await_child/1 is called on the live path, THE SYSTEM SHALL block the dirty NIF thread until the child workflow completes or fails, record ChildWorkflowCompleted or ChildWorkflowFailed through the Recorder (CO2), and return the result or error to the BEAM process. WHEN called during replay, THE SYSTEM SHALL return the recorded outcome without waiting for a real child (C27).

**Acceptance:**
- nif_child.rs implements await_child_impl that takes a child WorkflowId, constructs a NifContext, resolves the await-child Command, and on ResumeLive delegates to crate::child::spawn_and_wait (or the await path of the child service).
- On the live path, the NIF blocks via Handle::block_on until the child's completion notifier resolves, then records ChildWorkflowCompleted (success) or ChildWorkflowFailed (failure).
- On success the NIF returns {ok, result_payload}; on failure {error, reason}.
- On the replay path, the recorded outcome is returned immediately.
- A test runs spawn + await on the live path and asserts both ChildWorkflowStarted and ChildWorkflowCompleted appear in the store.

## Boundaries

- SHALL NOT reimplement child workflow spawning, supervision placement, or linked exit propagation — those are AT's child service (D4).
- SHALL NOT bypass the Recorder to call EventStore::append directly (CO2).
- SHALL NOT bypass the Resolver to skip replay checks (CO1).
- SHALL NOT import beamr outside the runtime module (CO8).
- SHALL NOT handle child workflow retry or restart — that is AT's machinery.
