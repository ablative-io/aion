# AX-004: Wire WorkflowDefinition typed entry contract

**Cluster:** AX (Aion Hardening)  
**Priority:** Medium  
**Owner:** Charge  
**Depends on:** Gleam decoder pipeline fix (beamr closure issue)  
**Files:** `crates/aion/src/lifecycle/start.rs`, `gleam/aion_flow/src/aion/workflow/define.gleam`

## Problem

The Gleam SDK defines a `WorkflowDefinition` type that carries input/output/error codecs and the typed entry function. The engine is supposed to call the manifest entry function to get the definition, then use the codecs to decode input and encode output. Currently the engine just calls `run/1` directly with a raw binary, bypassing the typed contract.

## Requirements

- R1: Engine calls the manifest entry function which returns a WorkflowDefinition.
- R2: Engine extracts the input codec from the definition and uses it to decode the durable Payload into a typed Gleam value.
- R3: Engine calls the definition's entry_fn with the decoded input.
- R4: Engine encodes the result using the output/error codec from the definition.
- R5: This requires the Gleam decoder pipeline to work — closures must execute correctly through call_fun2.

## Acceptance Criteria

- [ ] hello-world workflow uses `pub fn run(input: GreetingInput)` (typed input, not raw String)
- [ ] Engine extracts and uses codecs from WorkflowDefinition
- [ ] Input/output serialization goes through the SDK's codec boundary
- [ ] clippy clean, fmt clean, all tests pass
