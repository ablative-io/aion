# AX-004: Wire WorkflowDefinition typed entry contract through engine

> **Cluster:** aion-hardening
> **Depends on:** AX-001
> **Blocked by:** beamr-decoder-closure-fix

## Purpose

The Gleam SDK defines a WorkflowDefinition type carrying input/output/error codecs and the typed entry function. The engine currently calls run/1 directly with a raw binary, bypassing the typed contract. This brief wires the proper entry contract so input is decoded and output is encoded through the SDK's codecs.

## Task

Modify the engine's start_workflow path to call the manifest entry function (which returns a WorkflowDefinition), extract the input codec, decode the durable Payload into a typed Gleam value, call the definition's entry_fn, and encode the result. Requires the Gleam decoder pipeline (closures via call_fun2) to work correctly — blocked until the beamr nested closure issue is resolved.

## Requirements

### R1: Engine consumes WorkflowDefinition

THE SYSTEM SHALL call the manifest entry function to obtain a WorkflowDefinition value. THE SYSTEM SHALL extract the input_codec and use it to decode the spawn Payload before calling the typed entry function.

**Acceptance:**
- The engine calls the entry function and receives a WorkflowDefinition (Gleam opaque type — BEAM tuple).
- The input codec is extracted and used to decode the Payload bytes.
- The typed entry function is called with the decoded input value.

**Files:**
- modify: crates/aion/src/lifecycle/start.rs
- modify: crates/aion/src/runtime/payload.rs

### R2: Hello-world uses typed entry

THE SYSTEM SHALL restore the hello-world workflow to use pub fn run(input: GreetingInput) with the typed GreetingInput record, not a raw String.

**Acceptance:**
- hello_world.gleam run function takes GreetingInput, not String.
- definition() returns a WorkflowDefinition with proper codecs.
- The full E2E path works: JSON input → codec decode → typed function → activity dispatch → worker → result → codec encode.

**Files:**
- modify: examples/hello-world/src/hello_world.gleam

## Boundaries

- SHALL NOT implement until the beamr decoder pipeline closure issue is resolved (nested call_fun2 closures in gleam@dynamic@decode).
- SHALL NOT change the WorkflowDefinition type in the Gleam SDK — the engine adapts to the existing contract.
