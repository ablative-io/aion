# AX-003: Remove diagnostic API from production code

**Cluster:** AX (Aion Hardening)  
**Priority:** Medium  
**Owner:** Charge  
**Files:** `crates/aion/src/runtime/handle.rs`, `crates/aion/src/lifecycle/start.rs`

## Problem

`run_until_exit_for_diagnostic` is a public method on `RuntimeHandle` that blocks the calling thread until a beamr process exits. It was added during E2E debugging and should not be in production code — it blocks HTTP handler threads and has no timeout.

## Requirements

- R1: Remove `run_until_exit_for_diagnostic` from the public API.
- R2: If diagnostic capability is needed, gate it behind `#[cfg(test)]` or a `diagnostics` feature flag.
- R3: `start.rs` must use `spawn_workflow_with_policy` (with trap_exit), not `spawn_workflow` followed by `run_until_exit`.
- R4: The `registered_nif_modules()` method should derive its list from the actual NIF registration entries, not a hardcoded vec.

## Acceptance Criteria

- [ ] `run_until_exit_for_diagnostic` not callable from non-test code
- [ ] `registered_nif_modules` returns entries from the NIF registration, not hardcoded
- [ ] start.rs uses spawn_workflow_with_policy as designed
- [ ] clippy clean, fmt clean, all tests pass
