# AX-002: Add tracing subscriber to aion-server

**Cluster:** AX (Aion Hardening)  
**Priority:** High  
**Owner:** Max Power  
**Files:** `crates/aion-server/src/main.rs`

## Problem

`aion-server` has no `tracing_subscriber` initialized. All `tracing::debug!`, `tracing::warn!`, `tracing::error!` calls across the engine and server are silently dropped. This made E2E debugging extremely difficult — errors were invisible.

## Requirements

- R1: Initialize `tracing_subscriber` in main.rs before any other initialization.
- R2: Default to `INFO` level, configurable via `RUST_LOG` env var.
- R3: Use `tracing_subscriber::fmt` with timestamps and module paths.
- R4: No new dependencies beyond `tracing-subscriber` (already in the workspace).

## Acceptance Criteria

- [ ] Server startup prints version/config info at INFO level
- [ ] Engine warnings (unresolved imports, module registration) visible in output
- [ ] `RUST_LOG=debug` shows detailed engine/scheduler activity
- [ ] clippy clean, fmt clean, all tests pass
