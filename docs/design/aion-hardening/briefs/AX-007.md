# AX-007: Schedule coordinator lifecycle event for uniform recovery

> **Cluster:** aion-hardening
> **Depends on:** AX-006

## Purpose

The schedule coordinator workflow has no WorkflowStarted event in the store, requiring a special-case skip in repopulate_active_workflows and a separate catchup_schedule_coordinator method. This is fragile — any code path that iterates active workflows must know about the exception. Give the coordinator a proper WorkflowStarted event so recovery handles it uniformly with all other workflows.

## Task

Record a WorkflowStarted event for the schedule coordinator during EngineBuilder::build() when the coordinator has no existing history. On subsequent builds against the same store, the coordinator already has history and the Recorder catches up normally (via catchup_schedule_coordinator). Remove the special-case skip in repopulate_active_workflows and ensure the coordinator is recovered through the standard active-workflow path.

## Requirements

### R1: Record coordinator WorkflowStarted on first engine build

WHEN EngineBuilder::build() creates a schedule coordinator against a store with no coordinator history, THE SYSTEM SHALL record a WorkflowStarted event for the coordinator workflow_id with workflow_type 'aion.schedule_coordinator' and empty input. WHEN the store already has coordinator history, THE SYSTEM SHALL skip the initial event and use catchup_schedule_coordinator to resume at the correct head.

**Acceptance:**
- First engine build against an empty store records one WorkflowStarted event for the coordinator.
- Second engine build against the same store does not record a duplicate WorkflowStarted.
- The coordinator's workflow_type is 'aion.schedule_coordinator'.
- The coordinator appears in store.list_active() after the first build.

**Files:**
- modify: crates/aion/src/engine/builder.rs
- modify: crates/aion/src/engine/api.rs

### R2: Remove special-case skip in repopulate_active_workflows

THE SYSTEM SHALL remove the schedule_coordinator skip in repopulate_active_workflows. The coordinator SHALL be recovered through the standard ActiveWorkflowRecoverySeam like any other workflow. THE SYSTEM SHALL provide a coordinator-specific recovery implementation that re-arms the schedule evaluator rather than spawning a beamr process.

**Acceptance:**
- repopulate_active_workflows no longer contains a skip for schedule_coordinator_workflow_id().
- The coordinator is recovered via the standard recovery seam path.
- After recovery, schedule_evaluator is re-armed and can fire timers.
- The DeferredActiveWorkflowRecovery (used in tests without schedule infrastructure) handles the coordinator gracefully (skips or provides a no-op recovery).

**Files:**
- modify: crates/aion/src/engine/builder.rs
- modify: crates/aion/src/durability/recovery.rs

### R3: Schedule lifecycle test passes without special-case handling

THE SYSTEM SHALL verify that the schedule lifecycle integration test continues to pass with the coordinator recovered through the standard path. The TestRecovery seam from AX-006 SHALL handle the coordinator as an active workflow without special casing.

**Acceptance:**
- cargo test -p aion --test schedule_lifecycle passes.
- The TestRecovery struct does not special-case the coordinator workflow_id.
- After engine restart, list_schedules returns the correct schedule state.
- cargo clippy --workspace --all-targets -- -D warnings passes.
- cargo test --workspace passes.

**Files:**
- modify: crates/aion/tests/schedule_lifecycle.rs

## Boundaries

- SHALL NOT change the schedule evaluator or coordinator's runtime behaviour — only its initialization and recovery path.
- SHALL NOT remove catchup_schedule_coordinator — it is still needed for the case where the coordinator has existing history.
- SHALL NOT spawn a beamr process for the coordinator — it is a virtual workflow that only holds schedule events.
- SHALL NOT change the coordinator's deterministic workflow_id.

## Verification

- cargo clippy --workspace --all-targets -- -D warnings passes.
- cargo fmt --check passes.
- cargo test --workspace passes.
- The schedule_coordinator skip no longer exists in builder.rs.
- store.list_active() includes the coordinator after first engine build.
