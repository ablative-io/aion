# AX-001: Fix payload_to_term memory leak

**Cluster:** AX (Aion Hardening)  
**Priority:** High  
**Owner:** Charge  
**Files:** `crates/aion/src/runtime/payload.rs`

## Problem

`payload_to_term` uses `Box::leak(heap)` to keep binary term memory alive after the function returns. This leaks memory — every workflow start leaks `2 + packed_word_count(input_len)` words.

## Requirements

- R1: `payload_to_term` must not leak memory.
- R2: The binary term must remain valid until the beamr process copies it onto its own heap during spawn.
- R3: The solution must work with beamr's `scheduler.spawn()` which takes `Vec<Term>` as arguments.

## Approach

Either:
1. Use beamr's spawn argument copying to handle ownership (check if spawn deep-copies boxed terms).
2. Park the heap on a thread-local like `engine_nifs.rs` does with `NIF_HEAP`, and drain after spawn.
3. Ask Bono for a spawn API that accepts owned binary data directly.

## Acceptance Criteria

- [ ] No memory leak on repeated workflow starts (verify with valgrind or heap profiling)
- [ ] Existing E2E test still works (NIF fires, worker receives task)
- [ ] clippy clean, fmt clean, all tests pass
