# AF-002: Codec and Duration: the typed boundary primitives

> **Cluster:** aion-flow
> **Depends on:** AF-001
> **Checklist:**
> - C6 — Codec(a) is defined as a paired encoder fn(a) -> String and decoder fn(String) -> Result(a, DecodeError) over the ffi string form.
> - C7 — Default JSON codec helpers build a Codec(a) from a gleam_json encoder and a decoder; a decode failure yields a typed DecodeError, never a panic.
> - C8 — Duration is a typed quantity with constructors milliseconds, seconds, minutes, hours, and days, normalised to a canonical internal representation.
> - C9 — Every time-quantity argument (sleep, with_timeout, activity.timeout, start_timer, backoff config) takes a Duration, not a bare Int.
> **Stories:**
> - S6 (Workflow Author, Writing Durable Workflows in Gleam) — As a workflow author, I want to sleep for a duration, start and cancel named timers, and wrap a wait with a timeout, using one unit-safe Duration type, so that durable waiting is expressed without unit-ambiguous integers.
> - S13 (Engine, Resolving the Bindings at Runtime) — As the engine, I want user data to arrive as an encoded payload with a typed codec on the SDK side so that my type-erased event store and replay never need to know the workflow's concrete Gleam types.

## Purpose

Define the two primitives that every typed boundary depends on: Codec(a), which carries a typed Gleam value across the type-erased ffi boundary, and Duration, the single unit-safe time quantity used by every timer and timeout.

## Task

Define Codec(a) as a paired encoder/decoder over the ffi string form, with default JSON codec helpers built on gleam_json and a typed DecodeError. Define Duration as a typed quantity with named constructors normalised to a canonical internal representation, used by every time-quantity argument in the SDK.

## Requirements

### R1: Codec(a) and default JSON helpers

THE SYSTEM SHALL define Codec(a) as a pair of an encoder fn(a) -> String and a decoder fn(String) -> Result(a, DecodeError) over the ffi string form, with default JSON codec helpers and a typed DecodeError.

**Acceptance:**
- codec.gleam defines Codec(a) carrying encode: fn(a) -> String and decode: fn(String) -> Result(a, DecodeError).
- A json_codec helper constructs a Codec(a) from a gleam_json encoder and a decoder; decoding malformed input returns DecodeError, never a panic.
- DecodeError is a typed value carrying enough context (the failing field / reason) to diagnose a boundary failure.
- A unit test round-trips representative values (record, list, nested) through a JSON codec and asserts equality, and asserts a malformed string yields DecodeError.

**Files:**
- create: gleam/aion_flow/src/aion/codec.gleam

**Checklist:**

- C7 — Default JSON codec helpers build a Codec(a) from a gleam_json encoder and a decoder; a decode failure yields a typed DecodeError, never a panic.

**Stories:**
- S13 (Engine, Resolving the Bindings at Runtime) — As the engine, I want user data to arrive as an encoded payload with a typed codec on the SDK side so that my type-erased event store and replay never need to know the workflow's concrete Gleam types.

### R2: Duration type and constructors

THE SYSTEM SHALL define Duration as a typed quantity with constructors milliseconds, seconds, minutes, hours, and days, normalised to a single canonical internal representation.

**Acceptance:**
- duration.gleam defines an opaque Duration with constructors milliseconds, seconds, minutes, hours, and days.
- All constructors normalise to one canonical internal representation (milliseconds) so the engine receives a single form; a to_milliseconds accessor exposes it for the ffi boundary.
- A unit test asserts that equivalent durations (e.g. minutes(1) and seconds(60)) compare equal and convert to the same canonical value.

**Files:**
- create: gleam/aion_flow/src/aion/duration.gleam

**Checklist:**
- C8 — Duration is a typed quantity with constructors milliseconds, seconds, minutes, hours, and days, normalised to a canonical internal representation.
- C9 — Every time-quantity argument (sleep, with_timeout, activity.timeout, start_timer, backoff config) takes a Duration, not a bare Int.

**Stories:**
- S6 (Workflow Author, Writing Durable Workflows in Gleam) — As a workflow author, I want to sleep for a duration, start and cancel named timers, and wrap a wait with a timeout, using one unit-safe Duration type, so that durable waiting is expressed without unit-ambiguous integers.

## Boundaries

- SHALL NOT define ActivityError or any workflow primitive — those are later briefs; this brief defines only Codec, DecodeError, and Duration.
- SHALL NOT declare an @external — encoding/decoding is pure Gleam; the ffi string form it produces is consumed by later briefs' typed wrappers.
- SHALL NOT bake in any default duration value.
