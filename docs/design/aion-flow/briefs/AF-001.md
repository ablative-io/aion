## Purpose

Stand up the aion_flow Gleam package and the single internal module that owns every @external(erlang) binding to the engine NIFs. This is the foundation every other brief builds on: a pure-Gleam, Hex-publishable package with one auditable binding seam.

## Task

Create the gleam/aion_flow package with a gleam.toml declaring gleam_stdlib and gleam_json as the only runtime dependencies. Create the top-level aion_flow.gleam package-doc / re-export module. Create aion/internal/ffi.gleam holding every @external(erlang, "aion_flow_ffi", "<fn>") declaration the SDK needs, each with the type-erased string/Result signature the engine NIFs expose. No other module in the package declares an @external. The package must build, check, and format clean with no engine present.

## Requirements

### R1: Package scaffold

THE SYSTEM SHALL create a valid Gleam package at gleam/aion_flow whose gleam.toml declares the package name, version, and Hex metadata with gleam_stdlib and gleam_json as the only runtime dependencies.

**Acceptance:**
- gleam/aion_flow/gleam.toml declares name = "aion_flow", a version, a description, and a licences/repository field set for Hex publication.
- Dependencies are exactly gleam_stdlib and gleam_json (plus dev-dependency gleeunit for tests); no Rust, beamr, or engine dependency appears.
- gleam build, gleam check, and gleam format --check pass clean for the package.
- The package has no Rust source and no build-script dependency on the engine crates.

**Files:**
- create: gleam/aion_flow/gleam.toml
- create: gleam/aion_flow/.gitignore

**Checklist:**

### R2: The @external binding layer

THE SYSTEM SHALL define aion/internal/ffi as the single module declaring every @external(erlang, "aion_flow_ffi", "<fn>") binding to the engine NIFs. No other module SHALL declare an @external.

**Acceptance:**
- aion/internal/ffi.gleam declares the @external bindings the SDK needs (activity dispatch, now, random, sleep/start_timer/cancel_timer, signal receive/send, query register/reply, child spawn/await, concurrency collectors), each using the Erlang module name "aion_flow_ffi".
- Each binding's Gleam signature uses the type-erased ffi form (String payloads, Result(String, String) returns) — the typed wrappers live in the author-facing modules.
- A doc comment states that "aion_flow_ffi" is the engine's NIF registry namespace registered via EngineBuilder::register_nifs (AE-004), resolved by beamr at runtime, and that gleam build type-checks the signatures with no engine present.
- No author-facing module anywhere in the package declares an @external (verified by inspection / a grep-style check in review).

**Files:**
- create: gleam/aion_flow/src/aion/internal/ffi.gleam

**Checklist:**
- C3 — Every @external(erlang, ...) declaration lives in the single internal module aion/internal/ffi; no author-facing module declares an @external.
- C4 — All ffi bindings use the Erlang module name "aion_flow_ffi" (the engine's NIF registry namespace registered via AE-004).

**Stories:**
- S12 (Engine, Resolving the Bindings at Runtime) — As the engine, I want every SDK primitive to bind to a NIF in one known registry namespace (aion_flow_ffi) so that I can register the native implementations in one place and have compiled workflows resolve against them.

- It re-exports (or documents the import paths for) the author-facing modules; it contains no @external declaration and no logic beyond re-exports.

## Boundaries

- SHALL NOT implement any engine NIF — the native side is AE/AN. This brief declares the Gleam @external signatures only.
- SHALL NOT define the typed author-facing wrappers (activity, workflow, signal, etc.) — those are later briefs; this brief only declares the raw ffi bindings they will call.
- SHALL NOT add a Rust crate or beamr dependency to the package.
