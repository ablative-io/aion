# AF-005: Workflow core: run, now, random, and the typed entry contract

> **Cluster:** aion-flow
> **Depends on:** AF-004
> **Checklist:**
> - C17 — workflow.run(Activity(i, o)) -> Result(o, ActivityError) is the single recorded activity dispatch, binding to the engine via ffi and decoding the typed result.
> - C18 — workflow.now() -> Timestamp binds to AD's determinism context (recorded event timestamp); the SDK exposes no wall-clock binding.
> - C19 — workflow.random() -> Float and workflow.random_int bind to AD's seeded RNG; the SDK exposes no entropy-source binding.
> - C20 — There is no generic side_effect(fn) escape hatch; the only side-effect dispatch is workflow.run (and all/race/map over activities).
> - C21 — workflow.define(name, input_codec, output_codec, error_codec, entry_fn) registers a typed entry contract consumable by the .aion manifest (AP).
> **Stories:**
> - S2 (Workflow Author, Writing Durable Workflows in Gleam) — As a workflow author, I want a single recorded way to invoke a side effect (workflow.run over an activity) and plain Gleam everywhere else, so that the recorded/deterministic boundary is obvious from the code's shape.
> - S3 (Workflow Author, Writing Durable Workflows in Gleam) — As a workflow author, I want workflow.now and workflow.random to be deterministic and to have no wall-clock alternative available, so that my workflow cannot accidentally become non-deterministic and desynchronise replay.
> - S13 (Engine, Resolving the Bindings at Runtime) — As the engine, I want user data to arrive as an encoded payload with a typed codec on the SDK side so that my type-erased event store and replay never need to know the workflow's concrete Gleam types.

## Purpose

Define the heart of the author surface: workflow.run (the single recorded activity dispatch), the determinism primitives now/random bound to AD, and workflow.define (the typed entry contract the .aion manifest consumes). Establish that there is no side-effect escape hatch other than run.

## Task

Create the aion/workflow module (declarations + re-exports) and the workflow/run.gleam and workflow/define.gleam submodules. Implement workflow.run as the single recorded activity dispatch via the ffi binding, encoding the input and decoding the typed output or a typed ActivityError. Bind workflow.now and workflow.random to AD's determinism context via ffi, with no wall-clock or entropy alternative. Define workflow.define for the typed entry contract.

## Requirements

### R1: workflow.run — the single recorded dispatch

THE SYSTEM SHALL define workflow.run(Activity(i, o)) -> Result(o, ActivityError) as the single recorded activity dispatch, binding to the engine via the ffi layer, encoding the activity input and decoding the typed output, with no generic side-effect escape hatch.

**Acceptance:**
- workflow/run.gleam defines run(Activity(i, o)) -> Result(o, ActivityError); it encodes the input via the activity's Codec, calls the ffi activity-dispatch binding (which records via AD and returns from history on replay), and decodes the result with the output Codec.
- A decode failure on the returned payload surfaces as a typed error (the engine-originated decode variant from AF-003), never a panic.
- No workflow.side_effect(fn) or equivalent is provided; a doc comment states that the only recorded side-effect path is run (and the concurrency combinators over activities) per CO7.
- A harness-backed test (using AF-011's mock once available, or a stub binding) asserts run returns the decoded typed result on success and the typed ActivityError on failure.

### R2: Determinism: now and random

THE SYSTEM SHALL define workflow.now() -> Timestamp and workflow.random() -> Float (plus random_int) bound to AD's determinism context via the ffi layer, exposing NO wall-clock or entropy-source binding.

**Acceptance:**
- workflow.now() returns the recorded event timestamp from AD's determinism context (via the ffi binding), documented as the only time source available to workflow code.
- workflow.random() -> Float and workflow.random_int(min, max) -> Int draw from AD's seeded RNG (keyed on WorkflowId + RunId), via the ffi binding.
- The SDK exposes no binding to a wall clock (gleam stdlib / erlang:now) or entropy source; a doc comment states the prohibition on calling them from workflow code (CO6).
- A test asserts now and random return values via the binding (using the harness's deterministic stub) and that repeated runs under the same seed produce the same random sequence.

### R3: workflow.define — typed entry contract

THE SYSTEM SHALL define workflow.define(name, input_codec, output_codec, error_codec, entry_fn) registering a typed entry contract that the .aion manifest (AP) consumes, carrying the entry function and the codecs for the engine's type-erased boundary.

**Acceptance:**
- workflow/define.gleam defines define taking the workflow name, input/output/error Codecs, and the entry function fn(i) -> Result(o, e), producing a typed definition value.
- A doc comment states the definition is consumed by aion-package (AP) to record the entry module/function and codecs; the SDK does not invoke the function itself (AE spawns the process).
- The definition carries enough to bridge the typed entry to the type-erased ffi boundary (encode input on entry, decode output/error on completion).
- A test constructs a define for a concrete typed workflow function and asserts the codecs and name are carried.

## Boundaries

- SHALL NOT implement the engine-side activity dispatch, recording, replay, or determinism context — those are AE/AD. This brief binds to them via ffi.
- SHALL NOT implement timers, signals, queries, children, or concurrency — those are AF-006 through AF-010.
- SHALL NOT build the .aion archive — that is AP; this brief provides the typed entry contract it consumes.
