# AF-004: Activities: typed Activity values, decorators, RetryPolicy and Backoff

> **Cluster:** aion-flow
> **Depends on:** AF-003
> **Checklist:**
> - C13 — activity.new(name, input, run) builds an opaque Activity(i, o) carrying the name, typed input, output Codec, and runner fn(i) -> Result(o, ActivityError).
> - C14 — activity.retry, activity.timeout, and activity.heartbeat each return a new Activity(i, o), composing in a pipeline.
> - C15 — RetryPolicy carries max_attempts and a Backoff; no default RetryPolicy is baked in — an undecorated activity runs once.
> - C16 — Backoff is a sum type with Exponential, Linear, and Fixed variants carrying their typed parameters.
> **Stories:**
> - S5 (Workflow Author, Writing Durable Workflows in Gleam) — As a workflow author, I want to compose retry, timeout, and heartbeat onto an activity in a readable pipeline so that the activity's policy reads as data next to its definition.

## Purpose

Make activities typed values. activity.new builds an Activity(input, output) whose type parameters flow into workflow.run; the retry/timeout/heartbeat decorators attach typed policy data in a composable pipeline; RetryPolicy and Backoff are typed config with no baked-in defaults.

## Task

Define aion/activity: an opaque Activity(i, o) carrying name, typed input, output Codec, runner, and policy config. Define activity.new and the retry/timeout/heartbeat decorators (each returning a new Activity(i, o)). Define RetryPolicy and the Backoff sum type (Exponential | Linear | Fixed). Bake in no default policy — an undecorated activity runs once.

## Requirements

### R1: Activity(i, o) and activity.new

THE SYSTEM SHALL define an opaque Activity(i, o) built by activity.new(name, input, run) carrying the name, typed input, an output Codec, and the runner fn(i) -> Result(o, ActivityError).

**Acceptance:**
- activity.gleam defines an opaque Activity(i, o) type parameterised by input and output.
- activity.new(name: String, input: i, run: fn(i) -> Result(o, ActivityError)) constructs an Activity(i, o); the output Codec is supplied (as an argument or via a labelled field) so the result can be decoded at the boundary.
- The type parameters i and o are carried such that workflow.run(Activity(i, o)) can return Result(o, ActivityError) (verified once AF-005 lands; this brief establishes the shape).
- A unit test constructs an Activity with concrete input/output types and reads back its name.

### R2: retry / timeout / heartbeat decorators

THE SYSTEM SHALL define activity.retry, activity.timeout, and activity.heartbeat, each taking and returning an Activity(i, o) so they compose in a pipeline.

**Acceptance:**
- activity.retry(Activity(i, o), RetryPolicy) -> Activity(i, o) attaches the retry policy.
- activity.timeout(Activity(i, o), Duration) -> Activity(i, o) attaches the timeout.
- activity.heartbeat(Activity(i, o), Duration) -> Activity(i, o) attaches the heartbeat interval for long-running activities.
- Each decorator returns a new Activity(i, o) preserving the type parameters; a unit test composes new |> retry |> timeout |> heartbeat and asserts the config is carried.

### R3: RetryPolicy and Backoff

THE SYSTEM SHALL define RetryPolicy (max_attempts + Backoff) and Backoff as a sum type with Exponential, Linear, and Fixed variants, with no default policy baked into the SDK.

**Acceptance:**
- RetryPolicy carries max_attempts and a Backoff (and any further documented fields, e.g. non-retryable-error filter, expressed in terms of AF-003's classification).
- Backoff is Exponential(initial: Duration, multiplier, max: Duration), Linear(initial: Duration, increment: Duration, max: Duration), Fixed(delay: Duration).
- No constructor supplies a default RetryPolicy, default timeout, or default backoff numbers; a doc comment states that an activity with no retry decorator runs exactly once (CO8 / no-assumed-defaults).
- A unit test constructs each Backoff variant and a RetryPolicy and asserts the values are carried.

## Boundaries

- SHALL NOT bake in any default retry/timeout/backoff value (CO8).
- SHALL NOT declare an @external — the Activity value is pure Gleam data; the ffi dispatch is AF-005.
