# AF-008: Queries: typed read-only handler registration and reply

> **Cluster:** aion-flow
> **Depends on:** AF-007
> **Checklist:**
> - C28 — query.handler(name, Codec(a), fn() -> a) registers a read-only handler whose return type a is fixed at registration; the reply is encoded by the SDK.
> - C29 — A query records no event (binds to AT's read-only query service); an unknown query name yields a typed QueryError.
> **Stories:**
> - S7 (Workflow Author, Writing Durable Workflows in Gleam) — As a workflow author, I want to wait for a typed signal and register typed query handlers so that external interaction and inspection are as type-safe as the rest of the workflow.

## Purpose

Let a workflow expose typed, read-only inspection of its state. A query handler registered for type a returns a; the engine's query service calls it and the SDK encodes the reply. No event is recorded for a query.

## Task

Implement aion/query: query.handler(name, Codec(a), fn() -> a) registering a typed read-only handler bound to AT's query service via the ffi layer, with the reply encoded by the SDK and unknown-query handling returning a typed QueryError. A query records no event.

## Requirements

### R1: Typed query handler registration

THE SYSTEM SHALL define query.handler(name, Codec(a), fn() -> a) registering a read-only handler whose return type a is fixed at registration, with the reply encoded by the SDK.

**Acceptance:**
- query.gleam defines handler(name: String, Codec(a), reply: fn() -> a) registering the handler with AT's query service via the ffi layer; the return type a is fixed at registration.
- When the engine dispatches a query, the SDK invokes the handler, encodes the returned value with the Codec, and replies on the engine's reply channel.
- By type and by documented convention the handler returns a value and does not run an activity (it is read-only).
- A harness test registers a handler returning a typed state value and asserts a dispatched query returns the decoded value.

**Files:**
- create: gleam/aion_flow/src/aion/query.gleam
- modify: gleam/aion_flow/src/aion/internal/ffi.gleam

### R2: No-record guarantee and unknown-query handling

THE SYSTEM SHALL ensure a query records no event (binding to AT's read-only query service) and that an unknown query name yields a typed QueryError.

**Acceptance:**
- A doc comment states that a query records no Event (AT D6/CO7) and never blocks workflow progress.
- Dispatching an unregistered query name returns a typed QueryError (unknown query), not a panic and not a fabricated answer.
- A harness test asserts: a query produces no recorded observation in the env's capture, and an unknown query returns the typed QueryError.

## Boundaries

- SHALL NOT implement the query service, dispatch, timeout, or reply channel — those are AT-007. This brief binds to them.
- SHALL NOT allow a query handler to record an event or run an activity.
- SHALL NOT let a query return value cross the boundary without a Codec (CO9).
