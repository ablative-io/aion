# AK-001: Schedule core types, policies, and event family in aion-core

> **Cluster:** aion-schedules
> **Checklist:** C1, C2, C3, C4
> **Stories:** S5, S7

## Purpose

Define the core domain types for scheduling: ScheduleId, trigger specifications (cron expression or fixed interval), overlap policy, catch-up policy, and the schedule event family. These types are the foundation every other AK brief builds on. All types live in aion-core so they are available to the engine, store, and server crates.

## Task

Add schedule types to aion-core: ScheduleId newtype (UUID-based, matching WorkflowId/RunId pattern), TriggerSpec enum (Cron with expression string, Interval with Duration), OverlapPolicy enum (Skip, BufferOne, CancelPrevious, AllowAll), CatchUpPolicy enum (All, One, Skip), and ScheduleConfig struct bundling them with target workflow type and input Payload. Add the schedule event family: ScheduleCreated, ScheduleUpdated, SchedulePaused, ScheduleResumed, ScheduleDeleted, ScheduleTriggered. Each event carries an EventEnvelope and schedule-specific fields.

## Requirements

### R1: ScheduleId newtype

THE SYSTEM SHALL define ScheduleId as a UUID-based newtype in aion-core, following the same pattern as WorkflowId and RunId (Display, FromStr, serde, Clone, Eq, Hash).

**Acceptance:**
- ScheduleId exists in aion-core with the same derives and trait implementations as WorkflowId.
- ScheduleId::new_v4() generates a random UUID.
- Round-trip serde serialization works correctly.

**Files:**
- create: crates/aion-core/src/schedule.rs
- modify: crates/aion-core/src/lib.rs

**Checklist:**

### R2: TriggerSpec, OverlapPolicy, and CatchUpPolicy enums

THE SYSTEM SHALL define TriggerSpec as an enum with variants Cron { expression: String } and Interval { period: Duration }. THE SYSTEM SHALL define OverlapPolicy with variants Skip, BufferOne, CancelPrevious, AllowAll. THE SYSTEM SHALL define CatchUpPolicy with variants All, One, Skip. All enums SHALL derive Debug, Clone, PartialEq, Eq, Serialize, Deserialize.

**Acceptance:**
- TriggerSpec::Cron and TriggerSpec::Interval exist with the specified fields.
- OverlapPolicy has exactly four variants: Skip, BufferOne, CancelPrevious, AllowAll.
- CatchUpPolicy has exactly three variants: All, One, Skip.
- All types round-trip through serde correctly.

**Files:**

### R3: ScheduleConfig configuration struct

THE SYSTEM SHALL define ScheduleConfig carrying: trigger (TriggerSpec), overlap_policy (OverlapPolicy), catch_up_policy (CatchUpPolicy), workflow_type (String), and input (Payload).

**Acceptance:**
- ScheduleConfig exists with all specified fields.
- Default overlap_policy is Skip; default catch_up_policy is One.
- ScheduleConfig derives Debug, Clone, Serialize, Deserialize.

**Files:**

### R4: Schedule event family

THE SYSTEM SHALL add schedule events to the Event enum: ScheduleCreated { envelope, schedule_id, config }, ScheduleUpdated { envelope, schedule_id, config }, SchedulePaused { envelope, schedule_id }, ScheduleResumed { envelope, schedule_id }, ScheduleDeleted { envelope, schedule_id }, ScheduleTriggered { envelope, schedule_id, workflow_id, run_id }. All exhaustive matches on Event SHALL handle the new variants.

**Acceptance:**
- All six schedule event variants exist on Event with the specified fields.
- Each variant carries an EventEnvelope.
- ScheduleTriggered carries the workflow_id and run_id of the triggered execution.
- All exhaustive matches in aion-core handle the new variants.
- cargo clippy -p aion-core --all-targets -- -D warnings passes.
