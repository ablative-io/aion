# AK-002: Cron expression parsing and next-fire-time calculation

> **Cluster:** aion-schedules
> **Depends on:** AK-001
> **Checklist:** C11
> **Stories:** S5

## Purpose

Integrate a cron expression parser and implement next-fire-time calculation for both cron and interval triggers. This is the time-math foundation the ScheduleEvaluator uses to determine when each schedule should fire next.

## Task

Add a cron parsing dependency (saffron or cron crate) to the aion crate. Implement a next_fire_time function that, given a TriggerSpec and a reference DateTime, returns the next DateTime the schedule should fire. For Cron, use the parser to compute the next matching time. For Interval, add the period to the reference time. Handle edge cases: invalid cron expressions return a typed error, intervals of zero duration are rejected.

## Requirements

### R1: Cron expression parsing with validated input

THE SYSTEM SHALL parse standard 5-field cron expressions (minute, hour, day-of-month, month, day-of-week) and reject invalid expressions with a typed error. Extended 6-field (with seconds) SHOULD be supported if the chosen crate supports it.

**Acceptance:**
- "0 0 * * *" (midnight daily) parses successfully.
- "*/5 * * * *" (every 5 minutes) parses successfully.
- "invalid" returns a typed parse error.
- Empty string returns a typed parse error.

### R2: Next-fire-time calculation for cron and interval triggers

THE SYSTEM SHALL implement next_fire_time(trigger: &TriggerSpec, after: DateTime<Utc>) -> Result<DateTime<Utc>, ScheduleError> that returns the next fire time strictly after the given reference time. For Cron triggers, this uses the parsed expression. For Interval triggers, this adds the period to the reference time.

**Acceptance:**
- A cron expression "0 0 * * *" with reference 2026-06-07T00:00:00Z returns 2026-06-08T00:00:00Z.
- An interval of 5 minutes with reference T returns T + 5 minutes.
- An interval of zero duration returns an error.
- Multiple successive calls return strictly increasing times.
