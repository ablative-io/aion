# AK-004: Engine schedule management API

> **Cluster:** aion-schedules
> **Depends on:** AK-001, AK-003
> **Checklist:** C8
> **Stories:** S1, S3, S6

## Purpose

Expose schedule management through the Engine public API: create, update, pause, resume, delete, list, and describe. Each mutation records the corresponding schedule event and updates the evaluator's state.

## Task

Add schedule management methods to Engine: create_schedule, update_schedule, pause_schedule, resume_schedule, delete_schedule, list_schedules, describe_schedule. Each mutation method records the corresponding event through a schedule-specific Recorder, notifies the ScheduleEvaluator, and returns the result. Query methods (list, describe) project from schedule state.

## Requirements

### R1: Engine::create_schedule

THE SYSTEM SHALL add create_schedule(config: ScheduleConfig) -> Result<ScheduleId, EngineError> that generates a ScheduleId, records ScheduleCreated, arms the evaluator, and returns the id.

**Acceptance:**
- create_schedule returns a ScheduleId.
- A ScheduleCreated event is recorded.
- The evaluator has an armed timer for the new schedule.
- The schedule appears in list_schedules.

**Files:**
- modify: crates/aion/src/engine/api.rs

**Checklist:**

**Stories:**
- S1

### R2: Engine update, pause, resume, delete

THE SYSTEM SHALL add update_schedule (changes config), pause_schedule (disarms timer), resume_schedule (re-arms timer), and delete_schedule (permanently disarms). Each records the corresponding event.

**Acceptance:**
- update_schedule records ScheduleUpdated and the evaluator uses the new config.
- pause_schedule records SchedulePaused and disarms the timer.
- resume_schedule records ScheduleResumed and re-arms the timer.
- delete_schedule records ScheduleDeleted and permanently disarms.
- Operating on a non-existent schedule returns an error.

**Files:**
- modify: crates/aion/src/engine/api.rs

**Checklist:**
- C8

**Stories:**
- S3

### R3: Engine list_schedules and describe_schedule

THE SYSTEM SHALL add list_schedules() returning all non-deleted schedules with their state, and describe_schedule(id) returning the full state of a single schedule including trigger history.

**Acceptance:**
- list_schedules returns all non-deleted schedules.
- describe_schedule returns config, is_paused, last_triggered_at, next_trigger_at.
- describe_schedule on a non-existent id returns an error.
- Deleted schedules do not appear in list_schedules.

**Files:**
- modify: crates/aion/src/engine/api.rs
