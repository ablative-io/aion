# AK-005: Server schedule endpoints and integration test

> **Cluster:** aion-schedules
> **Depends on:** AK-001, AK-003, AK-004
> **Checklist:** C9, C12
> **Stories:** S1, S6

## Purpose

Expose schedule management via gRPC and HTTP endpoints in aion-server, and provide an end-to-end integration test proving the full schedule lifecycle: create, fire on interval, overlap prevention, recovery re-arm.

## Task

Add schedule management endpoints to aion-server (gRPC and HTTP): CreateSchedule, UpdateSchedule, PauseSchedule, ResumeSchedule, DeleteSchedule, ListSchedules, DescribeSchedule. Each endpoint delegates to the Engine methods. Write an integration test that creates a schedule, verifies it fires, verifies overlap policy prevents concurrent executions, and verifies recovery re-arms timers after restart.

## Requirements

### R1: gRPC schedule endpoints

THE SYSTEM SHALL add gRPC service methods for schedule management matching the Engine API: CreateSchedule, UpdateSchedule, PauseSchedule, ResumeSchedule, DeleteSchedule, ListSchedules, DescribeSchedule.

**Acceptance:**
- Proto definitions exist for all seven schedule RPCs.
- Server handlers delegate to Engine methods.
- Each RPC returns appropriate error codes for invalid input and not-found cases.

**Files:**
- modify: crates/aion-proto/src/lib.rs
- modify: crates/aion-server/src/grpc.rs

**Checklist:**
- C9

**Stories:**
- S6

### R2: HTTP schedule endpoints

THE SYSTEM SHALL add HTTP REST endpoints for schedule management: POST /schedules, PUT /schedules/:id, POST /schedules/:id/pause, POST /schedules/:id/resume, DELETE /schedules/:id, GET /schedules, GET /schedules/:id.

**Acceptance:**
- All seven HTTP endpoints exist and delegate to Engine methods.
- Responses use appropriate HTTP status codes (201 for create, 404 for not found, etc.).

**Files:**
- modify: crates/aion-server/src/http.rs

**Checklist:**
- C9

**Stories:**
- S6

### R3: Integration test: schedule lifecycle with recovery

THE SYSTEM SHALL have an integration test that: creates a schedule with a short interval, verifies it fires and starts a workflow, verifies overlap policy (Skip) prevents a concurrent start, and verifies that after simulated recovery the schedule re-arms and fires again.

**Acceptance:**
- Schedule creation succeeds and returns a ScheduleId.
- Within the expected interval, a workflow is started with the schedule's configured input.
- A ScheduleTriggered event is recorded.
- With overlap policy Skip and a still-running workflow, the next trigger is skipped.
- After recovery, the schedule re-arms and fires again.

**Files:**
- create: crates/aion/tests/schedule_lifecycle.rs

**Checklist:**
- C12

**Stories:**
- S1
