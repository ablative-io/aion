# AO-001: Wire non-resident signal delivery through residency-aware routing

> **Cluster:** aion-operations
> **Depends on:** AX-005
> **Checklist:**
> - C1 — Engine signal() branches on workflow residency: Resident delivers immediately, NonResident records and defers, Terminal returns typed error, Unknown returns WorkflowNotFound.
> - C2 — SignalResumeHandoff::deliver_deferred() is invoked when a workflow transitions from Suspended to Resident, delivering all queued signals in FIFO order.
> - C3 — A signal to a non-resident workflow appears in the workflow's durable history as a SignalReceived event even before the workflow resumes.
> - C4 — On engine restart, in-memory SignalResumeHandoff is empty; AD replay re-delivers recorded signals from history during deterministic replay (no double-delivery).
> **Stories:**
> - S7 (Workflow Author, Writing workflows that use signals for coordination) — As a workflow author, I want signals sent to my workflow while it is suspended to be delivered when it resumes so that I do not lose coordination messages from external systems.
> - S8 (Workflow Author, Writing workflows that use signals for coordination) — As a workflow author, I want a clear error when I signal a terminal workflow so that I can distinguish between 'not yet delivered' and 'permanently unreachable'.

## Purpose

Complete the signal delivery contract so that signals to suspended workflows are recorded and delivered on resume, matching Temporal's guarantee that signals reach their target regardless of workflow residency state.

## Task

Extend ConcreteSignalRouter to branch on workflow residency (Resident/NonResident/Terminal) and wire the existing SignalResumeHandoff for deferred delivery. Add a resume-transition hook in the engine that flushes deferred signals when a workflow becomes resident. The SignalResumeHandoff struct already exists, is tested, and is correct — only the wiring into the engine's signal flow and resume path is missing. IMPORTANT: 'non-resident' here means HandleResidency::Suspended — the WorkflowHandle still exists in the Registry (with its single-writer Recorder), but the BEAM process is not running. If the handle is entirely missing from the Registry, that is Unknown (WorkflowNotFound), not NonResident.

## Requirements

### R1: ConcreteSignalRouter branches on residency

WHEN Engine::signal() is called, THE SYSTEM SHALL resolve workflow residency and route accordingly: Resident records and delivers to mailbox, NonResident records and defers via SignalResumeHandoff, Terminal returns SignalRouterError::Terminal, Unknown returns EngineError::WorkflowNotFound. THE SYSTEM SHALL NOT deliver to a non-resident workflow's mailbox. THE SYSTEM SHALL NOT record an event for a terminal or unknown workflow.

**Acceptance:**
- Signalling a resident workflow records SignalReceived and delivers a mailbox marker (existing behaviour preserved).
- Signalling a non-resident workflow records SignalReceived in the workflow's durable history and registers the signal in SignalResumeHandoff.
- Signalling a terminal workflow returns a typed error and appends no event.
- Signalling an unknown workflow returns WorkflowNotFound and appends no event.
- The router does not call deliver_signal_received for non-resident targets.

### R2: Engine holds SignalResumeHandoff and exposes it to the router

THE SYSTEM SHALL hold a shared SignalResumeHandoff instance in the Engine, accessible to the ConcreteSignalRouter for deferral and to the resume path for delivery. THE SYSTEM SHALL NOT create multiple handoff instances per engine lifecycle.

**Acceptance:**
- Engine struct has a field for Arc<SignalResumeHandoff>.
- ConcreteSignalRouter receives the handoff reference at construction time.
- EngineBuilder wires the handoff through signal_router_factory.
- The same handoff instance is used by both the router and the resume path.

### R3: Resume transition flushes deferred signals

WHEN a workflow transitions from Suspended to Resident (handle re-registered after recovery or explicit resume), THE SYSTEM SHALL invoke SignalResumeHandoff::deliver_deferred() for that workflow, delivering all queued signals in FIFO order through the workflow's mailbox. THE SYSTEM SHALL NOT block the resume path on signal delivery failure — failed deliveries are logged and dropped (the signal is already recorded; replay will re-deliver).

**Acceptance:**
- After a workflow is made resident, all previously deferred signals are delivered to its mailbox.
- Signals are delivered in the order they were deferred (FIFO).
- A delivery failure for one signal does not prevent delivery of subsequent signals.
- On engine restart, the handoff is empty — no stale signals carried across lifecycles.

## Boundaries

- SHALL NOT implement signal persistence beyond the existing event recording — durability is handled by AD replay, not by persisting the handoff queue.
- SHALL NOT change the SignalResumeHandoff API — it is already correct and tested.
- SHALL NOT implement non-resident signal delivery for the schedule coordinator workflow.
- SHALL NOT add dependencies to the aion engine crate.
