# WA-006: Agentic-first authoring: aion new agent scaffold

> **Cluster:** aion-authoring
> **Depends on:** WA-001
> **Design anchor:**
> - ADR-003 — No default timeouts anywhere in the engine — The engine imposes no activity time bound of its own. Activity waits are unbounded, terminated only by completion, worker loss, server shutdown, or a workflow-level timeout the author explicitly chose. Rejected: a bigger default — agentic activities legitimately run for over an hour, and any number we picked would be ADR-001 violated.
> **Checklist:**
> - C25 — `aion new agent` scaffolds a durable agent loop (scout -> act -> verify -> signal-gated review) parameterised by prompts + schemas + gate, generalising the stacked-dev shape.
> - C26 — The agent scaffold compiles and runs a trivial agent end-to-end, including a human-in-the-loop signal wait with a timeout.
> - C27 — The agent scaffold's human approval pause is a workflow.receive with a timeout, not a bespoke polling mechanism.
> **Stories:**
> - S12 (Human Developer, Authoring a durable agent and proving determinism) — As a developer, I want to scaffold a durable agent loop with a human approval pause from prompts, schemas, and a gate, so that a new agentic family is configuration rather than bespoke code.

## Purpose

Generalises the dogfooded stacked-dev shape into a scaffold (RM-028) so a new agentic family is configuration, not bespoke code. Human-in-the-loop is just workflow.receive with a timeout — LangGraph's headline feature is already an Aion primitive — and durable suspend-for-weeks is already free. This is where the market is sprinting and where Aion's first consumer already lives.

## Task

Scaffold a durable agent loop from configuration. In: `aion new agent` that scaffolds a durable agent loop (scout -> act -> verify -> signal-gated review) parameterised by prompts + schemas + gate, with the human approval pause expressed as a workflow.receive with a timeout; and an end-to-end proof that the scaffold compiles and runs a trivial agent including the signal wait. Out: bundling a specific agent runtime (the norn driver stays worker-side, ADR-011) and any default timeout on the agent steps or the approval wait (ADR-003).

## Requirements

### R1: Scaffold the durable agent loop

WHEN `aion new agent` runs, THE SYSTEM SHALL scaffold a durable agent loop (scout -> act -> verify -> signal-gated review) parameterised by prompts + schemas + gate. The human approval pause SHALL be a workflow.receive with a timeout, not a bespoke polling mechanism.

**Acceptance:**
- `aion new agent` emits a runnable agent-loop package parameterised by prompts, schemas, and a gate (C25).
- The scaffold's human approval pause is a workflow.receive with a timeout (C27).
- The scaffold does not bundle a specific agent runtime; the agent step is parameterised (ADR-011).

### R2: The scaffold compiles and runs end-to-end

WHEN the scaffolded package is built, THE SYSTEM SHALL compile and run a trivial agent end-to-end, including the human-in-the-loop signal wait that resolves on a signal and times out otherwise.

**Acceptance:**
- The scaffold compiles under gleam build and the worker build (C26).
- A trivial agent runs end-to-end through scout -> act -> verify -> review.
- The approval signal wait resolves when a signal arrives and times out when none does within the configured deadline.

## Boundaries

- SHALL NOT bundle a specific agent runtime into the scaffold — the agent step is parameterised; the norn driver stays worker-side (ADR-011).
- SHALL NOT add a default timeout to the agent steps or the approval wait (ADR-003) — deadlines are author-chosen.
- SHALL NOT reimplement the codegen of WA-001; it consumes it.
