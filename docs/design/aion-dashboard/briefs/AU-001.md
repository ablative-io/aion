## Purpose

Stand up apps/aion-dashboard as a workspace member on the exact apps/web stack, so every later brief builds on a familiar, house-consistent foundation that the aion-server can serve as static assets.

## Task

Create the apps/aion-dashboard Vite single-page application matching apps/web: React 19, TypeScript, Tailwind v4, Bun, Biome (100-char), shadcn/Radix primitives. Wire house scripts, a base-path-configurable build, the SPA entry, and the shadcn primitives directory. No application features yet — this is the shell of the project.

## Requirements

### R1: Workspace member and house scripts

THE SYSTEM SHALL create apps/aion-dashboard as a workspace member whose package.json declares the same script set as apps/web (dev, build, preview, check, format, typecheck) and the same toolchain (React 19, TypeScript, Vite, Tailwind v4, Bun, Biome).

**Acceptance:**
- apps/aion-dashboard/package.json exists, is registered in the workspace, and declares dev (vite), build (vite build), preview, check (biome check .), format, and typecheck (tsc --noEmit).
- Dependencies match the apps/web stack versions (React 19, Vite 7, Tailwind v4, @biomejs/biome, TanStack Query); no alternate framework, styling system, or state library is introduced.
- biome.json mirrors apps/web's config including the 100-char line width.
- tsconfig.json mirrors apps/web's TypeScript configuration.
- bun install resolves cleanly for the new workspace member.

### R2: Vite build with configurable base path and SPA entry

THE SYSTEM SHALL configure Vite to build a static SPA bundle and to accept a configurable base path so aion-server (AW) can serve the bundle under a sub-path. THE SYSTEM SHALL provide the SPA entry (index.html) and the React root mount (src/main.tsx).

**Acceptance:**
- vite.config.ts uses the React and Tailwind v4 plugins and reads the base path from an env var (defaulting to '/') so AW can mount the bundle under a sub-path.
- index.html is the SPA entry referencing src/main.tsx.
- src/main.tsx mounts the React root into the document.
- bun run build produces a static bundle in dist/.
- tsc --noEmit passes on the scaffold.
