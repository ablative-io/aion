## Purpose

Define the Nif descriptor that carries a NIF's identity, native shim, dirty flag, and determinism tag, and the deterministic_nif declaration surface — the inline, never-recorded path. This is where the author's ordinary typed Rust function becomes a beamr NativeFn without the author ever touching a raw term.

## Task

In descriptor.rs, define the Determinism enum (Pure | SideEffectful) and the Nif descriptor (module name, function name, arity, NativeFn, dirty flag, Determinism tag) with an accessor for the tag so the engine can choose binding mode. In declare.rs, implement deterministic_nif: a builder/macro that takes a Rust fn/closure whose arguments are FromTerm and whose return is IntoTerm, and generates the NativeFn shim performing arity check, per-argument typed decode (reporting the failing index), body invocation, IntoTerm return encoding, and panic containment. It produces a Nif tagged Determinism::Pure.

## Requirements

### R1: Determinism tag and Nif descriptor

THE SYSTEM SHALL define Determinism as an enum with exactly Pure and SideEffectful, and a Nif descriptor recording module name, function name, arity, the NativeFn shim, a dirty-scheduler flag, and a Determinism tag, with an accessor exposing the tag.

**Acceptance:**
- descriptor.rs defines enum Determinism { Pure, SideEffectful } deriving Clone, Copy, Debug, PartialEq, Eq.
- descriptor.rs defines a Nif struct holding the module atom-name, function atom-name, arity (u8), the NativeFn (beamr's fn(&[Term], &mut ProcessContext) -> Result<Term, Term>), a dirty: bool flag, and a determinism: Determinism field.
- Nif exposes accessors for determinism(), is_dirty(), and its module/function/arity identity, so the engine's registration path (AE) can choose inline-binding for Pure and activity-body for SideEffectful.
- A doc comment states the engine reads the Determinism tag to decide binding mode and that a SideEffectful NIF is never bound inline.

### R2: deterministic_nif declaration

THE SYSTEM SHALL provide deterministic_nif, taking a Rust fn/closure with FromTerm arguments and an IntoTerm return, generating the NativeFn shim and producing a Nif tagged Determinism::Pure. The generated shim SHALL check arity, decode each argument by position, invoke the body, and encode the return, so author code never receives a raw &[Term].

**Acceptance:**
- declare.rs provides deterministic_nif (a declarative macro and/or generic builder) that, given a module name, function name, and a typed body, yields a Nif with determinism = Pure.
- The generated shim verifies the received argument slice length matches the declared arity and returns a TermError-derived error term on mismatch.
- Each argument is decoded via FromTerm by position; a decode failure reports the zero-based failing index (per AN-001 TermError) and short-circuits without calling the body.
- The body's return value is encoded via IntoTerm through the raw seam; the author body signature is ordinary typed Rust (e.g. fn(u64, String) -> SomeSerializableType), never fn(&[Term], ...).
- A unit test declares a pure helper (e.g. concatenating two decoded strings), invokes the generated NativeFn with term arguments through a ProcessContext, and asserts the decoded result and the Pure tag.

### R3: Heap-safe return and panic containment

THE SYSTEM SHALL ensure the generated return encoding is heap-safe (uses the raw module's ProcessContext allocation path, never Box::leak in an author-reachable path) and SHALL contain a panic raised inside the author body, converting it to a typed NIF error term rather than unwinding across the FFI boundary.

**Acceptance:**
- Return encoding goes through the raw module's safe allocators; no Box::leak appears in the generated shim or any author-reachable path.
- The generated shim wraps the body invocation so a panic is caught (catch_unwind at the FFI boundary) and converted to an error term carrying a descriptive message, never propagating a panic into beamr.
- A unit test declares a helper whose body panics, invokes the NativeFn, and asserts it returns an error term (the VM/test does not abort).
- A doc comment records that panic containment exists so a single buggy NIF cannot crash a scheduler thread.

## Boundaries

- SHALL NOT implement activity_nif (the SideEffectful path) — that is AN-005, though it reuses this descriptor and the same shim-generation machinery.
- SHALL NOT implement NifSet / registration — that is AN-006.
- SHALL NOT call beamr's BifRegistryImpl::register; deterministic_nif only produces a Nif descriptor.
