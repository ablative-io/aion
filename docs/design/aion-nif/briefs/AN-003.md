# AN-003: Payload and JSON bridge — terms over aion-core Payload and serde structs

> **Cluster:** aion-nif
> **Depends on:** AN-002

> - C15 — The payload module is the only place aion-nif references aion-core's Payload and ActivityError.
> **Stories:**
> - S2 (In-VM Activity Author, Writing Native Rust NIFs for Workflows) — As an in-VM activity author, I want to write a NIF directly over my own Serialize/Deserialize structs so that workflow values marshal across the boundary without manual JSON glue.

## Purpose

Connect the term layer to the engine's value model. Workflow values cross the NIF boundary as aion-core::Payload; this bridge converts a Payload to and from a Term and lets a NIF author work directly over ordinary Serialize/Deserialize Rust structs instead of hand-marshalling JSON.

## Task

In the payload module, implement Payload<->Term conversion via the JSON content-type (a term decodes to serde_json::Value through beamr's term_to_value, then into a Payload; and the reverse). Provide a blanket FromTerm/IntoTerm for any T: Serialize + DeserializeOwned routed through Payload, so a NIF can take and return its own structs. This module is the single place aion-nif references aion-core's Payload and ActivityError.

## Requirements

### R1: Payload <-> Term conversion

THE SYSTEM SHALL convert an aion-core Payload to a Term and a Term to a Payload via the JSON content-type, round-tripping losslessly. THE SYSTEM SHALL use beamr's term_to_value / value_to_term to bridge through serde_json::Value.

**Acceptance:**
- payload.rs converts a Term to a Payload by decoding the term to serde_json::Value (term_to_value) and constructing a JSON-tagged Payload (Payload::from_json), and converts a Payload to a Term by reading its JSON value (Payload::to_json) and encoding via value_to_term through the context.
- A round-trip test (Payload -> Term -> Payload) holds for payloads wrapping null, bool, number, string, array, and object JSON values.
- A non-JSON-content-type Payload is handled explicitly (documented behaviour: error or pass-through), not silently mis-decoded.
- Conversion errors surface as TermError, not panics.

### R2: Serde blanket bridge

THE SYSTEM SHALL provide a way to decode any T: DeserializeOwned from a Term and encode any T: Serialize into a Term, routed through Payload, so a NIF body can be written over ordinary Rust structs without manual term or JSON handling.

**Acceptance:**
- payload.rs provides functions (or a wrapper type) from_term_via_payload::<T>(term, ctx) and into_term_via_payload::<T>(value, ctx) for T: Serialize + DeserializeOwned, implemented in terms of the Payload<->Term conversion from R1.
- A deserialisation failure (term JSON does not match T) yields a TermError carrying the serde message, never a panic.
- A unit test round-trips a small derived struct (a couple of fields of mixed types) through a Term and asserts equality.
- A doc comment shows the intended use: an author writes a NIF taking and returning their own struct, and the declaration macros (AN-004) use this bridge to marshal it.

## Boundaries

- SHALL NOT define the Nif descriptor, Determinism tag, or declaration macros — those are AN-004.
- SHALL NOT define ActivityError handling for side-effectful NIFs beyond importing the type — that is AN-005.
- SHALL be the only module that imports aion-core::Payload; other modules use the term traits, not Payload directly.
