# AN-007: NifContext scoped storage — replace global TERM_STORAGE

> **Cluster:** aion-nif
> **Depends on:** AN-004, AN-005
> **Checklist:**
> - C32 — NifContext wraps ProcessContext and owns scoped term storage that lives for exactly one NIF invocation; no global static or Mutex is used for term retention.
> - C33 — FromTerm and IntoTerm trait signatures take NifContext (or its constituent references) instead of bare ProcessContext, so heap-backed encoding routes through scoped storage rather than the global seam.
> - C34 — The generated declaration shims (deterministic_nif, activity_nif) construct a NifContext at call entry and drop it at call exit, bounding retained storage to one invocation.
> - C35 — The global TERM_STORAGE static and clear_term_storage are removed; no global mutable state remains in the raw module for term retention.
> **Stories:**
> - S9 (Reviewer, Auditing the Native Boundary) — As a reviewer, I want all unsafe confined to one documented FFI module with SAFETY comments so that I can audit the entire native-boundary risk surface in one place.

## Purpose

Replace the interim global TERM_STORAGE pattern with a scoped NifContext that owns term storage per NIF invocation. The global static Mutex accumulates heap allocations without bound under concurrent NIF calls from the BEAM scheduler — it was approved as a temporary bridge in AN-002 with the explicit constraint that it must not become permanent. NifContext eliminates the global state entirely: storage lives and dies with the NIF call.

## Task

Define NifContext wrapping ProcessContext plus a Vec of retained heap allocations. Update FromTerm and IntoTerm to take NifContext (or split references into ProcessContext + storage) so IntoTerm encodes through scoped storage rather than the global seam. Update the deterministic_nif and activity_nif macro shims to construct a NifContext at the start of each generated NativeFn and drop it when the function returns. Remove the global TERM_STORAGE static, clear_term_storage, and the begin_nif_call helper from raw.rs and declare.rs. Verify the change is contained to aion-nif internals — the declaration macro is the public surface; callers never see NifContext.

## Requirements

### R1: NifContext type

THE SYSTEM SHALL define NifContext wrapping a ProcessContext reference and an owned Vec of retained heap allocations, so term storage is scoped to one NIF invocation and freed when the context is dropped.

**Acceptance:**
- A new module (or addition to an existing module) defines NifContext holding a mutable reference to ProcessContext and a Vec<Box<[u64]>> for retained term heap slices.
- NifContext exposes access to the ProcessContext for atom table operations and term allocation.
- NifContext exposes a retain_heap method (or equivalent) that the raw module's owned_* functions use instead of the global static.
- When NifContext is dropped, its retained heap storage is freed (the Vec is dropped, deallocating all retained slices).
- A unit test constructs a NifContext, retains heap storage through an IntoTerm call, and asserts the storage is freed on drop (no leaked allocations).

**Files:**
- create or modify: crates/aion-nif/src/context.rs (or inline in an existing module)
- modify: crates/aion-nif/src/lib.rs

### R2: Update FromTerm and IntoTerm signatures

THE SYSTEM SHALL update the FromTerm and IntoTerm trait signatures so that IntoTerm encodes through NifContext's scoped storage rather than the global TERM_STORAGE. FromTerm may take either NifContext or continue with ProcessContext where only atom resolution is needed.

**Acceptance:**
- IntoTerm::into_term takes a mutable reference that provides both ProcessContext access and scoped heap retention (either &mut NifContext or a split borrow pattern).
- FromTerm::from_term takes a reference that provides ProcessContext access for atom resolution.
- All existing FromTerm/IntoTerm implementations (primitives, collections, Payload bridge) are updated to use the new signatures.
- Existing round-trip tests continue to pass with the updated signatures.
- The raw module's owned_* functions take the scoped storage reference instead of pushing to the global static.

### R3: Update declaration shims

THE SYSTEM SHALL update the deterministic_nif and activity_nif macro shims to construct a NifContext at call entry and drop it at call exit, so retained term storage is bounded to one NIF invocation.

**Acceptance:**
- The generated NativeFn shim in deterministic_nif constructs a NifContext from the received ProcessContext at the top of the function body.
- Argument decoding and return encoding route through the NifContext.
- The NifContext is dropped (implicitly or explicitly) when the NativeFn returns, freeing all retained heap storage from that invocation.
- The activity_nif shim follows the same pattern.
- The begin_nif_call helper in declare.rs is removed (its purpose was to clear global storage between calls; NifContext's scoped lifetime replaces it).
- Existing declaration tests (typed invocation, arity error, decode failure, panic containment) continue to pass.

### R4: Remove global TERM_STORAGE

THE SYSTEM SHALL remove the TERM_STORAGE global static, the clear_term_storage function, and the keep_owned_term helper that pushed to the global Vec. No global mutable state SHALL remain in the raw module for term retention.

**Acceptance:**
- The static TERM_STORAGE: OnceLock<Mutex<Vec<Box<[u64]>>>> is removed from raw.rs.
- The pub fn clear_term_storage() is removed from raw.rs.
- The begin_nif_call() function is removed from declare.rs.
- The raw module's owned_* functions take a scoped storage parameter instead of using the global static.
- cargo clippy --workspace --all-targets -- -D warnings passes (no dead code from removed globals).
- A grep for TERM_STORAGE across the crate confirms zero references remain.

## Boundaries

- SHALL NOT change the public declaration macro API (deterministic_nif!, activity_nif!) — the NifContext is an internal implementation detail constructed and dropped by the generated shim.
- SHALL NOT expose NifContext in any public function signature that NIF authors interact with — the macro is the public surface.
- SHALL NOT introduce any unsafe outside raw.rs.
- SHALL NOT change the Nif descriptor or NifSet types — those are unchanged by this refactor.
